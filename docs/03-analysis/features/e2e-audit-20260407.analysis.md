## 교훈 및 패턴

### Mock 스키마 drift 방지
- API 타입 변경 시 E2E mock도 함께 갱신 필수
- `PortfolioSummary` 구버전/현행 스키마 불일치가 런타임 에러 유발

### API 경로 정확성
- Offering API: `/ax-bd/offerings/` 가 아닌 `/offerings/` (접두사 없음)
- Sections 응답: `{ sections: [] }` wrapping 필요 (배열 직접 반환 아님)
- Export: `fetchApi` 아닌 `fetch` + `text()` 반환 — mock도 `body` + `contentType` 사용

### 온보딩 투어 간섭
- 새 라우트 테스트 시 `localStorage.setItem("fx-tour-completed", "true")` 필수
- 투어 팝업이 콘텐츠를 가려 assertion 실패 유발

### strict mode violation
- `getByText("AI Foundry")` 같은 부분 텍스트가 2곳에 매치될 때 `.first()` 사용
- 또는 `{ exact: true }` 옵션으로 정확 매치
