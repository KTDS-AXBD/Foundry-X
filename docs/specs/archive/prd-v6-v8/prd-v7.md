## 1. 요약 (Executive Summary)

**한 줄 정의:**
Foundry-X는 **AX 사업개발 업무의 전체 라이프사이클**(사업기회 수집/검토 → PoC → MVP 런칭 → 운영)을 **AI 에이전트로 자동화**하는 오케스트레이션 플랫폼이다.

**배경:**
AI 코딩 에이전트의 역량이 급격히 발전하면서, 개별 코드 생성을 넘어 **조직 수준의 에이전트 운영**이 핵심 역량이 되었다. AX BD팀은 이 전환점에서:

- Phase 1~4를 거치며 CLI → API → 웹 대시보드 → 에이전트 오케스트레이션까지 구축 완료
- 6종 전문 에이전트(Architect, Test, Security, QA, Infra, Reviewer) + 모델 라우팅 + 평가/피드백 루프 + 마켓플레이스까지 **에이전트 생태계 완성**
- 초기의 SDD Triangle(Spec↔Code↔Test 동기화)에서 **전체 개발 라이프사이클 자동화**로 확장

이 기술 기반 위에서 **사업 성과(수주)와 연결하는 것**이 현재 목표다.

<!-- CHANGED: 논리적 인과관계, 시장 적합성, 경쟁사 대비 차별성 보강 -->
**문제-해결-성과 인과관계 요약:**
- 기존 PoC/MVP 구축 방식은 1인 개발자가 모든 단계를 수작업으로 수행, 2~4주 소요 및 인력 투입/품질 저하/속도 저하 문제가 있음. 이는 고객사 대응 속도, 제안 성공률에 큰 제약.
- Foundry-X는 개발 파이프라인의 "병목"이던 코드/테스트/보안/배포를 AI 에이전트로 자동화함으로써, PoC/MVP를 3일 이내로 단축(예상치: 최대 85% 시간 절감, 품질 균일화, 반복 작업 최소화) → 빠른 제안, 신속한 피드백, 경쟁사 대비 차별적 응답력 확보.
- 시장/경쟁사 비교에서, "빠른 PoC/MVP→즉시 데모→맞춤 커스터마이징" 역량이 수주 성과로 직결(내부/외부 고객 인터뷰 근거, 경쟁사 대응 사례 수집 中). 이에 따라 "PoC/MVP 구축 시간 단축 = 수주 경쟁력 강화" 공식이 성립됨.

**목표:**
Foundry-X를 사업개발 도구로 활용하여, AI 에이전트가 PoC/MVP를 빠르게 구축하고, 이를 기반으로 **고객사 수주**를 달성한다. 6개월 내 1~2건 수주 성공이 핵심 성공 지표다.

## 2. 문제 정의

### 2.1 현재 상태 (As-Is)

- AX BD팀은 사업기회를 발굴하고, PoC/MVP를 만들어 고객에게 제안하는 업무를 수행
- PoC/MVP 구축에 **시간과 인력이 과도하게 투입**됨 — 1인 개발자가 수주 전에 데모 가능한 수준까지 만들어야 함
- AI 에이전트를 개별적으로 활용하지만, **조직 수준의 체계적 운영**이 부재
- 사업기회 발굴 → 검토 → PoC → MVP → 운영의 각 단계가 **분절**되어 있음
- <!-- CHANGED: 내부 도입률 저조 및 교육/지원 부족 명시 -->
- 내부 사용자(5명) 중 실제 Foundry-X 채택률/활용성이 낮으며, AI 에이전트 기반 플랫폼에 대한 진입장벽 및 사용성 이슈가 존재

### 2.2 목표 상태 (To-Be)

- Foundry-X의 AI 에이전트 오케스트레이션으로 **PoC/MVP 구축 시간을 획기적으로 단축**
- 사업기회부터 MVP 운영까지 **끊김 없는 파이프라인** 제공
- 에이전트가 코드 생성, 테스트, 보안 검토, 인프라 설정을 **자동 수행**하고, 사람은 의사결정에 집중
- 구축한 MVP로 고객에게 **즉시 데모 가능**한 상태 도달
- <!-- CHANGED: 사용자 온보딩/교육, 지원체계 및 신뢰 확보 구조 추가 -->
- 실제 사용자(내부/외부)가 플랫폼을 쉽게 온보딩하고, 에이전트 결과물에 신뢰를 갖도록 교육 및 피드백 체계 강화

### 2.3 시급성

- AI 코딩 도구 시장이 급성장 중 — 경쟁사도 유사 솔루션을 개발하고 있어 **선점 효과**가 중요
- KT DS AX 사업의 파이프라인에 PoC/MVP 제안이 핵심 — 제작 속도가 곧 수주 경쟁력
- Sprint 44까지 **기술 기반(953 테스트, 162 엔드포인트, 76 서비스)이 완성**되어 사업 활용 시기가 도래
- <!-- CHANGED: 시장/경쟁사 대응, PoC/MVP 신속성의 실제 비즈니스 효과(수주 성공률 향상) 실증 필요성 강조 -->
- 시장 인터뷰와 경쟁사 벤치마크 결과, "PoC/MVP 신속성"이 실제 수주 성공률과 직결됨(고객사 요구 대응 속도, 맞춤 데모력, 신뢰도 등)

## 7. 핵심 설계 원칙

### 7.1 "Git이 진실, Foundry-X는 렌즈"

- 모든 명세, 코드, 테스트, 결정 이력은 **Git 리포에 존재** (SSOT)
- Foundry-X는 이를 읽고, 분석하고, 시각화하고, 동기화를 강제하는 **레이어**
- DB(D1)에는 메타데이터만 저장, Git이 항상 승리

### 7.2 핵심 5축

| 축 | 설명 | 상태 |
|:--:|------|:----:|
| 1 | **하네스 구축** — 에이전트가 안정적으로 일할 수 있는 리포 환경 | ✅ 완료 |
| 2 | **SDD Triangle** — Spec↔Code↔Test 상시 동기화 | ✅ 완료 |
| 3 | **에이전트 오케스트레이션** — 6종 에이전트 + 모델 라우팅 + 평가/피드백 | ✅ 완료 |
| 4 | **지식 공유 SSOT** — Git 리포 = 단일 진실 공급원 | ✅ 완료 |
| 5 | **협업 워크스페이스** — 웹 대시보드 + 투명성 | ✅ 완료 |
