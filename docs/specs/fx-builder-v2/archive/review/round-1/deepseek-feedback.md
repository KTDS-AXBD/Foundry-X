## 기술 검토 의견: Prototype Builder v2 PRD

### 1. 기술적 실현 가능성
**부분적으로 가능하나 중대한 제약 존재**

**긍정적 요소:**
- LLM 기반 평가 체계(Vision API, 의미적 PRD 비교)는 기술적으로 실현 가능
- WSL + max-cli 로컬 실행은 비용 문제 해결에 효과적 접근
- impeccable 참조문서 통합은 프롬프트 엔지니어링으로 구현 가능

**주요 기술적 제약:**
1. **Vision API 비용 및 성능 병목**
   - 스크린샷 평가 시 토큰 소모량이 예상보다 클 수 있음(고해상도 이미지)
   - 평가 지연 시간 증가로 O-G-D 루프 전체 성능 저하 우려
   - "빌드당 $0" 목표는 Vision API 사용 시 달성 불가

2. **max-cli 운영 환경의 불안정성**
   - WSL 단일 머신에서의 큐 관리만으로는 프로덕션 신뢰성 확보 어려움
   - CLI 프로세스 크래시/메모리 누수 시 전체 시스템 중단 가능성
   - rate limit 회피 전략이 구체화되지 않음(백오프만으로 불충분)

3. **토큰 한도 초과 위험**
   - impeccable 7도메인 + 안티패턴 + 기존 프롬프트 = 시스템 프롬프트 폭발
   - Claude의 컨텍스트 윈도우 한계(200K 토큰) 내에서 모든 참조문서 포함 불가능

### 2. 아키텍처 적합성
**현재 구조는 확장성에 심각한 결함 존재**

**문제점:**
1. **모놀리식 평가 파이프라인**
   - 5개 차원(LLM) + Vision API 평가가 순차적 실행 → 지연 시간 가산
   - 부분 실패 시 전체 평가 실패로 이어지는 취약성
   - 평가 모듈 간 의존성 관리 체계 미비

2. **로컬 CLI 의존성의 아키텍처적 위험**
   - 빌드 시스템이 특정 개발자 머신(WSL)에 결합됨
   - 확장 시 다중 머신 분산 처리로의 전환 비용 과다
   - 상태 관리(큐, 로그, 결과)가 로컬 파일 시스템에 의존

3. **데이터 일관성 보장 부재**
   - O-G-D 루프의 다중 라운드에서 평가 기준 일관성 유지 메커니즘 없음
   - LLM 판별의 비결정성(non-determinism)으로 인한 수렴 불안정성

**제안 아키텍처 개선:**
```
[현재] PRD → Generator → Discriminator(모든 평가) → 결과
[제안] PRD → Generator → 평가 디스패처 → 병렬 평가기(Vision, PRD-LLM, UI-LLM 등) → 결과 집계기
```

### 3. 구현 복잡도
**심각하게 과소평가됨**

**숨겨진 복잡도:**
1. **LLM 평가의 재현성 보장**
   - 동일 입력에 대해 다른 점수 출력 가능성(LLM의 확률적 특성)
   - O-G-D 루프가 진동(oscillate)하며 수렴 실패 가능성 높음

2. **Vision API의 정량화 난이도**
   - "레이아웃 균형", "색상 조화" 등의 주관적 개념을 정량 점수로 변환의 불확실성
   - 평가 기준을 LLM이 이해할 수 있는 구체적 지시문으로 변환 필요

3. **에지 케이스 처리**
   - max-cli 프로세스 무응답 시 타임아웃/재시작 전략
   - 부분적 평가 실패 시 fallback 평가 전략
   - 매우 긴 PRD 문서 처리(토큰 초과 시 요약 필요)

4. **성능 최적화 필요**
   - 스크린샷 생성 → Vision API 전송 → 응답 대기 시간 최소화
   - 다중 LLM 호출의 병렬화 및 캐싱 전략

### 4. 기술 리스크
**상위 3개 우선순위 리스크:**

1. **수렴 실패 리스크 (위험도: 높음)**
   - LLM 기반 평가의 비결정성으로 O-G-D 루프가 진동하며 품질이 개선되지 않을 수 있음
   - "5라운드 내 80점 도달" 목표가 현실적으로 달성 불가능할 가능성

   - max-cli 업데이트/변경 시 호환성 문제로 전체 시스템 중단 가능성

3. **비용 예측 불가능성 (위험도: 중간)**
   - Vision API 비용이 예상보다 높아 경제적 타당성 상실 가능성
   - LLM 평가 토큰 사용량이 누적되어 월 예산($100) 초과 가능성
