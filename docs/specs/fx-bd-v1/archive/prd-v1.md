**버전:** v1
**날짜:** 2026-03-30
**작성자:** AX BD팀
**상태:** 🔄 검토 중

---

## 1. 요약 (Executive Summary)

**한 줄 정의:**
AX BD팀 6명이 사업 아이템을 수집부터 Offering Pack 생성까지 하나의 플랫폼에서 끊김 없이 추적·공유·의사결정할 수 있도록 Foundry-X의 6단계 파이프라인 갭을 보강한다.

**배경:**
Foundry-X에는 6단계(수집→발굴→형상화→검증/공유→제품화→GTM) 각각에 대한 기능이 부분적으로 구현되어 있으나, 단계 간 연결이 단절되어 전체 파이프라인이 매끄러운 흐름으로 작동하지 않는다. 특히 산출물 공유, 게이트 프로세스(ORB/PRB), Offering Pack 생성 기능이 부재하여 실무 적용이 불가능한 상태이다.

**목표:**
내부 온보딩(F114, 6명)과 연계하여 팀원이 실제 사업개발 업무를 Foundry-X 위에서 수행할 수 있는 환경을 구축한다.

---

## 2. 문제 정의

### 2.1 현재 상태 (As-Is)
- 각 단계의 핵심 기능(BMC, PRD, Prototype, Six Hats 등)은 API/서비스 수준에서 구현됨
- 단계 간 전환 워크플로가 없어, 사용자가 수동으로 단계를 이동해야 함
- 산출물(BMC, PRD, 프로토타입 등)에 대한 팀 내 공유/알림/리뷰 요청 기능 부재
- KT DS 내부 게이트 프로세스(ORB/PRB) 연계 없음
- 사업계획서(BDP) 편집/최종본 생성 없음 (초안 생성만 존재)
- Offering Pack(영업/제안용 패키지) 개념 자체가 미구현
- MVP 개발 추적 및 자동화 워크플로 부재

### 2.2 목표 상태 (To-Be)
- 사업 아이템이 1단계 수집부터 5단계 Offering Pack 생성까지 하나의 대시보드에서 추적 가능
- 각 단계 전환 시 자동 알림 + 리뷰 요청 워크플로 작동
- ORB/PRB 게이트 준비 문서가 기존 산출물에서 자동 생성
- 팀원 6명이 실제 사업개발 업무에 Foundry-X를 일상적으로 사용

### 2.3 시급성
- 내부 온보딩(F114) 진행 중 — 실질적 업무 적용이 가능해야 온보딩 의미가 있음
- 바로 구현 필요 (Phase 6~7에 포함)

## 4. 기능 범위

### 4.1 핵심 기능 (Must Have)

| # | 기능 | 단계 | 설명 | 우선순위 |
|---|------|------|------|----------|
| 1 | **파이프라인 통합 대시보드** | 전체 | 아이템별 현재 단계, 진행률, 다음 액션을 한 눈에 조회하는 대시보드. 칸반 또는 파이프라인 뷰 | P0 |
| 2 | **산출물 공유 시스템** | 2~5 | 링크 공유 + 팀 대시보드 통합 뷰 + 단계 전환 시 알림/리뷰 요청 | P0 |
| 3 | **BDP 편집/버전관리** | 3 | 사업계획서 초안→편집→최종본 워크플로. 마크다운 에디터 + 버전 히스토리 | P0 |
| 4 | **사업제안서 자동 생성** | 4 | 사업계획서에서 게이트 제출용 요약본 자동 생성 | P1 |
| 5 | **ORB/PRB 게이트 준비** | 4 | KT DS ORB→PRB 게이트 제출 문서 패키지 자동 구성 (사업제안서 + BMC + 평가결과 + 기술검증) | P0 |
| 6 | **Offering Pack 생성** | 5 | 영업/제안용 패키지 번들 — 사업제안서 + 프로토타입 데모 링크 + 기술검증 결과 + 가격 정보 | P0 |
| 7 | **MVP 추적 + 자동화** | 5 | MVP 개발 상태(In Dev/Testing/Released) 추적 + PoC 환경 자동 배포/테스트 파이프라인 | P1 |

### 4.2 부가 기능 (Should Have)

| # | 기능 | 단계 | 설명 | 우선순위 |
|---|------|------|------|----------|
| 8 | **IR Bottom-up 채널** | 1 | 사내 현장(엔지니어/영업) 제안 전용 등록 폼 → 사업 아이템 자동 변환 | P2 |
| 9 | **전문 검토 공유 뷰** | 2 | 사업화 전문 검토 결과(Six Hats, AI 리뷰 등)를 팀 전체가 조회하는 전용 뷰 | P2 |
| 10 | **단계 전환 자동화** | 전체 | Commit Gate 통과 시 자동 단계 전환 + 다음 단계 액션 안내 | P2 |

### 4.3 제외 범위 (Out of Scope)
- **CRB/ERB 게이트**: KT DS 계약/실행 단계 — 다른 팀 영역
- **GTM 별도 정의**: Offering Pack 생성+공유로 대체, 별도 GTM 모듈은 이번 범위 외
- **외부 고객 포털**: 내부 팀용만 (외부 공유는 Offering Pack 링크로 대체)
- **사내 시스템 직접 연동**: ORB/PRB는 Foundry-X에서 문서 생성까지만, 사내 시스템 API 연동은 향후

### 4.4 외부 연동
| 시스템 | 연동 방식 | 필수 여부 |
|--------|-----------|-----------|
| Discovery-X | API (기존 ingest 확장) | 필수 |
| KT DS ORB/PRB | 문서 생성 (수동 제출) | 필수 |
| Cloudflare D1 | 기존 DB 확장 | 필수 |
| GitHub | 기존 연동 유지 | 선택 |
