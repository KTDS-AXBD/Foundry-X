# FX-BD-V1: AX BD 사업개발 파이프라인 End-to-End 통합 PRD

**버전:** final (v3 기반)

**작성자:** AX BD팀
**상태:** ✅ 착수 준비 완료

## 1. 요약 (Executive Summary)

**한 줄 정의:**
AX BD팀 6명이 사업 아이템을 수집부터 Offering Pack 생성까지 하나의 플랫폼에서 끊김 없이 추적·공유·의사결정할 수 있도록 Foundry-X의 6단계 파이프라인 갭을 보강한다.

**배경:**
Foundry-X에는 6단계(수집→발굴→형상화→검증/공유→제품화→GTM) 각각에 대한 기능이 부분적으로 구현되어 있으나, 단계 간 연결이 단절되어 전체 파이프라인이 매끄러운 흐름으로 작동하지 않는다. 특히 산출물 공유, 게이트 프로세스(ORB/PRB), Offering Pack 생성 기능이 부재하여 실무 적용이 불가능한 상태이다.

**목표:**
내부 온보딩(F114, 6명)과 연계하여 팀원이 실제 사업개발 업무를 Foundry-X 위에서 수행할 수 있는 환경을 구축한다.
<!-- CHANGED: 목표와 기능 간 일치성 보완 필요성 반영 -->
- 실무자가 아이템 수집부터 의사결정(Go/Hold/Drop) 및 Offering Pack 생성, 문서 공유와 리뷰, 게이트 통과까지 End-to-End로 업무를 수행할 수 있는 구체적 워크플로우 제공을 포함한다.
- 의사결정 지원 및 단계별 산출물 생성, 공유 및 리뷰, 내부 게이트(ORB/PRB) 문서 자동화, Offering Pack 패키징 등 모든 주요 프로세스를 한 플랫폼 내에서 처리할 수 있도록 한다.

## 1.1 주요 워크플로 (Step-by-Step 플로우 및 액터/시스템 액션)

### End-to-End Step-by-Step 시나리오 (AX BD팀원 관점)

| 단계 | Owner | 주요 액션 | 시스템 지원/자동화 | 산출물/이벤트 |
|------|-------|-----------|--------------------|--------------|
| 1. 아이템 등록 | 팀원 | 신규 아이템 등록, 1차 설명 입력 | 등록 폼, 자동 ID 부여, 알림 | 아이템 생성, 담당자 지정 |
| 2. 산출물 작성 | 팀원 | BMC, PRD 등 작성/업데이트 | 마크다운 에디터, 버전 관리 | 산출물 버전 생성 |
| 3. 산출물 공유/리뷰 | 팀원 → 팀장/팀원 | 리뷰 요청, 공유 링크 생성 | 알림(인앱/이메일), 리뷰 UI | 리뷰 코멘트, 승인/거절 |
| 4. 의사결정(Go/Hold/Drop) | 팀장 | 리뷰/평가 후 단계 결정 | Go/Hold/Drop 버튼, 이력 로그 | 상태 변경, 승인 알림 |
| 5. 게이트 문서 자동생성 | 팀원/팀장 | ORB/PRB 제출용 서류 자동 패키징 | 문서 번들 자동 생성, 다운로드 | 게이트 패키지 파일 |
| 6. Offering Pack 생성 | 팀원 | 최종 사업제안/프로토타입/가격 등 번들링 | 자동 패키징, 외부 공유 링크 | Offering Pack 번들 |
| 7. MVP 개발 추적 | 팀원 | MVP 진행상태 업데이트 | 상태 변경 UI, 자동 알림 | 상태 이력, 배포 알림 |

**플로우 차트(텍스트 버전):**

1. 등록(팀원) → 2. 산출물 작성(팀원) → 3. 공유/리뷰(팀원↔팀장) → 4. 의사결정(팀장) →  
5. ORB/PRB 문서 패키징(팀원/팀장) → 6. Offering Pack(팀원) → 7. MVP/PoC 진행(팀원)

## 2. 문제 정의

### 2.1 현재 상태 (As-Is)
- 각 단계의 핵심 기능(BMC, PRD, Prototype, Six Hats 등)은 API/서비스 수준에서 구현됨
- 단계 간 전환 워크플로가 없어, 사용자가 수동으로 단계를 이동해야 함
- 산출물(BMC, PRD, 프로토타입 등)에 대한 팀 내 공유/알림/리뷰 요청 기능 부재
- KT DS 내부 게이트 프로세스(ORB/PRB) 연계 없음
- 사업계획서(BDP) 편집/최종본 생성 없음 (초안 생성만 존재)
- Offering Pack(영업/제안용 패키지) 개념 자체가 미구현
- MVP 개발 추적 및 자동화 워크플로 부재

### 2.2 목표 상태 (To-Be)
- 사업 아이템이 1단계 수집부터 5단계 Offering Pack 생성까지 하나의 대시보드에서 추적 가능
- 각 단계 전환 시 자동 알림 + 리뷰 요청 워크플로 작동
- ORB/PRB 게이트 준비 문서가 기존 산출물에서 자동 생성
- 팀원 6명이 실제 사업개발 업무에 Foundry-X를 일상적으로 사용
<!-- CHANGED: 목표와 기능 간 Mapping 보완 -->
- 의사결정(Go/Hold/Drop) 지원을 위한 워크플로 및 이슈 트래킹, 리뷰 요청 절차 포함
- 산출물 공유, 알림(이메일/인앱), 내부 피드백 수렴 루프 내장
- 단계별 자동화(예: 단계 전환, 문서 생성, 알림 발송 등) 구현

### 2.3 시급성
- 내부 온보딩(F114) 진행 중 — 실질적 업무 적용이 가능해야 온보딩 의미가 있음
- 바로 구현 필요 (Phase 6~7에 포함)

#### KPI 현실성 명시
- "팀원 주간 사용률 100%"는 초기 온보딩(6명) 기준 목표이며, 실제 활용도 및 적응 기간을 고려해 3개월 후 80% 이상 유지로 재조정 가능(피벗 조건 참조)

### 5.4 QA/UAT/운영 기준

#### UAT(사용자수용테스트)
- 대상: BD팀 6명 전원
- 운영: 시범 운영(2주), 실업무 시나리오 기반 테스트(실제 사업 아이템 등록~Offering Pack까지 End-to-End)

#### 운영/장애 대응 체계
<!-- CHANGED: 운영/장애 대응 체계 상세 추가 -->

- 장애 유형별(긴급/일반) 대응 플로우: 슬랙/이메일 알림→임시 조치→긴급 공지(내부)→근본 원인 분석→복구/패치
- 장애 리포트/로그 자동 수집, 1주 내 원인/재발방지 대책 보고

#### 테스트/배포/롤백 계획
<!-- CHANGED: 테스트/배포/롤백 플랜 추가 -->
- 배포 전: 스테이징 환경에서 QA/UAT 완료 시점에만 본운영 배포

- 롤백: 장애 발생 시 이전 버전 신속 복구(배포 스크립트 내 롤백 옵션), 데이터 백업본 활용

#### 교육/Change Management
- 온보딩 완료 시점 가이드 세션 필수, 문의/피드백 채널(슬랙) 상시 운영
- 신규 배포/기능 추가 시, Release Note/FAQ/가이드 문서 동시 배포

| 데이터 백업/복구 정책 미흡 | D1 장애 시 데이터 유실 | 자동/수동 백업, 복구 시나리오/테스트 시행 |

### 9.1 시장 적합성 및 유사 서비스와의 차별성
- 파편화된 시스템과 수동 프로세스의 비효율 해소, 전 과정을 단일 플랫폼에서 관리하는 "Single Source of Truth" 전략 부합
- 단계 전환, 알림, 문서 자동화 등 프로세스 자동화 및 가시성 강화는 생산성 향상/애자일 트렌드와 일치
- 산출물(문서/프로토타입 등) 공유 및 통합 대시보드는 협업 및 지식 공유 강화 요구 반영
- IR Bottom-up 채널·전문 검토 뷰 등은 내부 혁신 아이디어 관리 플랫폼의 특성
- 유사 툴(Jira, Asana 등) 대비 6단계 파이프라인, ORB/PRB 게이트, BD 특화 기능(Offering Pack 등) 내재화로 차별성

### 9.2 AI 트렌드 및 활용 확장성
<!-- CHANGED: AI 활용의 확장성(시장/트렌드 관점 누락) 지적 반영 -->
- 기존 "AI 에이전트" 활용(문서 자동화, 리뷰)에 더해,
    - 단계별 AI 기반 산출물 리뷰 및 품질 보증(자동 피드백/제안)
    - AI 기반 사업 아이템 스코어링/의사결정 보조 기능 확장 고려
    - AI 활용 범위: 자동화, Notification, QA, 데이터 품질점검 등 점진 확대, 단, 반복 QA/리뷰 위주, 운영/승인/예외는 사람 개입

- *AI 기반 자동 사업 아이템 추천* 등은 Out-of-scope로 관리(향후 검토)

## 11. 현업 시나리오 기반 Use Case/테스트 시나리오
<!-- CHANGED: 현업 시나리오로 검증된 Use Case/테스트 시나리오 별도 섹션 신설 -->
### 11.1 End-to-End Use Case 예시

**A. 신규 사업 아이템 등록~Offering Pack 생성(팀원 1명 기준)**
1. 팀원 A, 신규 사업 아이템 등록(등록 폼 작성, 설명 입력)
2. BMC/PRD/프로토타입 등 산출물 작성(에디터)
3. 산출물 리뷰 요청(팀장/팀원 지정, 알림 발송)
4. 팀장/팀원 리뷰→코멘트→승인/반려(상태 변경)
5. 팀장, 단계별 의사결정(Go/Hold/Drop) 버튼 클릭
6. ORB/PRB 제출용 문서 자동 패키징(다운로드)
7. Offering Pack 최종 생성(패키징, 외부 공유 링크 생성)
8. MVP 개발 상태(진행/테스트/배포) 업데이트(알림)
9. 의사결정/이력/알림 UI에서 전체 내역 검증

**B. 장애 복구 시나리오**
1. 팀원이 문서 저장/단계 전환 중 장애 발생
2. 장애 알림→시스템 관리자 신속 대응→백업본 복구→서비스
