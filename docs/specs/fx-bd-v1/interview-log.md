## Part 3: 무엇을 (범위/기능)

### Prep HRD 정의
**Q: 다이어그램의 'Prep HRD'가 무엇인가요?**
> A: Pre-PRB (Proposal Review Board 준비). KT DS 내부 사업검토 프로세스:
> ORB(Opportunity Review Board) → PRB → CRB(Contract) → ERB(Execution) 4단계 게이트

**Q: Foundry-X에서 어디까지 지원?**
> A: ORB + PRB만. CRB/ERB는 다른 팀 영역.

### 사업제안서 vs 사업계획서
**Q: 사업제안서와 사업계획서(BDP)를 어떻게 구분하나요?**
> A: 사업제안서 = 사업계획서의 요약본 (게이트 제출용)

### Offering Pack
**Q: Offering Pack이 무엇인가요?**
> A: 영업/제안용 패키지 (사업제안서 + 프로토타입 데모 + 기술검증 결과 + 가격 등 번들)

### 공유 기능
**Q: 공유 기능의 형태는?**
> A: 3가지 모두 필요 — 링크 공유 + 대시보드 통합 뷰 + 알림/리뷰 요청

### MVP 개발 범위
**Q: MVP 개발을 Foundry-X에서 어떤 수준으로 지원?**
> A: 추적 + 자동화 (상태 추적 + PoC 자동 배포/테스트 파이프라인)

### GTM
**Q: 6단계 GTM은?**
> A: Offering Pack이 GTM 대체 — 별도 정의 없이 Offering Pack 생성+공유가 GTM 첫 단계

### IR Bottom-up
**Q: IR Bottom-up 수집이 무엇인가요?**
> A: 사내 현장(엔지니어/영업)이 Bottom-up으로 제안하는 사업 아이디어
