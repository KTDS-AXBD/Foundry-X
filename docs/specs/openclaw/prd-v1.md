## 1. 요약 (Executive Summary)

**한 줄 정의:**
OpenClaw는 AX BD팀 사업개발/기획 담당자(비개발자)가 **프롬프트 입력만으로 PoC/프로토타입을 자동 생성하고 배포까지 완료**할 수 있는 AI PoC 빌더 플랫폼이다.

**배경:**
AI 코딩 에이전트의 역량이 급격히 발전하면서, Bolt.new/Lovable 같은 "프롬프트→앱" 빌더가 시장에 등장했다. AX BD팀은 고객사에 PoC를 빠르게 보여줘야 하는데, 현재는 개발자가 수작업으로 2~4주 소요된다. OpenClaw는 Foundry-X의 AI 에이전트 기술 기반 위에, 비개발자도 프롬프트만으로 PoC를 생성하고 실제 배포까지 할 수 있는 도구를 만들어 이 병목을 해소한다.

**목표:**
비개발자가 요구사항을 입력하면 AI가 코드를 생성하고, 원클릭으로 Cloudflare에 배포하여 고객에게 바로 데모 URL을 전달할 수 있는 상태. PoC 제작 시간을 2~4주에서 **수 시간 이내**로 단축한다.

---

## 2. 문제 정의

### 2.1 현재 상태 (As-Is)
- PoC/프로토타입 제작에 **2~4주 소요** (1인 개발자 수작업)
- 사업개발 담당자는 개발 역량이 없어 직접 PoC를 만들 수 없음
- 고객 요구사항 접수 → 개발자 전달 → 스케줄 조율 → 개발 → 배포 과정에 병목 발생
- Bolt.new, Lovable 같은 외부 서비스는 사내 보안 정책/기술 스택과 맞지 않음

### 2.2 목표 상태 (To-Be)
- 비개발자가 **웹 UI에서 프롬프트 입력** → AI가 코드 생성 → **원클릭 배포**
- PoC 제작 시간: **수 시간 이내** (기존 대비 85%+ 단축)
- 고객에게 즉시 데모 URL 전달 가능
- Foundry-X 기술 스택(Cloudflare) 위에서 운영되어 사내 보안 정책 준수

### 2.3 시급성
- 내부 로드맵 상 예정된 작업 (높은 시급성은 아님)
- 다만 AI 빌더 시장이 빠르게 성장 중이므로, 사내 역량 확보 차원에서 조기 착수가 유리
- Foundry-X Phase 5(고객 파일럿) 이후 자연스러운 확장 방향

## 3. 사용자 및 이해관계자

### 3.1 주 사용자
| 구분 | 설명 | 주요 니즈 |
|------|------|-----------|
| AX BD팀 사업개발 담당 | 비개발자, 고객 요구사항을 받아 PoC를 제안해야 함 | 코딩 없이 빠르게 PoC 생성 + 배포 + 데모 URL 공유 |
| AX BD팀 기획 담당 | 비개발자, 아이디어를 프로토타입으로 검증해야 함 | 프롬프트로 아이디어를 즉시 동작하는 앱으로 전환 |

### 3.2 이해관계자
| 구분 | 역할 | 영향도 |
|------|------|--------|
| 팀 리더 | PoC 품질 승인, 고객 제안 의사결정 | 높음 |
| 고객사 담당자 | PoC 데모 수혜자, 피드백 제공 | 높음 |
| 개발자 (현재) | PoC 개발 병목 → OpenClaw로 부하 감소 | 중간 |

### 3.3 사용 환경
- 기기: **PC** (웹 브라우저)
- 네트워크: **인터넷** (Cloudflare 기반)
- 기술 수준: **비개발자** (코딩 경험 없음)

## 4. 기능 범위

### 4.1 핵심 기능 (Must Have)
| # | 기능 | 설명 | 우선순위 |
|---|------|------|----------|
| 1 | 프롬프트 입력 UI | 요구사항을 자연어로 입력하는 웹 인터페이스. 예시 프롬프트 제공 | P0 |
| 2 | AI 코드 생성 | 프롬프트 기반으로 풀스택 앱 코드(프론트엔드+백엔드) 자동 생성 | P0 |
| 3 | 실시간 프리뷰 | 생성 중/생성 후 결과물을 브라우저에서 바로 확인 | P0 |
| 4 | 원클릭 배포 | 생성된 코드를 Cloudflare Pages/Workers에 자동 배포 + URL 발급 | P0 |
| 5 | 반복 수정 | "이 부분 수정해줘" 대화형 수정 → AI가 코드 업데이트 → 재프리뷰 | P0 |

### 4.2 부가 기능 (Should Have)
| # | 기능 | 설명 | 우선순위 |
|---|------|------|----------|
| 1 | 프로젝트 관리 | 생성한 PoC 목록 조회, 이력 관리 | P1 |
| 2 | 템플릿 라이브러리 | 자주 쓰는 PoC 유형(대시보드, CRUD, 챗봇 등) 템플릿 | P1 |
| 3 | 공유 링크 | 생성된 데모 URL을 팀/고객에게 공유 + 접근 권한 관리 | P1 |
| 4 | 코드 다운로드 | 생성된 코드를 zip/git으로 다운로드 | P2 |
| 5 | Foundry-X 연동 | Foundry-X 에이전트가 OpenClaw로 PoC 생성 자동화 | P2 |

### 4.3 제외 범위 (Out of Scope)
- **프로덕션 수준 앱 개발**: OpenClaw는 PoC/데모 목적. 프로덕션 배포는 별도 프로세스
- **커스텀 백엔드 인프라**: MVP에서는 Cloudflare 기본 인프라만 사용
- **멀티 사용자 실시간 협업**: 1인 작업 기준, 협업은 후속
- **모바일 앱 생성**: 웹 앱만 지원 (MVP)

### 4.4 외부 연동
| 시스템 | 연동 방식 | 필수 여부 |
|--------|-----------|-----------|
| Cloudflare Pages/Workers | API (wrangler) | 필수 — 배포 대상 |
| AI LLM (Claude/GPT) | API | 필수 — 코드 생성 엔진 |
| Foundry-X | [미확인] | 선택 — 후속 연동 |

## 5. 성공 기준

### 5.1 정량 지표 (KPI)
| 지표 | 현재값 | 목표값 | 측정 방법 |
|------|--------|--------|-----------|
| PoC 생성 시간 | 2~4주 | **수 시간 이내** | 프롬프트 입력~배포 완료 시간 측정 |
| 비개발자 자립 생성 비율 | 0% | **80%+** | 개발자 도움 없이 완성한 PoC 비율 |
| 고객 데모 전환율 | [미확인] | [미확인] | OpenClaw PoC로 데모 진행한 건수 |

### 5.2 MVP 최소 기준
- [ ] 비개발자가 프롬프트 입력 → 동작하는 웹 앱 코드 생성
- [ ] 생성된 코드를 원클릭으로 Cloudflare에 배포
- [ ] 배포된 URL로 고객이 실제 접속하여 데모 확인 가능
- [ ] 대화형 수정으로 최소 3회 이상 반복 수정 가능

### 5.3 실패/중단 조건
- 3개월 내 팀원 1명도 OpenClaw로 PoC를 자립 생성하지 못할 경우
- AI 코드 생성 품질이 수작업 대비 50% 이상 후처리가 필요할 경우
- Cloudflare 배포 파이프라인이 사내 보안 정책을 충족하지 못할 경우
