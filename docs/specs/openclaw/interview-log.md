# OpenClaw 요구사항 인터뷰 로그

## Part 1: 왜 (목적/문제)

**Q: 어떤 문제가 있어서, 또는 어떤 기회를 잡으려고 이걸 만들려고 하시나요?**
A: 사업 기회 포착 — AI/자동화 수요가 증가하는 시장 트렌드

**Q: 시급성이 있나요?**
A: 내부 로드맵 상 예정된 작업. 긴급하지 않으나 계획대로 진행.

**파악된 것:**
- [x] 핵심 기회: AI/자동화 수요 증가에 따른 PoC 빌더 시장
- [x] 현재 상태: PoC 제작에 2~4주 소요 (수작업 기반)
- [x] 목표 상태: 비개발자도 AI로 PoC를 빠르게 생성
- [x] 시급성: 중간 (내부 로드맵)

## Part 2: 누구를 위해 (사용자/이해관계자)

**Q: 누가 사용하게 되나요?**
A: 내부 팀원 (AX BD팀)

**Q: 기술 수준은?**
A: 비개발자 (사업/기획 직군)

**파악된 것:**
- [x] 주 사용자: AX BD팀 사업개발/기획 담당자 (비개발자)
- [x] 사용 환경: 웹 브라우저, 코딩 경험 없음
- [x] 이해관계자: 팀 리더 (의사결정), 고객사 (PoC 수혜자)

## Part 3: 무엇을 (범위/기능)

**Q: 핵심 기능은?**
A: PoC/프로토타입 제작을 손쉽게 — AI 자동 생성 방식

**Q: 레퍼런스는?**
A: Bolt.new / Lovable (프롬프트로 앱 생성하는 AI 빌더)

**Q: 범위는?**
A: MVP 최소 범위만

**파악된 것:**
- [x] 핵심 기능: 요구사항 입력 → AI가 PoC 코드 자동 생성
- [x] 레퍼런스: Bolt.new, Lovable
- [x] 범위: MVP (핵심 기능만, 고급 기능 후속)
- [x] 연동: 독립 실행 (MVP 단계)

## Part 4: 성공 기준

**Q: 성공 기준은?**
A: PoC 생성 시간 단축

**Q: MVP 최소 기준은?**
A: 코드 생성 + 실제 배포까지 가능한 수준

**파악된 것:**
- [x] KPI: PoC 생성 시간 단축 (기존 2~4주 → 목표 수일 이내)
- [x] MVP 기준: 코드 생성 + 배포 가능
- [ ] 실패 조건: 미정의
- [ ] 비기능 요구사항: 미정의
