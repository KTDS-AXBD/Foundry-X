## 1. 요약 (Executive Summary)

**한 줄 정의:**
OpenClaw는 AX BD팀 사업개발/기획 담당자(비개발자)가 **프롬프트 입력만으로 PoC/프로토타입을 자동 생성하고 배포까지 완료**할 수 있는 AI PoC 빌더 플랫폼이다.

**배경:**
AI 코딩 에이전트의 역량이 급격히 발전하면서, Bolt.new/Lovable 같은 "프롬프트→앱" 빌더가 시장에 등장했다. AX BD팀은 고객사에 PoC를 빠르게 보여줘야 하는데, 현재는 개발자가 수작업으로 2~4주 소요된다. OpenClaw는 Foundry-X의 AI 에이전트 기술 기반 위에, 비개발자도 프롬프트만으로 PoC를 생성하고 실제 배포까지 할 수 있는 도구를 만들어 이 병목을 해소한다.
<!-- CHANGED: 시장/경쟁/트렌드 관점 누락 지적 반영, 경쟁 서비스 발전 속도와 차별화, 아키텍처 유연성 언급 추가 -->
- 최근 LLM 모델(Claude, GPT 등)의 발전 속도에 따라, OpenClaw는 특정 모델에 종속되지 않고 다양한 AI 모델을 유연하게 활용할 수 있는 모듈형 아키텍처를 추진한다. 외부 솔루션 대비 "사내 보안 정책 준수", "Foundry-X 기술 스택과의 심화 연동", "도메인 특화 PoC 생성" 등 차별화 요소를 적극 도입할 계획이다.

**목표:**
비개발자가 요구사항을 입력하면 AI가 코드를 생성하고, 원클릭으로 Cloudflare에 배포하여 고객에게 바로 데모 URL을 전달할 수 있는 상태. PoC 제작 시간을 2~4주에서 **수 시간 이내**로 단축한다.
<!-- CHANGED: "수 시간 이내"의 의미 구체화 및 PoC 복잡도별 소요 시간 차이 언급 -->
- PoC의 복잡도(단순 단일 화면 앱, 외부 API 연동, CRUD 등)에 따라 생성 및 배포 소요 시간은 상이할 수 있으나, MVP 기준 최대 8시간 이내 완성(프롬프트 입력~배포) 목표로 한다.

## 2. 문제 정의

### 2.1 현재 상태 (As-Is)
- PoC/프로토타입 제작에 **2~4주 소요** (1인 개발자 수작업)
- 사업개발 담당자는 개발 역량이 없어 직접 PoC를 만들 수 없음
- 고객 요구사항 접수 → 개발자 전달 → 스케줄 조율 → 개발 → 배포 과정에 병목 발생
- Bolt.new, Lovable 같은 외부 서비스는 사내 보안 정책/기술 스택과 맞지 않음
<!-- CHANGED: 외부 서비스와의 불일치 구체적 사례 보완 -->
- 외부 서비스는 Cloudflare 기반 배포 미지원, 사내 인증/접근권한 정책 부적합, 고객 데이터 국외 전송 우려 등 실제 업무환경에서 활용이 제한됨

### 2.2 목표 상태 (To-Be)
- 비개발자가 **웹 UI에서 프롬프트 입력** → AI가 코드 생성 → **원클릭 배포**
- PoC 제작 시간: **수 시간 이내** (기존 대비 85%+ 단축)
- 고객에게 즉시 데모 URL 전달 가능
- Foundry-X 기술 스택(Cloudflare) 위에서 운영되어 사내 보안 정책 준수
<!-- CHANGED: 목표 상태 달성의 현실적 한계(복잡도에 따른 소요 시간 변동 등) 명시 -->
- 복잡도 높은 PoC(외부 API 연동, 화면 3개 이상 등)는 추가 시간이 소요될 수 있으나, 최대 8시간 이내 완료 목표

### 2.3 시급성
- 내부 로드맵 상 예정된 작업 (높은 시급성은 아님)
- 다만 AI 빌더 시장이 빠르게 성장 중이므로, 사내 역량 확보 차원에서 조기 착수가 유리
- Foundry-X Phase 5(고객 파일럿) 이후 자연스러운 확장 방향

<!-- CHANGED: AI 코드 생성 신뢰성/현실성 부족 및 기술적 검증 필요 지적 반영 -->
### 2.4 현실성 검토 및 기술 신뢰성
- Foundry-X AI 에이전트가 실제로 PRD에서 요구하는 수준(풀스택 앱 생성, 배포 자동화)을 충족할 수 있는지에 대한 PoC 파일럿 및 기술 검증이 선행되어야 함
- AI 코드 생성 품질, 배포 파이프라인의 안정성, 에러 복구 가능성 등이 MVP 전 단계에서 반드시 검증되어야 함
