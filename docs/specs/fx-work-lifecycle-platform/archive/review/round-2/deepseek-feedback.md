- **성능 한계**: 매번 raw 파일 fetch/파싱 → 트레이서빌리티 쿼리 시 실시간 성능 저하 예상

**B. AI 분류 파이프라인의 신뢰성 리스크**
- Claude Sonnet의 분류 정확도 75% 목표는 낙관적. 실제 자연어 Backlog 입력 시 맥락 오해로 잘못된 Track/Priority 배정 가능
- **제안**: 
  1. 3단계 fallback 체계: AI → regex 규칙 → 수동 검토 큐
  2. 분류 신뢰도 점수 표시 및 저신뢰 항목 자동 수동 검토 라우팅

**C. 트레이서빌리티 데이터 품질 보장의 구조적 취약성**
- PR body의 F번호 입력은 전적으로 개발자 의존적. 누락 시 전체 체인 단절
- **제안**: 
  1. PR 생성 시 CI 검증(git hook)으로 F번호 필수화
  2. branch명 자동 파싱으로 fallback 제공(sprint-xxx-fyyy 패턴)
