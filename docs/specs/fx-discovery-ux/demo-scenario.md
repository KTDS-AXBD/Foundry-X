## 데모 목적

> "Foundry-X에서 사업 아이템 발굴 프로세스를 직관적으로 시작하고, AI 비서 안내를 받으며 단계별로 진행할 수 있다"는 것을 체험시킨다.

**핵심 메시지:**
- CC Cowork은 "채팅", Foundry-X는 **"프로세스"** — 단계별 진행 추적 + 산출물 관리 + 팀 공유
- 어디서 시작해야 할지 모르겠으면 → **Help Agent에게 물어보세요**
- AI가 만든 결과물은 바로 검토·수정·승인 가능 → **HITL 패널**

## 데모 흐름 (4단계, 20분)

### Act 1: 온보딩 투어 (3분) — F265

**시나리오:** 처음 발굴 페이지에 방문하는 팀원의 시점

1. 사이드바에서 **"2. 발굴 → 🧭 발굴 위저드"** 클릭
2. **인터랙티브 투어 자동 시작** (5스텝)
   - Step 1: "아이템을 선택하세요" → BizItem 드롭다운 하이라이트
   - Step 2: "현재 단계를 확인하세요" → 좌측 스텝퍼 하이라이트
   - Step 3: "스킬을 실행해보세요" → 추천 스킬 영역 하이라이트
   - Step 4: "결과를 확인하세요" → 콘텐츠 패널 하이라이트
   - Step 5: "다음 단계로 이동하세요" → Next 버튼 하이라이트
3. 투어 완료 후 → "이제 직접 해볼까요?"

**포인트:**
- "처음 오신 분은 이 투어가 자동으로 시작돼요"
- "다시 보고 싶으면 '투어 다시 보기' 버튼을 누르면 돼요"

---

### Act 2: 발굴 위저드 탐색 (5분) — F263

**시나리오:** 사전 등록한 biz-item으로 프로세스 단계 탐색

1. **BizItem 드롭다운**에서 "AI 기반 제조 품질 예측" 선택
2. 좌측 스텝퍼에서 **2-0 (분류)** 단계 확인 — "현재 여기까지 왔어요"
3. 단계 클릭하며 각 단계별 콘텐츠 확인:
   - **목적**: "이 단계에서 뭘 해야 하는지"
   - **추천 스킬**: "어떤 도구를 쓸 수 있는지"
   - **예상 산출물**: "뭐가 나오는지"
   - **체크포인트**: "다음 단계로 넘어가려면 뭘 확인해야 하는지"
4. **"시작하기"** 버튼 클릭 → 단계 상태 IN_PROGRESS로 전환

**포인트:**
- "예전엔 메뉴 10개가 나열되어 있어서 뭘 눌러야 할지 몰랐잖아요"
- "이제 왼쪽 스텝퍼만 보면 내가 어디까지 했는지, 다음엔 뭘 해야 하는지 바로 보여요"
- "각 단계에 맞는 스킬이 자동으로 추천돼요"

---

### Act 3: Help Agent 질의 (7분) — F264

**시나리오:** 프로세스 진행 중 궁금한 것을 AI 비서에게 질문

1. 우측 하단 **💬 버튼** 클릭 → Help Agent 챗 패널 열림
2. 질문 시나리오 (순서대로):

| # | 질문 | 예상 응답 유형 | 설명 |
|---|------|:---:|------|
| 1 | "다음 뭐 해야 돼?" | 로컬 즉시 | → 현재 단계 기반 다음 액션 안내 |
| 2 | "이 단계에서 쓸 수 있는 스킬은?" | 로컬 즉시 | → 추천 스킬 목록 (bd-skills.ts) |
| 3 | "BMC가 뭐야?" | LLM 스트리밍 | → SSE로 타이핑 효과 + 설명 |
| 4 | "시장 규모 분석은 어떻게 해?" | LLM 스트리밍 | → 구체적 방법론 + 관련 스킬 추천 |
| 5 | "체크포인트 질문 보여줘" | 로컬 즉시 | → 현재 단계 체크포인트 |

3. **로컬 응답 vs LLM 응답 차이** 강조:
   - 1, 2, 5번: 즉시 응답 (100ms 이내)
   - 3, 4번: SSE 스트리밍으로 글자가 하나씩 나타남

**포인트:**
- "CC Cowork과 비슷하게 물어볼 수 있지만, **내가 지금 보고 있는 아이템과 단계를 자동으로 인식**해요"
- "단순한 질문은 API 비용 없이 즉시 답변, 복잡한 질문만 AI가 답변해요"
- "대화 이력이 저장되니까 나중에 다시 볼 수 있어요"

---

### Act 4: HITL 결과물 검토 (5분) — F266

**시나리오:** 스킬 실행 결과를 검토하고 승인

1. 위저드에서 **추천 스킬** 중 하나 실행 (예: "시장 스캔")
2. 실행 완료 → **사이드 드로어 자동 오픈** (HITL 패널)
3. 산출물 내용 확인 (마크다운 렌더링)
4. **4가지 액션 시연:**
   - ✅ **승인** → "이 결과가 좋으면 승인하면 다음 단계로 자동 연결돼요"
   - ✏️ **수정** → 에디터 전환, 내용 수정 후 저장
   - 🔄 **재생성** → "결과가 마음에 안 들면 AI에게 다시 만들어달라고 할 수 있어요"
   - ❌ **거부** → 사유 입력 → "왜 안 되는지 기록해두면 다음에 참고할 수 있어요"

**포인트:**
- "CC Cowork에서는 결과를 복사해서 따로 저장해야 했잖아요"
- "Foundry-X에서는 **결과가 자동 저장**되고, 승인하면 다음 단계 입력으로 연결돼요"
- "누가 언제 승인/거부했는지 **이력이 남아요** — 팀 공유 시 투명해요"

## 마무리 (3분)

### CC Cowork vs Foundry-X 비교 요약

| 항목 | CC Cowork | Foundry-X (신규) |
|------|-----------|------------------|
| 시작점 | 빈 채팅창 | 위저드 스텝퍼 (현재 단계 표시) |
| 다음 할 일 | 직접 기억 | 스텝퍼 + Help Agent 안내 |
| 산출물 관리 | 채팅 히스토리 검색 | 단계별 산출물 자동 저장 |
| 팀 공유 | 채팅 링크 공유 | 히스토리 + HITL 이력 |
| 프로세스 추적 | 없음 | 진행률 + 사업성 신호등 |

### Q&A + 피드백 수집

1. "어떤 점이 CC Cowork보다 나았나요?"
2. "어떤 점이 불편하거나 부족했나요?"
3. "가장 자주 쓸 것 같은 기능은?"
4. "Help Agent에게 물어보고 싶은 질문이 더 있나요?"

> 피드백은 NPS 서베이(F254)로 후속 수집, 즉석 피드백은 메모.

## 비상 대응

| 상황 | 대응 |
|------|------|
| Help Agent 응답 없음 | "OpenRouter 서버 지연입니다" → 로컬 응답 질문으로 전환 |
| 페이지 로딩 안 됨 | Workers cold start → 10초 대기 후 새로고침 |
| 스킬 실행 실패 | "Anthropic API 일시 지연" → 사전 실행한 결과물로 HITL 시연 |
| HITL 패널 안 열림 | 산출물 목록에서 직접 클릭으로 우회 |
