## Part 1: 왜 (목적/문제)

**Q: 지금 어떤 문제가 있어서 이걸 만들려고 하시나요?**

- 실제 사용자(AX BD팀) 입장에서 Foundry-X를 사용하려면, CC Cowork나 ChatGPT 대비 같거나 나은 사용자 경험이 필요
- 현재 발굴 단계 메뉴는 나열형이고 메뉴 중복이 있으며 어떻게 사용할지 잘 모르겠음
- 실질적으로 어떤 사업아이템을 발굴 프로세스에서 단계별로 진행하면서 HITL가 작동하고 인터랙션과 결과물이 확인되어야 함

**Q: 현재 어떤 도구를 사용하고 있나요?**

- CC Cowork 주력으로 사용 중
- Foundry-X 대신 CC Cowork을 선택하는 이유: 대화형 인터페이스가 직관적

**Q: Foundry-X를 안 쓰게 되는 가장 큰 장벽은?**

- **진입 장벽** — 어디서 시작해야 할지 모르겠고, 메뉴가 너무 많고 복잡해서 손이 안 감

## Part 2: 누구를 위해 (사용자/이해관계자)

**Q: 주 사용자는 누구인가요?**

- AX BD팀 7명 (+ 공용 계정 1개)

**Q: 기술 수준 분포는?**

- **소수 숙련(2~3명)** + **대다수 초보(4~5명)**
- AI 도구 익숙도와 사업개발 경험에서 차이
- UX 설계의 기준점은 초보 사용자여야 함

---

## Part 3: 무엇을 (범위/기능)

**Q: 핵심 기능과 우선순위는?**

Must Have (1차 범위):
1. 발굴 프로세스 단계별 안내 UI — 2-0~2-10 나열형 → 단계별 흐름 재구성 + 다음 할 일 안내
2. Help Agent (개인 비서) — 사용자 컨텍스트 인식 + 단계별 추천 + 질문 응답 챗봇
3. 온보딩 투어 개선 — 처음 사용자가 5분 안에 발굴 프로세스 시작 가능한 가이드
4. HITL 인터랙션 + 결과물 확인 — 각 단계에서 AI 산출물을 사용자가 검토/수정/승인하는 흐름

**Q: Out-of-scope는?**

- 외부 연동(Discovery-X, 외부 API)은 2차로 미룸
- 프로세스 커스터마이징(Depth 조절, 개인별 패턴 학습)은 가능하면 포함, 아니면 2차

## Part 4: 어떻게 판단할 것인가 (성공 기준)

**Q: 성공 기준은?**

- **팀원 실사용** — 7명 중 과반수 이상이 실제 발굴 작업에 Foundry-X 사용
- **MVP 기준**: 4가지 Must Have 기능이 모두 동작하고 팀 데모 완료

---

## Part 5: 제약과 리소스 (현실 조건)

**Q: 시간/인력 제약은?**

- **2주 내 MVP** (Sprint 4~6회 분량)
- 개발: Sinclair + Claude Code
- 기존 인프라 활용: Hono Workers, D1, SSE, Vite+React Router

**Q: Help Agent LLM API는?**

- **OpenRouter 경유** (멀티모델 프록시) — SSE 스트리밍 응답
- 기존 Anthropic API 직접 호출이 아닌 OpenRouter를 통해 모델 유연성 확보
