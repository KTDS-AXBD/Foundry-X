## 기술 검토 의견

### 1. 기술적 실현 가능성

**주요 우려사항:**

1. **Claude CLI `--bare` 모드의 운영 리스크**
   - 문서에서 언급된 rate limit 확인이 미흡합니다. 구독 기반 CLI도 사용량 제한이 있을 가능성이 높으며, batch 처리 시 throttling 문제 발생 가능
   - `subprocess` 실행 환경에서의 안정성: 장시간 실행 시 메모리 누수, 프로세스 고아화, timeout 관리 미흡
   - WSL/tmux 의존성은 Railway Docker 환경과의 호환성 문제 야기 가능

2. **5차원 평가의 재현성과 객관성**
   - Vision API 기반 UI 평가는 주관적 요소가 크며, 동일 코드에 대해 일관된 점수 보장 어려움
   - LLM 기반 PRD 반영도 평가는 프롬프트 엔지니어링에 민감하며, 평가 비용 누적 가능성
   - 각 차원별 가중치(20%, 25% 등)가 임의적이며, 실제 고객 데모 품질과의 상관관계 검증 필요

3. **자동 개선 루프의 수렴 보장 불가**
   - "가장 낮은 차원 타겟 피드백" 접근법은 로컬 최적화(local optimization) 문제 야기
   - 5라운드 제한 내 80점 달성은 과도한 낙관적 가정
   - 개선 프롬프트의 효과 측정 메커니즘이 부재

### 2. 아키텍처 적합성

**개선 필요 사항:**

1. **듀얼 모드 아키텍처의 복잡성**
   - CLI/API 전환 로직이 운영 복잡도 증가시킴
   - **제안**: CLI 모드를 primary로 고정하고, API 모드는 완전 별도 파이프라인으로 분리
   - 상태 관리와 에러 핸들링이 모드별로 분기되어 유지보수 어려움

2. **평가 파이프라인의 단일 포인트 실패**
   - 5개 평가 차원이 직렬 실행되어 하나의 실패가 전체 평가 실패로 이어짐
   - **제안**: 평가 차원별 독립적 실행과 결과 병합으로 내결함성 향상

3. **데이터 흐름의 투명성 부족**
   - 라운드별 개선 결정 과정이 블랙박스화되어 디버깅 어려움
   - **제안**: 결정 로그(deterministic logging)와 추적 가능한 피드백 체인 구현

### 3. 구현 복잡도

**과소평가된 요소:**

1. **Vision API 평가의 실제 비용**
   - 스크린샷당 $0.0025~$0.01 (해상도에 따라) × 5차원 평가 × 여러 라운드 = 월 $5 예산 초과 가능성 높음
   - 스크린샷 캡처를 위한 headless 브라우저 인프라 필요

2. **동시성 관리**
   - 여러 PRD의 병렬 처리 시 CLI 모드의 rate limit 충돌 가능
   - Railway Hobby plan의 동시 컨테이너 제한(최대 1개)으로 확장성 제약

3. **상태 복구 메커니즘**
   - 개선 루프 중간 실패 시 재개(resume) 로직 미정의
   - 부분 완료된 평가 결과의 일관성 보장 어려움

### 4. 기술 리스크

**상위 3개 리스크:**

1. **Claude CLI 운영 리스크 (High)**
   - 구독 정책 변경, CLI 인터페이스 업데이트, rate limit 강화 등 외부 의존성
   - 장기 실행 프로세스 관리의 운영 부담

2. **평가 체계의 신뢰성 리스크 (High)**
   - 주관적 평가 기준으로 인한 "점수 인플레이션" 가능성
   - 고객 실제 만족도와의 괴리 발생 시 시스템 신뢰도 붕괴

3. **비용 전가 리스크 (Medium)**
   - API 비용은 제거되었지만, 평가를 위한 Vision API 비용이 새로운 비용 항목으로 부상
   - 인프라 운영 비용(Railway, Cloudflare)이 숨겨진 비용으로 작용

### 5. 착수 판단

**착수 판단: Conditional**

**전제조건:**

1. **Claude CLI `--bare` 모드의 실제 제약 확인**

   - 장시간 실행 안정성 테스트 결과 확보
   - Railway Docker와의 호환성 검증 완료

   - 평가 점수와 실제 인간 평가의 상관관계 검증(0.7 이상)
   - Vision API 비용 시뮬레이션으로 월 $5 예산 내 가능성 확인

2. **Phase 2: CLI 운영 안정화**

   - 인간 검토(human-in-the-loop) 단계를 초기에는 유지
