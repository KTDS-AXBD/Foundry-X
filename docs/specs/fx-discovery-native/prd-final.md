## 1. 요약 (Executive Summary)

**한 줄 정의:**  
Foundry-X를 발굴(2단계)·형상화(3단계) 전문 도구로 재구축하여, 사업 아이템 등록부터 산출물 생성까지 끊김 없는 단계별 흐름을 제공한다.

**배경:**  
MSA 재조정으로 Foundry-X는 AX BD 프로세스 중 2.발굴 + 3.형상화에 집중하는 서비스로 재정의됨. 그러나 현재 사이트는:
- 6단계 프로세스(수집~GTM) 메뉴가 모두 존재하여 서비스 정체성이 불명확
- Discovery 페이지가 외부 서비스(dx.minu.best)를 iframe으로 임베딩하여 내부 데이터와 단절
- 형상화 산출물(사업기획서, Offering, PRD, Prototype) 간 연결 고리가 없어 각각 독립된 목록 페이지로 존재
- 사용자가 사이트 접속 시 시작점을 찾지 못하고, 다음 액션을 알 수 없음

**목표:**  
사용자가 사업 아이템을 등록하면, 발굴 분석(11단계)을 단계별로 진행하고, 분석 완료 후 형상화 산출물(기획서→Offering→PRD→Prototype)을 파이프라인으로 생성할 수 있는 **일관된 흐름**을 구현한다.

## 2. 문제 정의

### 2.1 현재 상태 (As-Is)

| 문제 | 상세 | 심각도 |
|------|------|--------|
| **시작점 부재** | 사이트 접속 시 어디서 무엇을 해야 할지 막막. 대시보드는 운영 지표, Getting Started는 형식적 | Critical |
| **Discovery iframe 단절** | `discovery.tsx`가 `dx.minu.best` iframe. Foundry-X 내부 데이터(Offering, PRD 등)와 연결 불가 | Critical |
| **목록만 나열** | 발굴 페이지가 아이템 목록만 표시. 실제 분석 수행 불가 | Critical |
| **형상화 연결 부재** | 사업기획서/Offering/PRD/Prototype이 각각 독립 목록. 아이템과 연결 없음 | Critical |
| **데이터 불일치** | 단계/개수가 맞지 않는 데이터가 페이지 곳곳에 존재 | High |
| **버그** | Offering 카드 클릭 불가, 피치덱 에러 | High |
| **평가결과서 무의미** | 뭘 평가하는지 불명확, 연결 자료 없음 | Medium |
| **메뉴 과잉** | 6단계 전체 메뉴 존재. 1.수집/4.검증/5.제품화/6.GTM은 Foundry-X 범위 밖 | Medium |

### 2.2 목표 상태 (To-Be)

```
[위저드 온보딩] → [아이템 등록] → [발굴 11단계 분석] → [형상화 파이프라인]
                                       ↓                    ↓
                                  AI 자동 수행         기획서 → Offering → PRD → Prototype
                                  사용자 검토/보완      (이전 단계 완료 시 다음 잠금 해제)
```

- 사이드바는 **2.발굴 + 3.형상화만** 표시
- 모든 데이터가 **사업 아이템 하나를 중심**으로 연결
- 각 페이지에서 **현재 단계 + 다음 액션**이 명확히 표시

### 2.3 시급성

- 현재 사이트가 **사실상 사용 불가** 상태 — 방문자가 가치를 체험할 수 없음
- MSA 재조정이 완료되어 서비스 정체성이 확정된 시점
- 팀 내부 데모 및 고객 프레젠테이션에서 즉각 사용 필요

## 4. 기능 범위

### 4.1 핵심 기능 (Must Have) — MVP

| # | 기능 | 설명 | 우선순위 |
|---|------|------|----------|
| 1 | **사이드바 정리** | 2.발굴 + 3.형상화만 남기고 나머지 제거. Admin 메뉴는 유지 | P0 |
| 2 | **위저드형 온보딩** | 시작하기 페이지에서 사업 아이템 등록 위저드. 프롬프트 입력 또는 자료 업로드 | P0 |
| 3 | **아이템 등록** | 사업 아이템 CRUD. 제목, 설명, 배경, 키워드, 첨부 자료 | P0 |
| 4 | **발굴 분석 대시보드** | 아이템별 11단계 분석 진행 상태를 단계별 카드/타임라인으로 표시 | P0 |
| 5 | **발굴 분석 실행** | 각 분석 단계(2-0~2-10)를 AI가 자동 수행. 사용자는 결과 검토 + 보완 | P0 |
| 6 | **아이템 상세 페이지** | 아이템 기본 정보 + 발굴 분석 결과 + 형상화 산출물을 하나의 페이지에서 연결 | P0 |
| 7 | **사업기획서 생성** | 발굴 분석 완료 후, 분석 결과를 기반으로 사업기획서 자동 생성 | P0 |

| # | 기능 | 설명 | 우선순위 |
|---|------|------|----------|

| 2 | **PRD 생성** | Offering 완료 후 PRD 자동 생성 | P1 |

| 5 | **대시보드 허브** | 전체 아이템 현황, 단계별 통계, 최근 활동 | P1 |

## 5. 성공 기준

### 5.1 정량 지표 (KPI)

| 지표 | 현재값 | 목표값 | 측정 방법 |
|------|--------|--------|-----------|
| 아이템 등록→기획서 완료율 | 0% (불가) | > 50% | DB 집계 |
| 사용자 첫 아이템 등록까지 시간 | N/A (시작점 없음) | < 3분 | 세션 로그 |
| 페이지 간 이탈률 | 높음 (추정) | < 30% | Analytics |

### 5.2 MVP 최소 기준

- [ ] 위저드로 사업 아이템 등록 가능
- [ ] 등록된 아이템에 대해 11단계 발굴 분석 중 최소 3단계 실행 가능
- [ ] 발굴 완료 아이템에서 사업기획서 생성 가능
- [ ] 사이드바에 2.발굴 + 3.형상화만 표시

### 5.3 실패/중단 조건

- AI 분석 API 비용이 아이템당 $10 이상일 경우 비용 모델 재검토
- 11단계 분석 중 5단계 이상이 의미 없는 결과를 생성할 경우 분석 체계 재설계

## 6. 제약 조건

### 6.1 일정

- MVP 목표: Phase 24 (1~2주 내 Sprint 배치)
- 후속(P1/P2): Phase 25+

### 6.2 기술 스택

- 프론트엔드: React 18 + React Router 7 + Vite 8 (기존 packages/web)
- 백엔드: Hono + Cloudflare Workers + D1 (기존 packages/api)
- AI: Anthropic Claude API + OpenRouter (기존 인프라)
- 기존 시스템 의존: Foundry-X 모노리포 내에서 작업

### 6.3 인력/예산

- 투입 가능 인원: 1명 (AI 에이전트 + 개발자)
- AI API 비용: 기존 예산 내

### 6.4 컴플라이언스

- KT DS 내부 정책: SSO 인증 유지
- 보안 요구사항: 기존 JWT + RBAC 유지

## 부록 A: 현재 사이드바 vs 목표 사이드바

### As-Is (6단계)
```
대시보드
시작하기
─ 1. 수집 [이관 예정]
  ├ Field 수집
  ├ IDEA Portal
  └ 스크리닝
─ 2. 발굴
  ├ 발굴 (iframe → dx.minu.best)
  └ 평가 결과서
─ 3. 형상화
  ├ 사업기획서
  ├ Offerings
  ├ Offering Pack
  ├ PRD
  └ Prototype
─ 4. 검증
  ├ 검증
  └ 산출물 공유
─ 5. 제품화
  ├ 제품화
  └ Offering Pack
─ 6. GTM [이관 예정]
  ├ 대고객 선제안
  └ 파이프라인
위키
설정
```

### To-Be (2단계 집중)
```
대시보드 (내 아이템 현황)
─ 아이템 등록 (위저드)
─ 2. 발굴
  ├ 내 아이템
  └ [아이템별 분석 상세] (동적 라우트)
─ 3. 형상화
  ├ 사업기획서
  ├ Offering
  ├ PRD
  └ Prototype
  (각 페이지는 아이템 선택 → 해당 산출물 표시)
위키
설정
```

## 부록 B: 핵심 사용자 플로우

```
1. 로그인
2. 대시보드: 내 아이템 현황 카드 → "새 아이템 등록" CTA
3. 위저드: 프롬프트 입력 or 자료 업로드 → 아이템 생성
4. 아이템 상세: 기본 정보 + 발굴 진행 상태 (11단계 스텝퍼)
5. 분석 실행: "분석 시작" → AI 자동 수행 → 단계별 결과 표시
6. 분석 검토: 각 단계 결과를 펼쳐 확인 + 보완 입력
7. 발굴 완료: 종합 평가 요약 → "형상화 진행" CTA
8. 사업기획서: 발굴 결과 기반 자동 생성 → 편집 가능
9. (후속) Offering → PRD → Prototype 순차 진행
```
