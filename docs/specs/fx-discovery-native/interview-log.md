## 배경

MSA 구조 변경으로 Foundry-X는 2.발굴 + 3.형상화 단계에 집중하는 서비스로 재정의됨.
사용자 관점에서 현재 사이트의 심각한 UX 문제를 발견하고, 전면 재검토를 요청.

---

## 사용자 피드백 (원문)

### 전반적 문제
- 사이트에 접속해서 어디서 무엇을 해야할지 막막함
- 전체적으로 사이트에 올라와 있는 데이터가 단계도 안맞고 개수도 안맞음

### 발굴 (2단계)
- 발굴 페이지에 와서 내가 원하는 건 사업아이템을 프롬프트든 자료든 업로드해서 한단계, 한단계 진행하는 거였는데, 목록만 나오고 실제 분석을 할 수가 없음
- 평가결과서가 있는데 뭘 평가하겠다는 건지 모르겠고, 자료도 없음

### 형상화 (3단계)
- 사업기획서: 아이디어 목록이 떡하니 나타나서 왜 이것부터 시작하는지 모르겠음
- 아이디어 목록을 클릭하면 다시 2단계 사업 아이디어 페이지로 나옴
- 사업 아이디어 상세로 가면 너무 간략한 정보만 나오고 다음 액션과 연결이 안됨. 길을 잃음
- Offering: 목록은 나오는데 카드를 클릭이 안됨. 피치덱을 누르면 에러 발생
- PRD: 요구사항과 컨텍스트가 나오는데 아무런 사업아이템과 연결지점 없이 생뚱맞음. 여기서부터 PRD를 작성할 수 없음. 연결고리 없음
- Prototype: 페이지에 사업 아이템에서 생성하라고 나오는데, 뭘 어떻게? 연결 정보 전혀 없음

### 요청
- 전체적으로 철저하게 사용자 관점에서 하나하나 입력하고 연결해간다는 흐름으로 전체 메뉴 구성과 페이지를 재검토

## 인터뷰 Q&A

### Q1. 메뉴 범위 — 사이드바의 1.수집/4.검증/5.제품화/6.GTM 메뉴 처리?
**A: 2.발굴 + 3.형상화만 남기기**
- 나머지 단계는 완전히 제거
- Foundry-X = 발굴·형상화 전문 도구로 명확히 포지셔닝

### Q2. 핵심 흐름 — 사용자가 처음 와서 시작하는 모습?
**A: 위저드형 온보딩**
- 시작하기 페이지에서 위저드로 아이템 등록
- 자동으로 발굴 프로세스 시작
- 완료 시 형상화로 안내

### Q3. 발굴 분석 — 아이템 등록 후 어떤 분석을 제공?
**A: 기존 11단계 분석 유지**
- AX BD 2단계 발굴 프로세스(2-0~2-10)의 11개 분석을 순차적으로 보여주기
- AI가 자동 수행하고 사용자는 검토/보완

### Q4. 형상화 흐름 — 산출물 간 관계 표현?
**A: 파이프라인형 (순차)**
- 발굴 완료 → 사업기획서 → Offering → PRD → Prototype 순서대로 잠금 해제
- 이전 단계 미완료 시 다음 단계 비활성

### Q5. Discovery 구현 — iframe vs 네이티브?
**A: Foundry-X 네이티브로 전환**
- iframe 제거하고 Foundry-X 내부에서 직접 렌더링
- 데이터는 API로 가져오고, 형상화 산출물과 직접 연결 가능

### Q6. 데이터 정합성 — 기존 데이터 처리?
**A: 새로 시작 (Clean Slate)**
- 기존 데이터는 아카이브
- 새 구조로 처음부터 데이터 입력
- 깨끗하지만 기존 작업물 손실 감수

### Q7. 실행 전략 — 우선순위?
**A: 핵심 흐름 우선 (MVP)**
- 아이템 등록 → 발굴 분석 → 사업기획서 생성까지의 핵심 플로우만 먼저 완성
- 나머지(Offering/PRD/Prototype)는 후속
