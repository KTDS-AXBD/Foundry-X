## 기술 검토 의견

### 1. 기술적 실현 가능성

**긍정적 요소:**
- 기존 모노리포와 기술 스택(React 18, Hono, Cloudflare Workers, D1) 활용으로 기술 부채 최소화
- AI API 연동은 기존 인프라 활용 가능

**주요 우려사항:**

**1.1 AI 파이프라인 처리 시간 및 비용**
- 11단계 분석을 순차/병렬 처리할 경우 사용자 대기 시간 문제
- 각 분석 단계별 API 호출 비용 누적 → P0 기능만으로도 아이템당 $3-5 예상
- 장시간 실행 작업(>30초)에 대한 Worker 타임아웃(10분) 고려 필요

**1.2 데이터 일관성 관리**
- 발굴 분석 11단계 간 의존성 관리 복잡
- 형상화 파이프라인(기획서→Offering→PRD→Prototype)의 상태 동기화
- 사용자 수정 시 하위 단계 재생성 트리거 로직 미정의

**1.3 파일 업로드 처리**
- "자료 업로드" 기능 명시되었으나 스토리지 설계 누락
- 대용량 파일(PPTX, PDF) 처리와 텍스트 추출 파이프라인 필요

### 2. 아키텍처 적합성

**2.1 상태 관리 복잡도 과소평가**
- 단일 아이템에 대한 다단계 상태(발굴 11단계 + 형상화 4단계) 관리
- 실시간 진행 상태 표시를 위한 WebSocket/SSE 고려 필요
- 오프라인 편집 가능성과 동기화 전략 부재

**2.2 데이터베이스 스키마 설계 리스크**
- 기존 `biz_items` 테이블 재사용 시 확장성 제한
- 분석 결과(구조화/비구조화 데이터) 저장 방식 결정 필요
- 버전 관리(아이템 수정 이력, 산출물 버전) 요구사항 누락

**2.3 서비스 경계 모호**
- "Clean Slate" 접근이지만 기존 Offering/PRD/Prototype 코드 재사용 계획
- 모노리포 내 경계 불명확으로 점진적 리팩토링 어려움

### 3. 구현 복잡도

**3.1 과소평가된 복잡도 요소:**

**동시성 제어:**
- 동일 아이템에 대한 다중 사용자 편집 충돌
- AI 분석 실행 중 사용자 수정 시 충돌 처리

**에러 복구:**
- AI API 실패 시 재시도/폴백 전략
- 장시간 실행 작업의 중단/재개 메커니즘

**데이터 변환 파이프라인:**
- 발굴 분석 결과 → 기획서 → Offering → PRD → Prototype 변환 로직
- 각 단계별 출력 형식(HTML, PPTX, 마크다운 등) 변환 복잡도

**3.2 성능 고려사항:**
- 대시보드에서 다수 아이템의 진행 상태 집계 쿼리
- 분석 결과의 실시간 미리보기 렌더링 성능

### 4. 기술 리스크

**4.1 높음(High) 리스크:**

1. **AI 분석 품질 불확실성**
   - 11단계 분석이 의미 있는 결과 생성 보장 불가
   - 도메인 특화 프롬프트 튜닝에 장기간 소요 가능

2. **장시간 실행 작업 처리**
   - Cloudflare Workers의 10분 타임아웃 제한
   - 분석 작업 큐 관리 및 상태 추적 시스템 필요

3. **데이터 일관성 유지**
   - 다단계 파이프라인에서 부분 실패 시 롤백 복잡
   - 사용자 수정과 AI 재생성 간 충돌 관리

**4.2 중간(Medium) 리스크:**

1. **기존 코드베이스 의존성**
   - Offering/PRD/Prototype 모듈 재사용 시 기술 부채 상속
   - 점진적 마이그레이션 전략 부재

2. **확장성 제한**
   - 단일 D1 데이터베이스로 다중 팀/조직 확장 시 성능 저하
   - 분석 작업 부하 분산 메커니즘 미구현

### 5. 착수 판단

**착수 판단: Conditional**

**조건:**
1. **증분적 구현 로드맵 수립 필요**
   - P0 중에서도 우선순위 재조정(아이템 등록 → 3단계 분석 → 기획서 생성)
   - 11단계 분석을 한 번에 구현하지 않고 핵심 3-4단계로 시작

2. **아키텍처 결정사항 해결:**
   - 장시간 작업 처리 아키텍처(Queue + Worker) 설계
   - 데이터 스키마 버전 관리 전략 수립
   - 파일 업로드 및 처리 파이프라인 정의

3. **AI 품질 검증 프로토타입:**
   - 핵심 분석 단계 3개에 대한 프롬프트/결과 품질 사전 검증
   - 비용 모델 시뮬레이션(아이템당 예상 비용 산정)

4. **롤백 전략 마련:**
   - "Clean Slate" 실패 시 기존 데이터 복구 계획
   - 단계별 기능 활성화/비활성화 토글 메커니즘
