# IRIS PRD — Internal Risk Identification System

## 1. 개요 (Overview)

**IRIS(Internal Risk Identification System)**는 기업이 이미 운영 중인 DLP, CCTV, 출입통제, HR 데이터를 AI로 연결하여, 기존 보안 체계가 탐지하지 못했던 내부자 유출 패턴을 식별하는 보안 강화 플랫폼이에요.

- **사업 주체**: KT DS AX사업개발본부 BD팀
- **핵심 철학**: 새로운 보안 장비를 도입하는 것이 아니라, 기존에 이미 운영 중인 보안 데이터를 **연결**하여 보이지 않았던 패턴을 보여주는 것
- **현재 상태**: 시뮬레이션 데이터 기반 프로토타입 4종 완료, 2단계 발굴 프로세스 2회(v1+v2) 완료, 오픈소스 기반 아키텍처 설계 완료
- **프로토타입 데모**: dmp-portal.vercel.app

### 핵심 수치

| 항목 | 수치 |
|------|------|
| TAM (한국 시장) | 2,000억 원 |
| Phase 0 투자 | 5,700만 원 |
| 보수적 5년차 매출 | 118억 원 |
| 5년차 영업이익률 | 42% |

## 2. 문제 정의 (Problem Statement)

### 2.1 내부자 기술유출은 20년간 반복되고 있다

2003년부터 2025년까지 주요 사고만 12건 이상이 확인되며, 피해 기업 대부분이 국내 최고 수준의 보안 체계를 갖추고 있었어요.

| 사건 | 시기 | 유출 수법 | 피해 |
|------|------|-----------|------|
| 삼성전자 → CXMT | 2016~2020 | 손으로 필기하여 반출 + 핵심 인력 4명 이직 | 수십조 원 (D램 공정) |
| 쿠팡 고객 정보 | 2025.6~11 | 퇴직자 잔존 권한으로 5개월간 접근 | 3,370만 고객 정보 |
| 카드 3사 | 2012~2014 | 협력업체 직원 USB 복사 반출 | 1억 580만 건 (역대 최대) |
| SK하이닉스 HKMG | 2018~2020 | 협력업체 부사장 접근 권한 악용 | 수천억 원 |
| Google AI칩 설계 | 2022~2023 | 개인 클라우드 업로드 + 이중 근무 | 국가안보 위협 |
| Intel 기밀 반출 | 2024.7 | 퇴직 통보 후 개인 NAS로 18,000건 복사 | 소송 진행 중 |

### 2.2 기존 보안 체계의 구조적 한계

기존 보안이 나빠서가 아니라, **설계 목적이 "내부자 유출 패턴 탐지"가 아니었기 때문**이에요.

- **DLP**: 차단을 잘 함 (디지털 경로 2~3개 커버)
- **CCTV**: 녹화를 잘 함 (사후 포렌식 용도)
- **SIEM**: 수집을 잘 함 (외부 위협 중심)
- **문제**: 각 시스템이 **서로 연결되어 있지 않음** → 교차 패턴 탐지 불가

### 2.3 자가 점검 질문 (고객 Pain Point)

1. 직원이 기밀을 종이에 적어가면, 지금 알 수 있는가?
2. 퇴직 예정자의 파일 다운로드가 갑자기 8배 늘었다면, 보이는가?
3. 퇴직한 직원이 아직 시스템에 접속하고 있다면, 알림이 오는가?
4. 새벽 2시에 기밀 서버 접속한 직원이 있다면, 다음 날 알 수 있는가?
5. 협력업체 직원의 접근이 계약 범위를 넘었다면, 자동 알림이 오는가?

## 3. 제안 솔루션 (Proposed Solution)

### 3.1 IRIS 아키텍처

기존 3종 보안 데이터를 한 곳에서 연결하고 AI로 교차 분석해요. **교체가 아니라 보강**이에요.

```
소만사 DLP 로그      ─┐
파수 DRM 로그        ─┤
안랩 EPP 로그        ─┼─→ [IRIS 상관분석 엔진] → 교차 패턴 탐지 → 위험 알림
KT텔레캅 CCTV/출입   ─┤                                          + 대응 가이드
HR 시스템 데이터      ─┘                                          + 컴플라이언스 리포트
```

- **기술 스택**: 오픈소스 기반 (Wazuh/ELK/Grafana)
- **Build 영역**: 상관분석 엔진, UEBA (직접 개발)
- **Partner 영역**: DLP (소만사), CCTV AI (한화비전/KT텔레캅)

### 3.2 4가지 핵심 화면

| 화면 | 설명 | 데이터 소스 |
|------|------|------------|
| 기밀문서 이상 접근 | "김모 과장이 2주간 타 팀 기밀문서 47건 조회" | 기존 DLP 로그 |
| 야간 이상행위 히트맵 | "새벽 2시에 3명이 동시 접속" | 기존 접근 로그 |
| 퇴직자 위험 스코어 | "퇴직 예정자 5명 중 2명, 다운로드 8배 증가" | DLP + HR 연결 |
| CCTV AI 필기 감지 | "보안구역에서 필기 행위 자동 감지" | 기존 CCTV + AI |

### 3.3 제품 패키지 (5단계)

| 패키지 | 컨셉 | 단가 (인/월) | 연간 (2,000명) | 구축 기간 | 커버 시나리오 |
|--------|------|-------------|---------------|-----------|-------------|
| **Small Start** | 데이터 연결 + 시각화 | 5,000~7,000원 | 1.2~1.7억 | 6~8주 | 7~8개 |
| **Lite** | AI 행위분석 시작 | 8,000~12,000원 | 1.9~2.9억 | +3~4주 | 8~9개 |
| **Standard** | 물리보안 AI 추가 | 15,000~20,000원 | 3.6~4.8억 | +12~20주 | 10~11개 |
| **Full** | 전사 통합 + 자동 대응 | 25,000~35,000원 | 6~8.4억 | +24~36주 | 12개 |
| **Purview Add-on** | MS Purview 보완 | 3,000~5,000원 | 0.7~1.2억 | 3~5주 | +2~3개 |

- Small Start~Lite + Purview Add-on은 IRIS만으로 완결 (물리보안 파트너 불필요)
- 12대 유출 시나리오: 수기 기록, 출력물, USB, 사진 촬영, 화면 캡처, 구두 전달, 메일/메신저, 클라우드, 권한 남용, 퇴직 전 반출, 공모 반출, 외주업체 유출

### 3.4 기대 효과

| 효과 | 현재 | IRIS 도입 후 | 개선폭 |
|------|------|-------------|--------|
| 탐지 시나리오 | 2~3개 | 7~12개 | 3~4배 |
| 탐지 속도 | 수일~수개월 | 수분~수시간 | 100배+ |
| 보안팀 분석 시간 | 일 2~3시간 | 30분 | 75% 절감 |
| 감사 리포트 | 연 2~4주 수작업 | 자동 생성 | 90% 절감 |
| 퇴직자 감지 | 사후 발각 | 퇴직 통보 즉시 | 사전 예방 |

> **정직한 한계**: 위 수치는 PoC에서 실환경 기반으로 검증 필요. CCTV AI 정확도 60~80% 추정, 실환경 50% 이하 가능.

## 7. 리스크 및 대응 (Risks & Mitigation)

### 7.1 법적 리스크 = 제로 (Small Start)

- PIA 불필요 (기존 DLP와 동일 데이터)
- 노사 협의 불필요 (CCTV AI 미포함)
- **설계 원칙으로 영구 제외**: 키스트로크 로깅, 화면 녹화, 이메일 내용 분석

### 7.2 기대 효과의 정직한 한계

| 한계 | 설명 |
|------|------|
| 기대 효과 수치 | PoC 검증 전 **가설**. 실제 환경에서 달라질 수 있음 |
| CCTV AI 정확도 | 60~80% **추정**. 실환경 50% 이하 가능 |
| 3종 교차 분석 효과 | **이론적**. 교차 패턴 실제 발생 빈도 미확인 |
| Small Start | **AI가 아닌 룰 기반**. AI 분석은 Lite 이상 |

### 7.3 경쟁 리스크

- SK쉴더스가 시장 진입 가능성 있으나, 서비스 → 제품 전환 비용이 높고 현재 외부 위협에 집중 중
- 1~2년 내 레퍼런스 선점이 핵심 전략
