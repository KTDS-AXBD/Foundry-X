# Deny 반도체 — 내부자 리스크 의사결정 시스템 PRD (최종)

> **IRIS** (Insider Risk Intelligence System) — 반도체 산업 특화 내부자 리스크 의사결정 지원 플랫폼

---

## 1. 개요 (Overview)

### 1.1 사업 배경

반도체 산업은 공정 레시피, 수율 데이터, 마스크 설계, 특허 초안 등 **고가치 기밀 자산**이 집중된 산업이에요. 내부자(임직원·협력사·계약직)에 의한 정보 유출은 SIEM/DLP 같은 기존 보안 시스템의 단순 룰 기반 경보로는 **정상/비정상 판별이 어렵고**, 보안 담당자의 경험과 직관에 의존하는 구조예요.

### 1.2 대상 고객

- **Deny 반도체** (가칭) — 반도체 제조 기업
- 사업장 7개소: 서울HQ, 이천, 화성, 평택, 청주, 구미, 대전
- 15개 부서, 2,500명+ 임직원 (정규직·협력사·계약직)
- 3,000개+ 디지털 자산 (10개 시스템: PLM, MES, EDMS, GitLab, LIMS, Snowflake 등)

### 1.3 제안 범위

실시간 내부자 리스크 이벤트를 모니터링하고, 보안 담당자가 **근거 기반으로 정상/비정상 가능성을 평가**하며, 시나리오별 **업무 절차(플레이북)와 필요 서류**를 즉시 안내받을 수 있는 **의사결정 지원 시스템**이에요.

### 1.4 PoC 데이터 규모

| 항목 | 수량 |
|------|------|
| 임직원 프로필 | 2,500명 |
| 디지털 자산 | 3,000건 |
| 행동 이벤트 | 1,080건 (정상 + 이상) |
| 업무 컨텍스트 | 400건 |
| 플레이북 룰 | 600건 |
| 사용자별 베이스라인 | 2,500건 |

### 1.5 문서 범위

이 PRD v3(최종)은 v1(PoC 기반 초안) + v2(AI 보강)를 통합 확정한 프로덕션 레디 문서예요. 모든 섹션이 자립적으로 구성되어 있으며, v1/v2 참조 없이 단독으로 사용 가능해요.

## 2. 문제 정의 (Problem Statement)

### 2.1 핵심 문제

1. **Alert Fatigue**: SIEM/DLP가 대량 경보를 생성하지만, 정상 업무와 실제 위협을 구분하는 데 과도한 시간이 소요돼요.
2. **컨텍스트 부재**: 이벤트 단독으로는 업무 맥락(프로젝트 마감, 감사 대응, 임원 보고 등)을 알 수 없어 오탐률이 높아요.
3. **대응 절차 파편화**: 위협 판별 후에도 어떤 서류를 준비하고 누구에게 에스컬레이션해야 하는지 표준화되지 않았어요.
4. **반도체 특수성**: 공정 레시피, 마스크 데이터, 수율 보고서 등 반도체 고유 자산은 일반적인 DLP 정책으로 보호하기 어려워요.

### 2.2 기존 한계

| 기존 시스템 | 한계 |
|-------------|------|
| SIEM (Splunk, QRadar 등) | 룰 기반 경보 → 대량 오탐, 컨텍스트 미반영 |
| DLP (Symantec, Forcepoint 등) | 데이터 흐름 차단 → 업무 생산성 저하, 우회 경로 미탐 |
| EDR (CrowdStrike 등) | 엔드포인트 중심 → 내부자 의도/맥락 판단 불가 |
| IAM (Okta 등) | 접근 제어 → 권한 내 악용(Need-to-know 위반) 미탐 |

### 2.3 정량적 문제 규모

| 지표 | 추정치 (Deny 반도체 기준) | 산출 근거 |
|------|--------------------------|-----------|
| 일일 보안 경보 건수 | 3,000~5,000건 | 2,500명 x 13개 행동유형 x 다중 시스템 |
| 보안 담당자 1인 일일 처리량 | 50~80건 | Ponemon 2024 벤치마크 |
| 미처리 경보 비율 | 60~80% | 처리량 / 발생량 갭 |
| 1건당 평균 분석 시간 | 30~60분 | 복수 시스템 교차 확인 필요 |
| 연간 내부자 사고 비용 (글로벌 평균) | $15.4M/건 | Ponemon 2024 |
| 반도체 IP 유출 시 예상 피해 | 수천억 원 | 공정 레시피 1건 = 수년간 R&D 투자 |

## 3. 제안 솔루션 (Proposed Solution)

### 3.1 시스템 구성 — 3-Panel 의사결정 프레임워크

#### Panel 1: 실시간 이벤트 모니터링
- 이벤트 스트림(SIEM/DLP/EDR/IAM 연동) 실시간 표시
- 4대 KPI 실시간 집계:
  - 최근 5분 이벤트 수
  - 고위험 자산(Restricted/TopSecret) 접근 수
  - 대량 출력(200+ 페이지) 건수
  - USB/외부공유 이벤트 수
- 리스크 스코어(0~100) 자동 산정: LOW(0~34) / MED(35~59) / HIGH(60~79) / CRIT(80~100)

#### Panel 2: 사건 분석 — 가능성 평가 (Hypothesis-based)

**Step 1) 상황 요약 (근거 기반)**
- 이벤트 메타데이터 (행동 유형, 볼륨, 자산 등급, 수출통제 여부)
- 사용자 프로필 (부서, 직급, 고용유형, 보안등급)
- 자산 정보 (시스템, 보안등급, 민감도, 경로)
- 베이스라인 대비 편차 (일평균 출력/다운로드 vs 현재)
- 업무 컨텍스트 추정 (회의, 마감, 감사, 고객 대응 등)

**Step 2) 정상/비정상 가능성 각 3개 (확률순)**

각 가능성에 대해:
- 추정 확률 (%)
- 근거 기반 설명
- 예상 피해/영향 (정량 + 정성)
- Quick Checks (즉시 확인 항목)

**Step 3) 업무 절차 (Playbook)**

선택한 가능성에 따라 단계별 조치 + 필요 서류 생성

#### Panel 3: 플레이북 실행 + 서류 생성
- 판단(정상/비정상) 선택 시 해당 플레이북의 단계별 체크리스트 표시
- 필요 서류 템플릿 자동 생성 (이벤트 데이터 사전 채움)
- PDF/DOCX 다운로드
- SLA 카운트다운 표시

### 3.3 리스크 스코어링 로직

#### 3.3.1 Phase 1 — PoC (휴리스틱)
- 행동 유형 가중치: PRINT(볼륨/10, 최대 40) / DOWNLOAD(볼륨/12, 최대 35) / COPY_USB(+35) / SHARE_LINK(+28)
- 자산 등급 가중치: Restricted/TopSecret(+22) / 수출통제(+12)
- 리스크 힌트 태그(+10)

#### 3.3.2 Phase 2 — 파일럿 (통계 기반)
- **피어그룹 Z-score**: 동일 부서/직급/프로젝트 그룹 내 행동 편차 측정
- **시계열 이상탐지**: 개인 베이스라인 대비 급변 감지 (이동평균 + 표준편차)
- **복합 이벤트 처리 (CEP)**: 단일 이벤트가 아닌 이벤트 시퀀스 패턴 매칭
  - 예: "권한 상승 요청 → 고보안 자산 접근 → 대량 다운로드 → USB 복사" = 복합 위협

#### 3.3.3 Phase 3 — 본사업 (ML/AI)
- **UEBA 모델**: 사용자/엔터티 행동 프로파일링 (비지도학습)
- **그래프 분석**: 사용자-자산-시스템 관계 그래프에서 이상 패턴 탐지
- **자연어 처리**: 업무 컨텍스트(이메일, 캘린더, 프로젝트 관리 시스템) 자동 추출

### 3.4 업무 절차 (Workflow) 자동 생성

**비정상 판단 시 (5단계):**
1. 증적 확보 → 세션/디바이스/IP, 자산 메타데이터, 로그 보존
2. 업무 필요성 확인 (Need-to-know 검증)
3. 즉시 통제 (MFA 강화, 세션 종료, USB 차단, 링크 회수)
4. 조사/분석 (타임라인 재구성, 포렌식)
5. 에스컬레이션 + 후속조치 (SOC/CISO/HR/법무)

**정상 판단 시 (4단계):**
1. 증적 확보
2. 업무 필요성 확인
3. 정당성 문서화 (감사 대비)
4. 탐지 룰 튜닝 (오탐 저감)

### 3.5 문서/증적 템플릿 (15종+)

증적수집 체크리스트, 세션 트레이스 리포트, 파일 메타데이터, DLP/프린트스풀 로그, 업무 R&R 정의서, 프로젝트 참여자 명단, 임시권한 승인서, 통제조치 승인 기록, 계정/세션 차단 요청서, 행동 타임라인 리포트, 포렌식 요청서, 사고 보고서, 정당성 확인서, 탐지룰 변경요청서, Baseline 재학습 계획, 변경 이력 관리 로그 등

### 3.6 사용자 스토리 및 수용 기준

#### 페르소나 정의

| 페르소나 | 역할 | 주요 니즈 |
|----------|------|----------|
| P1. 보안 분석가 (SOC Analyst) | SIEM 경보 1차 분석, 일일 수백 건 처리 | 오탐 빠르게 걸러내기, 실제 위협 놓치지 않기 |
| P2. 보안 관리자 (Security Manager) | 보안 정책 수립, 팀 관리, 경영진 보고 | 팀 효율 지표, 대시보드, 정기 보고서 |
| P3. CISO | 전사 보안 총괄, 규제 대응, 투자 의사결정 | 리스크 현황 요약, ROI 수치, 컴플라이언스 상태 |
| P4. IT 관리자 | 시스템 연동, 로그 수집, 인프라 운영 | 안정적 연동, 로그 누락 없음, 성능 저하 없음 |
| P5. 법무/감사팀 | 증적 확보, 법적 대응, 감사 보고서 | 법적 증거력 있는 증적, 타임라인, 체인 오브 커스터디 |

#### US-01: 이벤트 실시간 모니터링 (P1)
> 보안 분석가로서, 실시간 이벤트 스트림에서 HIGH/CRIT 등급 이벤트를 즉시 식별하여 우선 분석하고 싶어요.

**수용 기준:**
- [ ] 이벤트 발생 후 5초 이내에 Panel 1에 표시
- [ ] 리스크 등급(LOW/MED/HIGH/CRIT)별 색상 구분 표시
- [ ] HIGH/CRIT 이벤트 발생 시 시각적 알림(깜빡임 또는 사운드)
- [ ] 최근 5분/1시간/24시간 기간 전환 가능
- [ ] 4대 KPI 카드가 5초 간격으로 자동 갱신

#### US-02: 가설 기반 분석 (P1)
> 보안 분석가로서, 이벤트 선택 시 정상/비정상 각 3개 가능성과 근거를 확인하여, 경험에 의존하지 않고 체계적으로 판단하고 싶어요.

**수용 기준:**
- [ ] 이벤트 클릭 시 Panel 2에 3단계 분석(상황 요약 → 가능성 목록 → 플레이북) 표시
- [ ] 각 가능성에 추정 확률(%), 근거, 예상 피해, Quick Checks 포함
- [ ] 분석 결과 로딩 시간 3초 이내
- [ ] 가능성 선택 시 해당 플레이북이 Panel 3에 즉시 표시
- [ ] 사용자가 가능성을 선택하면 해당 판단이 로그에 기록 (감사 추적)

#### US-03: 플레이북 실행 및 서류 생성 (P1)
> 보안 분석가로서, 판단 후 단계별 조치 체크리스트와 필요 서류를 자동으로 받아 빠짐없이 대응하고 싶어요.

**수용 기준:**
- [ ] 판단(정상/비정상) 선택 시 해당 플레이북의 단계별 체크리스트 표시
- [ ] 각 단계에 필요한 서류 템플릿을 자동 생성 (이벤트 데이터 사전 채움)
- [ ] 서류를 PDF/DOCX 포맷으로 다운로드 가능
- [ ] 체크리스트 진행 상태 저장 (세션 종료 후에도 유지)
- [ ] SLA 시간 카운트다운 표시 (플레이북 룰의 sla_minutes 기반)

#### US-04: 대시보드 및 보고서 (P2)
> 보안 관리자로서, 팀의 일일/주간/월간 처리 현황과 효과 지표를 한눈에 확인하고 싶어요.

**수용 기준:**
- [ ] 기간별(일/주/월) 처리 건수, 평균 처리 시간, 오탐률 표시
- [ ] 리스크 등급별 분포 차트 (시계열 트렌드)
- [ ] 시나리오별 발생 빈도 통계
- [ ] 사업장별/부서별 필터링
- [ ] PDF 보고서 자동 생성 (경영진 보고용)

#### US-05: 증적 체인 관리 (P5)
> 법무/감사팀으로서, 사건 발생부터 종결까지의 모든 판단 근거와 조치 이력을 변경 불가한 형태로 보관하고 싶어요.

**수용 기준:**
- [ ] 모든 사용자 판단(가능성 선택, 플레이북 실행)에 타임스탬프 + 사용자 ID 기록
- [ ] 증적 데이터는 append-only 저장 (수정/삭제 불가)
- [ ] 사건 타임라인을 시각적으로 표시 (이벤트 → 분석 → 판단 → 조치)
- [ ] 체인 오브 커스터디 보고서 자동 생성
- [ ] 보관 기간: 최소 5년 (산업기술보호법 기준)

#### US-06: 시스템 연동 관리 (P4, 파일럿)
> IT 관리자로서, SIEM/DLP/EDR/IAM 시스템 연동 상태를 모니터링하고, 로그 수집 누락을 즉시 감지하고 싶어요.

**수용 기준:**
- [ ] 연동 시스템별 상태(정상/지연/실패) 대시보드 표시
- [ ] 로그 수집 지연 5분 초과 시 알림
- [ ] 일일 로그 수집 건수 통계 (시스템별)
- [ ] 연동 설정 변경 이력 관리
