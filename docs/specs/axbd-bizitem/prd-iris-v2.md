## 1. 개요 (Overview)

> 1차 PRD §1~§4 내용을 계승. 아래는 변경/추가 사항만 기술.

**IRIS(Internal Risk Identification System)**는 기업 내부에서 이미 운영 중인 DLP, CCTV, 출입통제, HR 데이터를 AI로 교차 연결하여, 기존 보안 체계가 탐지하지 못했던 내부자 유출 패턴을 식별하는 보안 강화 플랫폼이에요.

- **사업 주체**: KT DS AX사업개발본부 BD팀
- **핵심 철학**: 새 장비 도입이 아닌, 기존 보안 데이터의 **교차 연결**로 사각지대 제거
- **현재 상태**: 시뮬레이션 프로토타입 4종 완료, 2단계 발굴 2회 완료
- **프로토타입**: dmp-portal.vercel.app

### 1.1 제품 비전

> "모든 기업이 이미 가진 보안 데이터만으로, 내부 유출을 사전에 감지할 수 있게 한다."

### 3.4 데이터 파이프라인

```
[데이터 소스] → [Collector Agent] → [Kafka] → [Logstash Parser] → [Elasticsearch]
                                       ↓
                              [IRIS Engine] → [PostgreSQL (이벤트/점수)]
                                       ↓
                              [알림 서비스] → [이메일/Slack/Teams]
```

- **처리 지연 목표**: 소스 → ES 인덱싱 ≤ 60초, ES → 알림 ≤ 300초
- **버퍼링**: Kafka가 소스 장애/지연 시 최대 24시간 버퍼
- **재처리**: Kafka offset 기반 재처리 지원 (오탐 룰 수정 후 재분석)

### 3.5 UEBA 모델 설계

| 항목 | 상세 |
|------|------|
| **입력 피처** | 로그인 시간, 파일 접근 빈도, 접근 문서 등급, 출입 시각, 이메일 발송량 등 20+ 피처 |
| **기준선** | 개인별 30일 이동 평균 + 부서 평균 2중 기준 |
| **이상 탐지** | Isolation Forest + Autoencoder 앙상블 |
| **학습 주기** | 주 1회 재학습 (증분), 월 1회 전체 재학습 |
| **설명 가능성** | SHAP 기반 피처 기여도 — "왜 이 사람이 위험한가" 설명 |
| **정확도 목표** | Precision ≥ 70%, Recall ≥ 60% (Phase 1 PoC 기준) |

### 4.1 데이터 소스별 연동 사양

#### 4.1.1 DLP (소만사 Privacy-i)

| 연동 방식 | Syslog (UDP/TCP 514) 또는 CSV Export (SFTP) |

| 인증 | IP 화이트리스트 + 전용 계정 |
| 일일 예상량 | 50만~200만 건/일 (2,000명 기준) |

| 항목 | 상세 |
|------|------|
| 연동 방식 | REST API (Pull, 5분 주기) |
| 수집 필드 | timestamp, card_id, employee_id, zone_name, zone_level(일반/보안/최고보안), direction(입/출) |
| 인증 | API Key + mTLS |
| 일일 예상량 | 1만~5만 건/일 |

#### 4.1.3 HR 시스템

| 항목 | 상세 |
|------|------|
| 연동 방식 | SFTP (CSV) 또는 REST API |
| 수집 필드 | employee_id, name(해시), department, position, hire_date, resign_date, resign_notice_date, status(재직/퇴직예정/퇴직) |
| 수집 주기 | 일 1회 배치 (새벽 2시) |
| 개인정보 처리 | 이름은 SHA-256 해시 + 별도 매핑 테이블 (CISO만 접근 가능) |

### 6.2 ACV 3.2억 원 산출 근거 (5년차 평균)

```
5년차 총 매출 118억 ÷ 5년차 누적 고객 37개사 = ACV 약 3.2억 원

고객 믹스 가정:
- Small Start 40% (14개사): 평균 1.5억 × 14 = 21억
- Lite 30% (11개사): 평균 2.4억 × 11 = 26.4억
- Standard 20% (8개사): 평균 4.2억 × 8 = 33.6억
- Full 10% (4개사): 평균 7.2억 × 4 = 28.8억
- Purview Add-on (8개사 중복): 평균 1.0억 × 8 = 8억
합계: 117.8억 ≈ 118억
```
