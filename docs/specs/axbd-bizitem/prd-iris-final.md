# IRIS PRD v3 (Final) — Internal Risk Identification System

> **문서 목적**: v1(사업기획서 기반)과 v2(AI 보강)를 통합 확정한 최종 PRD. 실행 가능한 수준의 기능 정의, 수용 기준, 우선순위 백로그, Definition of Done을 포함한다.

## 1. 제품 비전 및 개요

### 1.1 제품 비전

> "모든 기업이 이미 가진 보안 데이터만으로, 내부 유출을 사전에 감지할 수 있게 한다."

### 1.2 개요

**IRIS(Internal Risk Identification System)**는 기업이 이미 운영 중인 DLP, CCTV, 출입통제, HR 데이터를 AI로 교차 연결하여, 기존 보안 체계가 탐지하지 못했던 내부자 유출 패턴을 식별하는 보안 강화 플랫폼이에요.

- **사업 주체**: KT DS AX사업개발본부 BD팀
- **핵심 철학**: 새로운 보안 장비 도입이 아닌, 기존 보안 데이터의 **교차 연결**로 사각지대 제거
- **현재 상태**: 시뮬레이션 데이터 기반 프로토타입 4종 완료, 2단계 발굴 프로세스 2회(v1+v2) 완료
- **프로토타입 데모**: dmp-portal.vercel.app

### 1.3 핵심 수치 요약

| 항목 | 수치 |
|------|------|
| TAM (한국 시장) | 2,000억 원 |
| SAM (대기업 100개사) | 700억 원 |
| SOM (보수적 5년차) | 118억 원 |
| Phase 0 투자 | 5,700만 원 |
| BEP | 3년차 (영업이익 10.8억) |
| 5년차 영업이익률 | 42% |

### 1.4 IRIS 아키텍처 개요

```
소만사 DLP 로그      ─┐
파수 DRM 로그        ─┤
안랩 EPP 로그        ─┼─→ [IRIS 상관분석 엔진] → 교차 패턴 탐지 → 위험 알림
KT텔레캅 CCTV/출입   ─┤                                          + 대응 가이드
HR 시스템 데이터      ─┘                                          + 컴플라이언스 리포트
```

- **Build 영역**: 상관분석 엔진, UEBA (자체 개발)
- **Partner 영역**: DLP (소만사), CCTV AI (한화비전/KT텔레캅)
- **기술 스택**: 오픈소스 기반 (Wazuh/ELK/Grafana) + 자체 엔진

## 2. 문제 정의

### 2.1 내부자 기술유출은 20년간 반복되고 있다

2003년부터 2025년까지 주요 사고만 12건 이상이 확인되며, 피해 기업 대부분이 국내 최고 수준의 보안 체계를 갖추고 있었어요.

| 사건 | 시기 | 유출 수법 | 피해 |
|------|------|-----------|------|
| 삼성전자 → CXMT | 2016~2020 | 손으로 필기하여 반출 + 핵심 인력 이직 | 수십조 원 (D램 공정) |
| 쿠팡 고객 정보 | 2025.6~11 | 퇴직자 잔존 권한으로 5개월간 접근 | 3,370만 고객 정보 |
| 카드 3사 | 2012~2014 | 협력업체 직원 USB 복사 반출 | 1억 580만 건 (역대 최대) |
| SK하이닉스 HKMG | 2018~2020 | 협력업체 부사장 접근 권한 악용 | 수천억 원 |
| Google AI칩 설계 | 2022~2023 | 개인 클라우드 업로드 + 이중 근무 | 국가안보 위협 |
| Intel 기밀 반출 | 2024.7 | 퇴직 통보 후 개인 NAS로 18,000건 복사 | 소송 진행 중 |

### 2.2 기존 보안 체계의 구조적 한계

기존 보안이 나빠서가 아니라, **설계 목적이 "내부자 유출 패턴 탐지"가 아니었기 때문**이에요.

| 시스템 | 잘하는 것 | 못하는 것 |
|--------|-----------|-----------|
| DLP | 디지털 경로 2~3개 차단 | 교차 패턴 탐지 |
| CCTV | 영상 녹화 (사후 포렌식) | 실시간 행위 분석 |
| SIEM | 외부 위협 로그 수집 | 내부자 행동 분석 |
| **공통 문제** | 개별 동작 | **서로 연결되어 있지 않음** |

### 2.3 고객 자가 점검 질문 (Pain Point)

1. 직원이 기밀을 종이에 적어가면, 지금 알 수 있는가?
2. 퇴직 예정자의 파일 다운로드가 갑자기 8배 늘었다면, 보이는가?
3. 퇴직한 직원이 아직 시스템에 접속하고 있다면, 알림이 오는가?
4. 새벽 2시에 기밀 서버 접속한 직원이 있다면, 다음 날 알 수 있는가?
5. 협력업체 직원의 접근이 계약 범위를 넘었다면, 자동 알림이 오는가?

### 6.4 데이터 파이프라인

```
[데이터 소스] → [Collector Agent] → [Kafka] → [Logstash Parser] → [Elasticsearch]
                                       ↓
                              [IRIS Engine] → [PostgreSQL (이벤트/점수)]
                                       ↓
                              [알림 서비스] → [이메일/Slack/Teams]
```

- **처리 지연**: 소스 → ES 인덱싱 ≤ 60초, ES → 알림 ≤ 300초
- **버퍼링**: Kafka 최대 24시간 버퍼 (소스 장애/지연 시)
- **재처리**: Kafka offset 기반 (오탐 룰 수정 후 재분석)

### 6.5 UEBA 모델 설계

| 항목 | 상세 |
|------|------|
| 입력 피처 | 로그인 시간, 파일 접근 빈도, 문서 등급, 출입 시각, 이메일 발송량 등 20+ 피처 |
| 기준선 | 개인별 30일 이동 평균 + 부서 평균 2중 기준 |
| 이상 탐지 | Isolation Forest + Autoencoder 앙상블 |
| 학습 주기 | 주 1회 증분, 월 1회 전체 |
| 설명 가능성 | SHAP 기반 피처 기여도 ("왜 이 사람이 위험한가") |
| 정확도 목표 | Precision ≥ 70%, Recall ≥ 60% (Phase 1 PoC 기준) |

## 7. 데이터 아키텍처 및 연동 사양

### 7.1 데이터 소스별 연동

#### DLP (소만사 Privacy-i)

| 항목 | 상세 |
|------|------|
| 연동 방식 | Syslog (UDP/TCP 514) 또는 CSV Export (SFTP) |
| 수집 필드 | timestamp, user_id, action, file_name, file_path, dest_type, file_size, sensitivity_level |
| 수집 주기 | 실시간 (Syslog) 또는 5분 배치 (CSV) |
| 인증 | IP 화이트리스트 + 전용 계정 |
| 일일 예상량 | 50만~200만 건/일 (2,000명) |

#### 출입통제 (KT텔레캅)

| 항목 | 상세 |
|------|------|
| 연동 방식 | REST API (Pull, 5분 주기) |
| 수집 필드 | timestamp, card_id, employee_id, zone_name, zone_level, direction |
| 인증 | API Key + mTLS |
| 일일 예상량 | 1만~5만 건/일 |

#### HR 시스템

| 항목 | 상세 |
|------|------|
| 연동 방식 | SFTP (CSV) 또는 REST API |
| 수집 필드 | employee_id, name(SHA-256 해시), department, position, hire_date, resign_date, status |
| 수집 주기 | 일 1회 (새벽 2시) |
| 개인정보 처리 | 이름 해시 + 별도 매핑 테이블 (CISO만 접근) |

#### CCTV AI (Phase 2 — Standard 패키지)

| 항목 | 상세 |
|------|------|
| 연동 방식 | Webhook (Push) — AI 분석 결과만 (영상 원본 미수집) |
| 수집 필드 | timestamp, camera_id, zone_name, event_type, confidence_score, snapshot_url |
| 인증 | Webhook Secret + IP 화이트리스트 |

| 내부자 유출이 본업 | 경쟁사는 부가 기능, IRIS는 12대 시나리오가 설계 기준 |
| 제품 통합 (vs 서비스) | SK쉴더스: 사람 수동 연결(서비스), IRIS: AI 자동 연결(제품) |

### 10.4 경쟁 비교

| 비교 | SK쉴더스 | IRIS |
|------|---------|------|
| 본질 | 서비스 (관제요원) | 제품 (AI) |
| 초점 | 외부 위협 (해킹) | 내부자 위협 (유출) |
| 교차 분석 | 수동 (수십 분) | AI 자동 (수 초) |
| 확장 비용 | 관제요원 비례 증가 | 서버 추가만 (40~60% 저렴) |

### 10.5 기대 효과

| 효과 | 현재 | IRIS 도입 후 | 개선폭 |
|------|------|-------------|--------|
| 탐지 시나리오 | 2~3개 | 7~12개 | 3~4배 |
| 탐지 속도 | 수일~수개월 | 수분~수시간 | 100배+ |
| 보안팀 분석 시간 | 일 2~3시간 | 30분 | 75% 절감 |
| 감사 리포트 | 연 2~4주 수작업 | 자동 생성 | 90% 절감 |
| 퇴직자 감지 | 사후 발각 | 퇴직 통보 즉시 | 사전 예방 |

> **정직한 한계**: 위 수치는 PoC에서 실환경 기반 검증 필요. CCTV AI 정확도 60~80% 추정(실환경 50% 이하 가능). 교차 분석 효과는 이론적이며 실제 발생 빈도 미확인. Small Start는 AI가 아닌 룰 기반.
