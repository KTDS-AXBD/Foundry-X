# XR Exhibition Studio — AI 도슨트 기반 VR 몰입형 전시 프로듀싱 PRD (최종)

---

## 1. 개요 (Overview)

**사업명:** XR Exhibition Studio

**한줄 정의:** 전시 기획사의 콘텐츠·IP 역량에 KT DS의 AI 도슨트 에이전트와 VR 인프라 운영을 결합해, 글로벌 수준의 VR 몰입형 전시를 국내에서 공동 기획·운영하는 신사업

**담당:** KT DS AX사업개발본부 BD팀 · 김정원

**엘리베이터 피치 (30초):**
> "전시 기획사와 문화공간은 VR 전시를 하고 싶지만, 수억 원의 콘텐츠 제작비와 기술 운영 부담이 장벽입니다. XR Exhibition Studio는 글로벌 VR 콘텐츠 파트너십과 KT DS의 AI 도슨트 에이전트를 결합해, 쿠푸왕급 VR 전시를 턴키로 제공합니다. 관람객은 AI 가이드와 대화하며 자신만의 전시를 경험하고, 운영사는 관람 데이터 기반으로 전시를 지속 최적화할 수 있습니다."

**핵심 가치 3가지:**
1. **AI가 안내하는 나만의 VR 전시** — AI 도슨트가 관람객 질문에 실시간 응답, 개인화된 해설 제공
2. **전시 기획사를 위한 턴키 기술 파트너** — VR 기술·AI·인프라를 원스톱 제공, 기획사는 콘텐츠에 집중
3. **데이터 기반 전시 최적화** — 관람객 행동 데이터(시선·동선·Q&A)로 전시 품질 지속 개선

**전략적 의미:**
- Agentic AI 전문회사 Identity 정착에 부합 — AI 도슨트는 자율적으로 관람객과 대화하며 경험을 안내하는 대표적 소비자 접점 Agentic AI
- 실패해도 AI 도슨트 기술은 관광·교육·기업 체험 등 타 도메인으로 전환 가능한 **재사용 AI 자산**

## 2. 문제 정의 (Problem Statement)

### 2-1. 추진 배경 (3축)

| 축 | 핵심 논거 |
|----|-----------|
| **수익성** | 글로벌 LB-VR 시장 CAGR 32%. 쿠푸왕 200만+ 관람, 아르떼뮤지엄 월 $350만 매출 실증. AI 도슨트 도입 시 인간 도슨트 대비 약 90%+ 비용 절감 |
| **KT 적합성** | KT 클라우드/5G/네트워크가 VR 전시 인프라의 핵심. 통신3사 메타버스 철수 후 "AI 기반 XR" 재진입 스토리. KT estate 등 그룹사를 통한 베뉴 접근 |
| **실행력** | AI 도슨트(LLM+음성+RAG) 자체 개발 역량 보유. SI/SM으로 대규모 인프라 구축·운영 경험. 콘텐츠·장비는 파트너 네트워크로 보완 |

### 2-2. Pain Point

- VR 콘텐츠 제작비 수십억원, 기간 수년 소요
- VR 인프라(트래킹, 네트워크, 서버) 설치·운영 역량 부재
- 기존 전시(미디어아트, 팝업) 포화로 차별화 어려움
- VR 전시를 원스톱으로 제공하는 국내 사업자 부재

### 2-3. 기존 VR 전시 vs XR Exhibition Studio

| 비교 항목 | 기존 VR 전시 | XR Exhibition Studio |
|-----------|-------------|---------------------|
| 관람 안내 | 고정 오디오 가이드 또는 인간 도슨트 | **AI 도슨트 에이전트** — 질문에 실시간 응답, 개인화 해설 |
| 언어 지원 | 2~3개 언어 오디오 트랙 | **다국어 실시간 대화** (한/영/중/일) |
| 개인화 | 없음 (모두 같은 경험) | 관심사·연령·관람 이력 기반 **맞춤 안내** |
| 운영 데이터 | 입장객 수 정도 | 시선·동선·Q&A·체류시간 **전방위 분석** |
| 도슨트 비용 | 인건비 약 2~3억/6개월 | AI로 **약 90%+ 절감** (LLM API 비용만 발생) |

### 2-4. 1차 고객: 전시 기획사

| 판단 기준 | 전시 기획사 | 복합쇼핑몰 | 지자체 |
|-----------|-----------|-----------|--------|
| Pain 직접성 | **높음** — VR 기술 역량 부재 | 중간 | 중간 |
| 예산 보유 | **보유** — 전시 프로젝트 예산 | 보유 | 정부 과제 필요 |
| 의사결정 속도 | **빠름** — 3~6개월 | 중간 | 느림 |
| 공동 기획 가능 | **최적** — 콘텐츠+기획 역량 | 어려움 | 제한적 |

**비치헤드 고객 후보:**
- GNC미디어 (국내 블록버스터 전시 대표 기업, 1993년 설립)
- 스타필드 코엑스몰 (일 15만명 방문)
- 영월군 ('왕과 사는 남자' 1,200만 흥행으로 단종 콘텐츠 투자 의지 급상승)

### 2-5. 페르소나 정의

| 페르소나 | 역할 | 핵심 니즈 | Pain |
|----------|------|-----------|------|
| **P1. 전시 기획 PD** | 전시 기획사 프로듀서 | 차별화된 전시 콘텐츠로 흥행 성공 | VR 기술 역량 없어 시도 불가 |
| **P2. 베뉴 운영자** | 복합몰/박물관 시설 관리자 | 집객력 높은 킬러 콘텐츠 유치 | 전시 직접 기획 불가, 기획사 의존 |
| **P3. 일반 관람객** | 20~40대 문화 소비자 | 몰입감 높은 새로운 경험 | 반복적 미디어아트에 피로감 |
| **P4. 외국인 관광객** | 한국 방문 관광객 | 언어 장벽 없는 문화 체험 | 오디오 가이드 2~3개 언어 한정 |
| **P5. KT DS PM** | 사업 운영/기술 총괄 | 안정적 운영 + 데이터 기반 의사결정 | 전시 도메인 노하우 부재 |
| **P6. AI 도슨트 관리자** | AI 모델/콘텐츠 운영 | 도슨트 품질 유지 + 신규 전시 빠른 세팅 | 전시별 도메인 지식 구축 비용 |

## 3. 제안 솔루션 (Proposed Solution)

### 3-1. 서비스 구성안

| 역할 | 내용 | Build / Partner |
|------|------|-----------------|
| 전시 기획 | 테마 선정, 스토리라인, 학술/IP 파트너 섭외 | **Build** |
| AI 도슨트 에이전트 | VR 내 AI 해설 가이드 (LLM+음성+맥락 인식+다국어) | **Build** (핵심 자체 기술) |
| AI 전시 분석 | 관람객 데이터 수집·분석 (시선, 동선, Q&A) | **Build** |
| 기술 운영 (SI/SM) | VR 인프라 설치·운영·관제, 네트워크, 클라우드 | **Build** |
| 고객·베뉴 발굴 | 전시장·복합몰·박물관 영업, 계약 | **Build** |
| VR 콘텐츠 제작 | 3D 환경, 인터랙션, 스토리 체험 | **Partner** (Excurio, ADMI 등) |
| VR 장비 | HMD, 트래커, 서버 등 하드웨어 | **Partner** (HTC, 렌탈사) |
| 마케팅·티켓 판매 | 브랜딩, 온라인 마케팅, 예매 시스템 | **Partner** |

### 3-2. 유저 시나리오

**본사업 시나리오: 글로벌 IP VR 전시 (예: 타이타닉 해저 탐사)**

1. **입장 & 오리엔테이션** — 관람객이 VR HMD를 착용하고 심해 잠수정에 탑승. AI 도슨트가 인사 및 탐사 안내
2. **자유 탐험 & AI 대화** — 해저 3,800m의 타이타닉호를 자유보행으로 탐험. 궁금한 것을 물으면 AI가 실시간 응답
3. **개인화 안내** — 관심사에 따라 AI가 경로 추천 (역사→선장실, 건축→계단 장식, 아동→게임형 미션)
4. **데이터 분석 & 리포트** — 관람 종료 후 운영사에 관람 분석 대시보드 제공 (인기 구간, FAQ, 체류 시간 등)

**PoC 시나리오: 한국 역사 VR (단종 — 영월 유배지)**

- **범위:** 영월 청령포 1개 공간 (국가유산청 3D 에셋 활용)
- **AI 도슨트:** AI 단종이 직접 자신의 이야기를 들려주는 형태
- **목적:** AI 도슨트 기술 실행성 검증 (응답 지연 <2초, 역사적 정확도 90%+)
- **데이터:** 국가유산청 3D 에셋(무료) + 역사 자료 RAG 구축
- **킬러 시나리오:** "영화에서 2시간 동안 본 단종의 세계를, VR로 직접 걸어 다니며 체험한다. AI 단종이 당신에게 직접 자신의 이야기를 들려준다."

### 3-3. Before → After 전환

| Before (전시 기획사의 현실) | After (XR Exhibition Studio) |
|-----------------------------|------------------------------|
| VR 전시를 하고 싶지만 기술 역량 없음 | KT DS가 기술 전체를 턴키 제공 |
| 콘텐츠·기술·장비·운영을 각각 다른 업체에 의뢰 | 기획사는 콘텐츠·스토리·IP에만 집중 |
| 통합 관리 불가, 기술 리스크 직접 부담 | 리스크 분담 (공동 투자 + 수익 배분) |
| 고정 오디오 가이드 → 일방적 관람 경험 | AI 도슨트가 관람객과 실시간 대화 |
| 관람객 행동 데이터 수집·활용 불가 | 데이터 기반 전시 지속 최적화 |

## 4. 사용자 스토리 및 수용 기준 (User Stories & Acceptance Criteria)

### 4-1. 관람객 경험 (P3, P4)

#### US-001: VR 전시 입장 및 오리엔테이션
- **As a** 관람객
- **I want to** VR HMD 착용 후 30초 이내에 AI 도슨트의 인사와 안내를 받고 싶다
- **So that** 처음 VR을 경험하더라도 당황하지 않고 전시에 몰입할 수 있다

**Acceptance Criteria:**
- [ ] HMD 착용 감지 후 10초 이내 AI 도슨트 아바타가 시야에 등장
- [ ] 관람객 언어 자동 감지 (한/영/중/일) 또는 수동 선택 UI 제공
- [ ] 오리엔테이션 안내 60초 이내 완료 (스킵 가능)
- [ ] VR 조작법 튜토리얼 포함 (이동, 상호작용, AI 호출 제스처)

#### US-002: AI 도슨트와 자연어 대화
- **As a** 관람객
- **I want to** 전시물 앞에서 궁금한 점을 음성으로 질문하고 즉시 답변을 받고 싶다
- **So that** 일방적 관람이 아닌 쌍방향 학습 경험을 할 수 있다

**Acceptance Criteria:**
- [ ] 음성 인식 정확도 95% 이상 (표준어 기준, 소음 환경 85dB 이하)
- [ ] 질문 후 응답 시작까지 2초 이내 (p95)
- [ ] 응답 길이 30초 이내 (관람 흐름 방해 방지)
- [ ] "잘 모르겠다"는 답변 비율 5% 이하 (도메인 내 질문 기준)
- [ ] 부적절한 질문(욕설, 무관 질문) 필터링 및 정중한 안내 응답

#### US-003: 개인화된 전시 경로 추천
- **As a** 관람객
- **I want to** 내 관심사에 맞는 관람 경로를 AI가 추천해주길 원한다
- **So that** 제한된 시간 안에 가장 흥미로운 전시물을 놓치지 않을 수 있다

**Acceptance Criteria:**
- [ ] 관람 시작 2분 이내 관심사 프로파일링 완료 (3개 질문 또는 행동 기반)
- [ ] 최소 3가지 경로 옵션 제안 (예: 역사 중심 / 건축 중심 / 하이라이트)
- [ ] 경로 변경 시 실시간 재추천 (관람객이 이탈한 경로 감지)
- [ ] 아동 모드 지원 (연령대별 난이도 조절)

#### US-004: 다국어 실시간 대화
- **As a** 외국인 관광객
- **I want to** 모국어로 AI 도슨트와 대화하고 싶다
- **So that** 전시 내용을 깊이 이해할 수 있다

**Acceptance Criteria:**
- [ ] 한국어, 영어, 중국어(보통화), 일본어 4개 언어 지원
- [ ] 언어 전환 5초 이내 (mid-session 전환 가능)
- [ ] 고유명사/역사 용어 번역 정확도 90% 이상
- [ ] 문화적 맥락 보충 설명 제공 (예: "단종은 조선의 6대 왕으로...")

### 4-2. 전시 기획사 (P1)

#### US-005: 전시 기획 협업 대시보드
- **As a** 전시 기획 PD
- **I want to** AI 도슨트에 투입할 도메인 지식을 웹 대시보드에서 관리하고 싶다
- **So that** 개발 팀 없이도 전시 콘텐츠를 업데이트할 수 있다

**Acceptance Criteria:**
- [ ] 도메인 지식 문서 업로드 UI (PDF, DOCX, TXT, 이미지)
- [ ] AI 도슨트 응답 미리보기 (시뮬레이터)
- [ ] Q&A 로그 조회 및 미응답 질문 리스트
- [ ] 전시 스토리라인/경로 편집기

#### US-006: 관람 데이터 분석 리포트
- **As a** 전시 기획 PD
- **I want to** 일별/주별 관람 데이터 분석 리포트를 받고 싶다
- **So that** 전시 콘텐츠와 동선을 데이터 기반으로 개선할 수 있다

**Acceptance Criteria:**
- [ ] 일별 자동 리포트 (관람객 수, 평균 체류시간, 인기 구간 Top 5)
- [ ] 주별 트렌드 리포트 (FAQ 변화, 이탈 구간, 재방문율)
- [ ] 히트맵 시각화 (동선/시선 분포)
- [ ] 리포트 PDF/Excel 다운로드

### 4-3. KT DS 운영 (P5, P6)

#### US-007: AI 도슨트 모니터링 대시보드
- **As a** AI 도슨트 관리자
- **I want to** 실시간 AI 도슨트 상태를 모니터링하고 이상 감지 시 알림을 받고 싶다
- **So that** 장애를 사전에 감지하고 관람 경험 저하를 방지할 수 있다

**Acceptance Criteria:**
- [ ] 실시간 동시 접속 세션 수, 평균 응답시간, 에러율 대시보드
- [ ] 응답시간 > 3초 또는 에러율 > 5% 시 알림 (Slack/Email)
- [ ] AI 응답 품질 샘플링 (일 100건 랜덤 검수)
- [ ] LLM API 사용량/비용 실시간 추적

#### US-008: 신규 전시 AI 도슨트 세팅
- **As a** AI 도슨트 관리자
- **I want to** 새로운 전시의 AI 도슨트를 2주 이내에 세팅하고 싶다
- **So that** 전시 오픈 일정에 차질이 없다

**Acceptance Criteria:**
- [ ] 도메인 지식 RAG 구축 파이프라인 (문서 수집 → 임베딩 → 인덱싱)
- [ ] 전시별 페르소나/톤 설정 UI (예: "단종 역할극" vs "중립 해설자")
- [ ] 사전 QA 테스트 100문항 자동 생성 + 정확도 측정
- [ ] A/B 테스트 기능 (응답 스타일, 추천 경로 등)

## 6. 기술 아키텍처 (Technical Architecture)

### 6-1. AI 도슨트 엔진 아키텍처

```
[VR HMD] --음성--> [Edge Server] --텍스트--> [AI Backend]
                                                   |
                        [ASR] -> [NLU/Router] -> [LLM + RAG] -> [Response Gen] -> [TTS]
                                      |                               |
                              [Context Manager]              [Safety Filter]
                                      |                               |
                              [Session Store]              [Quality Monitor]
```

#### 핵심 컴포넌트

| 컴포넌트 | 기술 스택 (권장) | 역할 |
|----------|-----------------|------|
| **ASR (음성인식)** | Whisper Large v3 / Google Cloud STT | 다국어 음성 → 텍스트 변환 |
| **NLU/Router** | Intent classifier + Entity extractor | 질문 유형 분류 (해설 요청 / 경로 안내 / 잡담 등) |
| **LLM** | Claude 3.5 Sonnet / GPT-4o (멀티 벤더) | 응답 생성 + 페르소나 유지 |
| **RAG** | Vector DB (Pinecone/Weaviate) + 도메인 문서 | 전시별 지식 기반 검색 |
| **Context Manager** | Redis + Session DB | 대화 이력 관리, 관람 위치 연동 |
| **Safety Filter** | Guardrails + Content Filter | 부적절 응답 차단, 역사적 사실 검증 |
| **TTS** | ElevenLabs / Azure TTS | 자연스러운 음성 합성 (캐릭터별 음색) |
| **Quality Monitor** | Prometheus + Grafana | 응답 품질/지연/에러 실시간 모니터링 |

### 6-2. 인프라 아키텍처

| 계층 | 구성 | 비고 |
|------|------|------|
| **엣지 (베뉴 현장)** | GPU 서버 1~2대 (ASR/TTS 로컬 추론) + 네트워크 장비 | 지연 최소화 |
| **클라우드 (KT Cloud)** | LLM 추론 서버 (GPU) + RAG DB + Session DB + 관리 API | 수평 확장 |
| **CDN/네트워크** | KT 5G + 전용 회선 | VR 스트리밍 대역폭 보장 |
| **관리 웹** | SPA (React) + REST API | 전시 기획사/운영자 대시보드 |

### 6-3. VR 통합 요구사항

### 10-7. 플라이휠 구조

AI 도슨트 엔진이 전시 운영 데이터를 축적할수록 품질이 개선되고, 개선된 품질이 더 많은 전시 계약을 유도하는 선순환 구조:

```
전시 운영 → 관람 데이터 축적 (시선·동선·Q&A·체류시간)
    ↓
데이터 축적 → AI 도슨트 품질 개선
    ↓
품질 개선 → 신규 전시·파트너 유치
    ↓
시간이 지날수록 경쟁자 모방 불가한 해자 형성
```

## 17. KT 연계 전략 (KT Synergy)

### 17-1. 차별화 요소 3가지

1. **AI 도슨트 에이전트** — VR 내 실시간 AI 해설·Q&A·개인화 안내. LLM + 도메인 지식 RAG + 음성 인터랙션. **시장에 존재하지 않는 차별화 기술.**
2. **운영 데이터 플라이휠** — 관람객 행동 데이터(시선·동선·Q&A·체류시간) 축적으로 전시 품질 지속 개선. **시간이 지날수록 경쟁자 모방 불가한 해자.**
3. **KT 인프라 번들** — 5G 네트워크 + 클라우드 렌더링 + 엣지 컴퓨팅. **KT DS 고유 자산으로 타사 모방 불가.**

### 17-2. KT 그룹 시너지

| KT 그룹 자산 | 활용 방안 |
|-------------|-----------|
| **KT 클라우드 / 5G / 네트워크** | VR 렌더링·AI 추론을 위한 클라우드 인프라, 대용량 VR 스트리밍을 위한 5G 네트워크, 저지연 엣지 컴퓨팅 |
| **KT estate / 그룹 고객 네트워크** | 전시장·복합쇼핑몰 등 베뉴 접근 채널, 기업 고객 네트워크를 통한 B2B 영업 |
| **메타버스 개발 인력·경험** | KT 그룹 메타버스 경험에서 축적한 가상공간 구축 노하우와 유경험 인력 전환 활용 |

### 17-3. 기대효과

**정량 효과:**

| 지표 | 효과 |
|------|------|
| 인간 도슨트 비용 절감 | 약 90%+ (인간 도슨트 약 2~3억/6개월 → LLM API 월 200~500만원) |
| 다국어 지원 | 기존 2~3개 → 4개+ (2배+) |
| 관람 데이터 수집 범위 | 전방위 (시선·동선·Q&A·체류시간) |
| 운영 가용성 목표 | 99.5%+ |

**정성 효과:**
- **관람 경험 혁신:** AI 도슨트와의 자연어 대화로 관람객 참여도·만족도 향상
- **Agentic AI 레퍼런스:** 소비자가 직접 체감하는 AI 에이전트 대표 사례 확보
- **AI 자산 축적:** AI 도슨트 엔진은 관광·교육·기업 체험으로 확장 가능한 플랫폼 자산
- **"메타버스" → "AI XR"로 브랜드 전환:** 통신3사 메타버스 실패 이미지를 AI 기반 XR로 차별화
