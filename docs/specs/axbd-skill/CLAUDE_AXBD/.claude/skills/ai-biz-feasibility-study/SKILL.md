# AI 사업 타당성 분석 (AI Business Feasibility Study)

사용자가 제시한 AI 사업 아이템에 대해 **4가지 축**으로 타당성을 분석하세요.

## 분석 프레임워크

### 1. 기술 타당성 (Technical Feasibility)
- **AI 모델 성숙도**: 필요한 AI 기술이 현재 상용 수준인지 (연구 단계 / PoC 가능 / 상용 가능)
- **데이터 확보 가능성**: 학습/추론에 필요한 데이터 소스, 양, 품질 평가
- **인프라 요구사항**: GPU/클라우드/온프레미스 요구 수준
- **기술 리스크**: 모델 정확도, 할루시네이션, 레이턴시 등 핵심 기술 리스크
- **내부 역량 Gap**: 현재 보유 기술 vs 필요 기술 격차

### 2. 시장 타당성 (Market Feasibility)
- **시장 규모**: TAM/SAM/SOM 추정 (Top-down + Bottom-up)
- **고객 Pain Point**: 해결하려는 문제의 심각도와 빈도
- **대안 솔루션**: 현재 고객이 사용하는 대안과 전환 비용
- **시장 타이밍**: 너무 이르거나 늦지 않은지 (Timing Risk)
- **규제 환경**: AI 기본법, 개인정보보호법, 산업별 규제

### 3. 재무 타당성 (Financial Feasibility)
- **초기 투자**: 개발 비용, 인프라 비용, 인력 비용 추정
- **운영 비용**: 월간 추론 비용, 데이터 비용, 유지보수 비용
- **수익 모델**: SaaS / API / 라이선스 / 컨설팅+구축 등
- **BEP 시나리오**: 낙관/기본/비관 3개 시나리오
- **Unit Economics**: LTV, CAC, Gross Margin 추정

### 4. 조직 타당성 (Organizational Feasibility)
- **팀 역량**: AI/ML 엔지니어, 도메인 전문가, PM 확보 현황
- **의사결정 구조**: 사업 승인/투자 의사결정 경로
- **파트너 필요성**: 외부 파트너(기술/데이터/채널) 필요 여부
- **조직 문화 적합성**: 기존 사업과의 시너지 또는 충돌

## 출력 형식

각 축에 대해:
1. **평가 점수** (1~5점)
2. **근거 요약** (2~3문장)
3. **핵심 리스크** (1~2개)
4. **리스크 완화 방안**

최종적으로:
- **종합 점수** (가중 평균)
- **Go / Conditional Go / Hold / Drop** 권고
- **다음 단계 액션 아이템** (3~5개)
