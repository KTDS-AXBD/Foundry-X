2
Git 설치
Install Git for Windows
4단계
▼
1
다운로드
git-scm.com/downloads/win 에서 64-bit Git for Windows Setup을 다운로드합니다.
2
설치 실행
설치 파일을 실행합니다. 설치 옵션은 모두 기본값(Next)으로 진행해도 무방합니다.
💡 참고: 설치 중 "Adjusting your PATH environment" 단계에서 "Git from the command line and also from 3rd-party software" (기본값)가 선택되어 있는지 확인하세요.
3
설치 확인
명령 프롬프트(cmd) 또는 PowerShell을 열고 아래 명령어를 입력합니다.

✓ 확인 포인트: 버전 번호가 출력되면 설치 성공
4
사용자 정보 설정
Git에 본인의 이름과 이메일을 등록합니다. (최초 1회만 설정)
git config --global user.name "홍길동" git config --global user.email "gildong.hong@ktds.co.kr"
✓ 확인 포인트: 오류 메시지 없이 명령이 실행되면 Step 2 완료

Claude Code 설치, 로그인 및 세팅
Claude Code CLI — Install, Login & Configuration
3단계
▼
설치 방법 비교 — Windows 노트북 기준
권장 — PowerShell 네이티브 설치
irm https://claude.ai/install.ps1 | iex
Node.js 설치 불필요, 자동 업데이트 지원, 가장 간편한 방법. 일반 사용자에게 권장합니다.
대안 — WinGet 패키지 설치
winget install Anthropic.ClaudeCode
Windows 패키지 관리자 사용. winget이 이미 설치된 환경에 적합합니다.
💡 npm 방식은 더 이상 권장하지 않습니다. 과거에는 npm install -g @anthropic-ai/claude-code 로 설치했으나, 현재는 네이티브 설치가 공식 권장 방법입니다. Node.js가 필요 없고 자동 업데이트가 지원됩니다. Docker 방식은 지원하지 않습니다.
1
PowerShell을 관리자 권한으로 실행
Windows 검색창에서 PowerShell 을 검색한 뒤, "관리자 권한으로 실행"을 클릭합니다.
💡 방법: 키보드에서 Win 키를 누르고 → "PowerShell" 입력 → 오른쪽에 나타나는 "관리자 권한으로 실행" 클릭
2
Claude Code 설치
아래 명령어를 PowerShell에 복사 → 붙여넣기(Ctrl+V) 한 뒤 Enter를 누릅니다.
irm https://claude.ai/install.ps1 | iex # 설치가 자동으로 진행됩니다 (1~2분 소요)
✓ 확인 포인트: 설치 완료 메시지가 나타나면 성공. PowerShell을 닫았다가 다시 열어주세요 (환경변수 반영을 위해).
3
설치 확인
새로 연 PowerShell에서 버전을 확인합니다.
claude --version # 예시 출력: claude v1.x.x
! 오류 시: 'claude'은(는) 인식할 수 없는 명령 → PowerShell을 닫고 다시 열어보세요
✓ 확인 포인트: claude --version 에서 버전 번호가 출력되면 Step 3 완료. 이어서 Step 4에서 작업 폴더를 연결합니다.
4
작업 폴더(CLAUDE_AXBD) 압축 해제
Extract Project Folder
2단계
▼
CLAUDE_AXBD 폴더란?
팀에서 제공하는 사전 구성된 작업 폴더입니다. 안에 AI 사업개발에 필요한 76개 분석 스킬, 36개 워크플로우, 프로세스 정의서, 팀 컨텍스트가 모두 포함되어 있습니다. 별도 설치 없이 압축만 풀면 바로 사용할 수 있습니다.
1
CLAUDE_AXBD.zip 다운로드
팀 공유 폴더(또는 전달받은 경로)에서 CLAUDE_AXBD.zip 파일을 다운로드합니다.
2
바탕화면에 압축 해제
다운로드한 zip 파일을 바탕화면에 압축 해제합니다.
① CLAUDE_AXBD.zip 파일을 마우스 우클릭
② "압축 풀기..." 또는 "여기에 풀기" 클릭
③ 압축 해제 위치를 바탕화면으로 지정
✓ 결과: 바탕화면에 CLAUDE_AXBD 폴더가 보이면 성공. 폴더 안에 CLAUDE.md, README.md 등이 있는지 확인하세요.
! 주의: 폴더가 CLAUDE_AXBD/CLAUDE_AXBD/ 처럼 이중으로 중첩되지 않았는지 확인하세요. 바탕화면 바로 아래에 CLAUDE_AXBD 폴더가 하나만 있어야 합니다.

Claude Desktop에서 사업 발굴 시작

5단계
▼
1
Claude Desktop 앱에서 Code 탭 진입
Claude Desktop 앱을 실행하고 Code 탭으로 이동합니다.
① Claude Desktop 앱 실행
② 화면 좌측의 Code 탭을 클릭합니다
💡 Code 탭이란? Claude Desktop 앱 안에 내장된 Claude Code 기능입니다. 터미널 없이 앱 안에서 바로 AI 사업개발을 수행할 수 있습니다.
2
CLAUDE_AXBD 폴더 연결
Code 탭에서 작업 폴더를 지정합니다. 이 단계가 가장 중요합니다.
① "Select folder" 버튼 클릭
② 바탕화면의 CLAUDE_AXBD 폴더를 선택
③ Claude가 폴더 안의 CLAUDE.md 를 자동으로 읽어 팀 컨텍스트를 인식합니다
💡 왜 폴더를 연결하나요? 폴더를 연결하면 안에 사전 배치된 76개 분석 스킬, 36개 워크플로우, 프로세스 정의를 Claude가 자동으로 인식합니다. 또한 AI가 생성하는 모든 산출물이 이 폴더의 outputs/ 에 저장됩니다.
3
스킬 인식 확인
대화창에서 / 를 입력하여 스킬이 정상 로딩되었는지 확인합니다.
① 대화 입력창에 / 를 입력
② 자동완성 목록에 스킬이 표시되는지 확인
③ /swot 또는 /ai-biz 를 입력해 필터링 확인
✓ 확인 포인트: /swot-analysis, /ai-biz-moat-analysis 등이 목록에 나타나면 스킬 인식 성공
! 목록이 비어있다면: 폴더가 정상 연결되지 않은 것입니다. Code 탭에서 폴더 경로가 CLAUDE_AXBD 를 가리키고 있는지 다시 확인하세요.
4
사업 발굴 시작하기
대화창에 아래와 같이 입력하면 AI가 자동으로 2-0 아이템 분류 프로세스를 시작합니다.
대화 입력 예시
나는 AX BD팀 홍길동입니다.
"AI 기반 손해사정 자동화"에 대해 2단계 발굴을 시작합니다.

보험사 손해사정 업무의 80%를 AI로 자동화하는 서비스입니다.
참고: https://example.com/관련기사
✓ 성공: Claude가 "이 아이템은 어디서 시작됐나요?" (Turn 1 출발점 파악)를 묻기 시작하면 프로세스가 정상 작동하는 것입니다!

Step 2-0 · 사업 아이템 분류
2-0
사업 아이템 분류
Item Classification Agent
1단계에서 들어온 사업 아이템을 AI Agent와 자연어 대화 3턴으로 5가지 유형(I/M/P/T/S)을 분류하고, 유형별 초기 분석 및 맞춤 분석 경로를 제안한다
AX BD팀 미션 컨텍스트 (AI 사전 인지)
MM→AI 신규사업 전환 · Agentic AI Identity · 반제품화(AI 자산) · FDE 모델(도메인+컨설팅+기술) · SI/ITO 시너지
Turn 1 · 출발점 파악
"이 아이템은 어디서 시작됐나요?
① 기존 서비스/URL ② 시장·산업·고객군 ③ 고객 문제 ④ AI 기술/트렌드 ⑤ 아이디어/방향성"
Turn 2 · 유형별 초기 분석
AI가 유형에 맞게 자동 분석 수행:
I→문제 가설 3개 | M→핵심 Jobs 5개 | P→심각도·빈도 추정 | T→산업별 문제 Top 5 | S→가치사슬+AI 기회
Turn 3 · AX BD 전략 적합성
"① 반제품/AI 자산 재사용? ② Agentic AI 요소? ③ SI/ITO 결합? ④ 추진 방향: 신시장 / AI 전환 / 내부 효율화?"
5가지 유형 분류 결과 → 맞춤 분석 경로 선택
Step 2-1 ~ 2-7 · 유형별 분기 경로
H
