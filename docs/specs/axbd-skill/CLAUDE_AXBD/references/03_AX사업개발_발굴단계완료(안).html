JTBD(Jobs-To-Be-Done) Framework

기존 플랫폼 사용자 vs Fooding 타겟 고객의 핵심 Job 비교

기존 플랫폼 사용자의 Job

When 식자재가 떨어져서 주문해야 할 때

I want to 여러 공급사 가격을 비교하고 편하게 주문하고 싶다

So that 시장에 직접 안 가도 되고 배송비를 아낄 수 있다

Pain: 여전히 "뭘 얼마나 사야 하는지"는 내 감에 의존

Fooding 타겟 고객의 Job

When 매일 저녁 마감 후 내일 식자재를 고민할 때

I want to AI가 최적 수량·가격·공급처를 근거와 함께 추천해주길

So that 새벽 시장을 안 가도 되고, 월 50만원+ 절감이 눈에 보인다

Pain: 가격비교 불가, 수요예측 불가, 원가관리 부재, AI 불신

해외 벤치마크 — 글로벌 AI 식자재 플랫폼

글로벌 트렌드 시사점

글로벌 AI 식자재 플랫폼

  • • Choco(독일, 2018): 1,500 셰프 + 10,000 공급사, AI 주문 자동화
  • • Pepper(미국): 식자재 유통사 디지털 OS, 10만+ 아이템 카탈로그
  • • BlueCart(미국): AI 인보이스 OCR, 자동 발주
  • • ELKASO(UAE): 1,400+ 레스토랑, 300+ 공급사 비교

KTDS 전환 시사점

  • • "구매 의사결정 AI"는 글로벌에서도 아직 초기 — 선점 기회
  • • Choco 모델(레스토랑↔공급사 연결) + AI 의사결정 = Fooding
  • • 해외는 채널/주문 자동화 중심, "왜 이걸 사야 하는지" 설명하는 AI는 없음

핵심 인사이트 — 레퍼런스 분석 결론

  • CJ+식봄 통합으로 시장 재편 중 — 독자 플랫폼 정면 승부 어려운 구도
  • 모든 플랫폼의 AI가 "주문 편의성"에 한정 — "구매 의사결정"을 돕는 AI = 없음
  • 사장이지에 식자재 기능 완전 부재 — "Missing Piece"로 포지셔닝
  • 정부 정책 타이밍 최적 — AI 도우미 50억 + 소상공인 5.4조
  • 소상공인 위기가 수요를 만듦 — 폐업률 > 개업률 역전

Strengths (강점)
  • Ontology/KG 기반 의사결정 기술 PoC 진행 중 (손해사정 → 식자재 응용)
  • KT 사장이지 채널 접근 — 모듈 탑재 가능한 유일한 포지션
  • KT Cloud 보안 인프라 — 소상공인 데이터 보안
  • SI 역량 — 기존 시스템 연동, 커스터마이징
  • 정부사업 수주 경험 — AI 바우처/스마트상점 참여 가능
  • E8 등 Ontology 협력사 네트워크
Weaknesses (약점)
  • 식자재 도메인 경험 없음 — 시장 리서치 출발, 현장 검증 미완
  • 소상공인 B2C 경험 부족 — UX/마케팅/CS 역량 약함
  • 식자재 데이터 미보유 — 시세, 공급사, 품질 데이터 확보 필요
  • MM 기반 수익 구조 관성 — 구독형 전환 조직 내 저항
  • 사장이지 MAU 불확실 — KT 채널 의존 시 효과 제한
Opportunities (기회)
  • 소상공인 AI 의사결정 시장 = 완전 공백
  • 2026 정부 정책 사상 최대 (5.4조 + AI 50억 신규)
  • 음식점 폐업률 사상 최고 → "생존" 위한 절박한 수요
  • CJ+식봄 통합으로 "AI 연동 파트너" 수요 발생
  • Ontology = "설명 가능한 AI" 차별화
Threats (위협)
  • CJ프레시웨이 자체 AI 고도화 (프레시엔 → 의사결정 확장 시 직접 경쟁)
  • 임팩티브AI 소상공인 시장 진출 (82억 시리즈A, 식품 유통 확장 중)
  • 소상공인 디지털 수용도 낮음
  • LLM 비용/성능 변동 → 마진 압박
  • KT 내부 의사결정 지연
