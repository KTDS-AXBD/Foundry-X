# 발굴 산출물 → 사업기획서 섹션 매핑 (가이드 §2 기반)

> **용도:** SKILL.md Step 1(아이템 확인) + Step 3(핵심 정보 수집)에서 참조
> **원칙:** 발굴 단계(2-0~2-8) 산출물을 자동으로 사업기획서 20섹션에 매핑

---

## 1. 매핑 테이블

| 발굴 단계 | 산출물 키워드 | 매핑 섹션 | 매핑 설명 |
|-----------|-------------|----------|----------|
| **2-0** 사업 아이템 구체화·분류 | 아이템 개요, 고객, 도메인 | Hero, Exec Summary, 01-4 | 사업의 기본 정의 + 고객/도메인 선정 근거 |
| **2-1** 레퍼런스 분석 | 경쟁사 분석, 시장 분석, 벤치마크 | 01-8 | 글로벌·국내 동향, 경쟁사 비교 |
| **2-2** 수요·시장 검증 | TAM/SAM/SOM, Pain Point, 시장 규모 | 01-2, 01-3, 01-7 | 문제/영역 근거 + 기술 차별점 + 시장 규모 |
| **2-3** 경쟁·자사 분석 | KT 그룹 시너지, 경쟁 구도, 중복 분석 | 01-8, 04-1, 04-2 | 동향 + 차별화 요소 + 그룹 시너지 |
| **2-4** 사업 아이템 도출 | AI 체크리스트, 아이템 정제 | 01 전체 | 문제 정의 섹션 전반의 기초 자료 |
| **2-5** 핵심 아이템 선정 | KT 방향성, 숏리스트, 수주달성 관점 | 01-1, 02-1, 02-3, 04-1, 04-2, 05-1 | KT 적합성 축 + 사업 개요 + 서비스 구성 + 차별화 + 데이터 |

## 2. 자동 매핑 절차 (Step 1 + Step 3)

### Step 1: 산출물 탐색

발굴 산출물 파일을 아래 순서로 탐색한다:

1. **현재 디렉토리**: 사업기획서 생성 요청이 있는 워킹 디렉토리
2. **프로젝트 docs/**: `docs/` 하위에 발굴 단계별 파일이 있는지 확인
3. **사용자 지정**: AskUserQuestion으로 파일 경로 확인

탐색 키워드 패턴:

| 발굴 단계 | 파일명 패턴 (예시) |
|-----------|-------------------|
| 2-0 | `*2-0*`, `*아이템*구체화*`, `*item*overview*` |
| 2-1 | `*2-1*`, `*레퍼런스*`, `*reference*`, `*경쟁사*` |
| 2-2 | `*2-2*`, `*수요*`, `*시장*검증*`, `*TAM*`, `*pain*point*` |
| 2-3 | `*2-3*`, `*경쟁*자사*`, `*KT*시너지*`, `*competition*` |
| 2-4 | `*2-4*`, `*아이템*도출*`, `*체크리스트*` |
| 2-5 | `*2-5*`, `*핵심*선정*`, `*숏리스트*`, `*shortlist*` |
| 2-6 | `*2-6*`, `*타겟*고객*`, `*페르소나*`, `*persona*` |
| 2-7 | `*2-7*`, `*비즈니스*모델*`, `*BM*`, `*수익*` |
| 2-8 | `*2-8*`, `*패키징*`, `*input*deck*`, `*종합*` |

### Step 3: 섹션별 데이터 매핑

발굴 산출물에서 추출한 데이터를 섹션별로 분배한다:

```
[산출물 파일 읽기]
    │
    ├── 2-0 발견 → Hero 타이틀 + Exec Summary 핵심 + 01-4 고객 근거
    ├── 2-1 발견 → 01-8 경쟁사/벤치마크 카드
    ├── 2-2 발견 → 01-2 트렌드 + 01-3 비교표 + 01-7 TAM/SAM/SOM 카드
    ├── 2-3 발견 → 01-8 동향 추가 + 04-1 차별점 + 04-2 시너지
    ├── 2-4 발견 → 01 전체 보강 (체크리스트 기반)
    ├── 2-5 발견 → 01-1 KT 적합성 축 + 02-1 사업 정의 + 02-3 구성안 + 05-1 데이터
    ├── 2-6 발견 → 01-4 고객 판단 기준 + 03-1 시나리오
    ├── 2-7 발견 → 03-2 R&R + 05-2 매출 시나리오 + 05-3 비용 산출
    └── 2-8 발견 → 전체 섹션 기초 자료 (미매핑 항목 보충)
```

---

## 3. 부족 정보 처리

모든 발굴 단계가 반드시 완료되어 있지는 않다. 누락된 단계의 처리:

| 상황 | 처리 |
|------|------|
| 해당 단계 산출물 없음 | AskUserQuestion으로 핵심 정보만 보충 수집 |
| 산출물 있으나 내용 부족 | 있는 내용 활용 + 부족분 AskUserQuestion |
| 2-8 패키징만 존재 | 2-8에서 전체 섹션 기초 자료 추출 |
| 산출물 전혀 없음 | 사용자 대화 기반으로 처음부터 수집 (기존 v1 방식) |

**우선순위**: 산출물 자동 매핑 > AskUserQuestion 보충 > 스킬이 추론
