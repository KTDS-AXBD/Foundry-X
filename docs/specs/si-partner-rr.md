# Foundry-X SI 파트너 역할 분담 정의서

> **목적**: Foundry-X 고객 파일럿 및 수주 활동에서 AX BD팀(내부)과 SI 파트너(외부)의 역할을 명확히 정의한다.
> **상태**: 초안 (내부 리뷰 후 확정)

---

## 1. 배경

Foundry-X는 PRD v8 기준 "AI 에이전트 오케스트레이션 플랫폼"으로, 6개월 내 1~2건 수주를 목표로 한다. AX BD팀은 1인 풀타임 + AI 에이전트로 개발을 수행하고 있으나, 고객사 대면/커스터마이징/운영 지원에는 외부 파트너 협력이 필수적이다.

---

## 2. 역할 분류

### 2.1 역할 매트릭스

| 역할 | 내부(AX BD) | 외부(SI 파트너) | 비고 |
|------|:----------:|:-------------:|------|
| **아키텍처 결정** | ✅ 주도 | 자문 | 핵심 기술 의사결정은 내부 |
| **코어 기능 개발** | ✅ 주도 | — | 에이전트 오케스트레이션, SDD 등 |
| **API 개발** | ✅ 주도 | 보조 | 에이전트 자동화 활용 |
| **프론트엔드 개발** | ✅ 주도 | 보조 | |
| **Azure 마이그레이션** | ✅ 설계 | ✅ 실행 | 공동 수행, 고객사 인프라 지원 |
| **고객사 커스터마이징** | 설계/감수 | ✅ 주도 | 고객 대면 + 요구사항 반영 |
| **고객사 배포** | 지원 | ✅ 주도 | 고객사 환경 접근은 SI가 담당 |
| **운영/모니터링** | ✅ 인프라 | ✅ 장애대응 | 인프라 관리는 내부, 고객 접점은 공동 |
| **보안 검토** | 설계 | ✅ 실행 | KT DS 보안팀 또는 외부 감사 |
| **사용자 교육** | ✅ 교재 | ✅ 교육 실행 | 교재/가이드는 내부, 교육 진행은 공동 |
| **영업/제안** | ✅ 기술 제안 | ✅ 사업 제안 | 기술 데모는 내부, 사업 제안서는 공동 |

### 2.2 책임 구분 원칙

1. **핵심 기술(Core)은 내부 전담**: 에이전트 오케스트레이션, SDD Triangle, Plumb 엔진 등 Foundry-X의 핵심 기술 자산은 내부에서만 수정
2. **고객 대면(Customer-facing)은 SI 주도**: 고객사 환경 접근, 커스터마이징 구현, 사용자 교육 실행은 SI가 주도
3. **공동 영역(Shared)은 사전 합의**: Azure 마이그레이션, 운영, 보안 등은 작업 전 범위/일정 합의

## 4. 커스터마이징 범위 가이드라인

### 4.1 표준 커스터마이징 (SI가 자체 수행 가능)

| 항목 | 설명 | 난이도 |
|------|------|:------:|
| UI 테마/브랜딩 | 로고, 색상, 폰트 변경 | 낮음 |
| SR 분류 규칙 추가 | SrClassifier에 고객사 SR 유형 추가 | 중간 |
| 워크플로우 커스텀 | SrWorkflowMapper에 고객사 프로세스 반영 | 중간 |
| 대시보드 위젯 배치 | 고객사 KPI 우선순위 반영 | 낮음 |
| 알림 채널 설정 | Slack/이메일/Teams 연동 설정 | 낮음 |

### 4.2 고급 커스터마이징 (내부 지원 필수)

| 항목 | 설명 | 난이도 |
|------|------|:------:|
| 에이전트 신규 역할 추가 | CustomRoleManager 활용 | 높음 |
| DB 스키마 확장 | D1 마이그레이션 추가 | 높음 |
| 인증 체계 변경 | SSO/SAML 연동 | 높음 |
| 외부 시스템 연동 | ITSM/ERP API 연동 | 높음 |

### 4.3 제외 범위 (커스터마이징 불가)

- 에이전트 오케스트레이션 코어 로직 변경
- SDD Triangle / Plumb 엔진 수정
- 멀티테넌시 아키텍처 변경
- 인증/인가 코어 변경
