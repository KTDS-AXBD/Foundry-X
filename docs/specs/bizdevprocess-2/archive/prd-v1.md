# AX BD팀 사업개발 Harness Engineering PRD

**버전:** v1
**날짜:** 2026-03-24
**작성자:** AX BD팀
**상태:** 🔄 검토 중

---

## 1. 요약 (Executive Summary)

**한 줄 정의:**
AX BD팀 담당자가 pm-skills 기반으로 사업 아이템을 체계적으로 분석하고, Discovery 9개 완료기준을 달성하여 PRD를 자동 생성하는 사업개발 Harness Engineering 시스템.

**배경:**
AX BD팀은 사업개발 업무에서 담당자별 분석 방법론이 부재하여, 분석 시간 낭비·산출물 품질 편차·아이템 누락이 반복되고 있다. "뭐하는 팀?" 이라는 인식 문제까지 겹치면서 팀 신뢰도가 위협받고 있다.

**목표:**
Claude Desktop + pm-skills 플러그인을 활용하여 5가지 시작점별 분석 경로를 표준화하고, Discovery 9개 완료기준 체크리스트를 통해 일관된 품질의 PRD를 자동 산출하는 시스템을 구축한다.

---

## 2. 문제 정의

### 2.1 현재 상태 (As-Is)
- 담당자별 분석 방법론이 없어 각자 다른 방식으로 사업 아이템을 분석
- 분석 소요 시간이 담당자마다 상이하고 비효율적
- 산출물 품질이 들쭉날쭉하여 의사결정 신뢰도 저하
- Discovery 완료 기준이 없어 분석 중 아이템 누락 발생
- RNP 500개 Use Case 등 대량 선별이 수동으로 불가능

### 2.2 목표 상태 (To-Be)
- 5가지 시작점에 따라 표준화된 분석 경로가 제공됨
- Discovery 9개 완료기준이 모두 충족되어야 PRD 단계로 진입
- pm-skills 18개 스킬이 각 단계에 매핑되어 실행 가이드 역할
- 일관된 품질의 PRD가 자동 생성됨
- 담당자 경험 수준과 무관하게 동일한 분석 수준 달성

### 2.3 시급성
- **팀 신뢰도**: 분석 방법론이 없는 상태가 지속되면 팀의 존재 가치에 대한 의문 지속
- **수동 유지 한계**: 현재 수동 방식으로는 '26년 연간 목표(Q개 아이템 선제안, 연판 매출 OO억원) 달성 불가능

## 4. 기능 범위

### 4.1 핵심 기능 (Must Have)

| # | 기능 | 설명 | 우선순위 |
|---|------|------|----------|
| 1 | **5가지 시작점 분류** | 사업 아이템의 출발점(아이디어/시장·타겟/고객문제/기술/기존서비스)을 식별하고 해당 분석 경로로 안내 | P0 |
| 2 | **시작점별 분석 경로** | 각 시작점에 맞는 단계별 활동 + pm-skills 스킬 매핑 가이드 제공 | P0 |
| 3 | **Discovery 9개 완료기준 체크리스트** | 문제/고객정의, 시장기회, 경쟁환경, 가치제안, 수익구조, 리스크, 규제/기술, 차별화, 검증실험 — 각 항목의 완료 조건 명시 | P0 |
| 4 | **pm-skills 스킬 실행 가이드** | 18개 스킬을 언제/어떻게 실행하는지 단계별 안내 (초급자도 따라할 수 있는 수준) | P0 |
| 5 | **PRD 자동 생성** | Discovery 9개 기준 충족 시 수집된 분석 결과를 PRD 템플릿에 자동 매핑 | P0 |

### 4.2 부가 기능 (Should Have)

| # | 기능 | 설명 | 우선순위 |
|---|------|------|----------|
| 1 | **다중 AI 검토** | 생성된 PRD를 ChatGPT/Gemini/DeepSeek 등 외부 AI로 자동 검토 (ax-req-interview Phase 2 방식) | P1 |
| 2 | **멀티 페르소나 사전 평가** | BDP-001의 2-9 (8개 KT DS 내부 페르소나 AI 평가) 연동 | P1 |
| 3 | **Six Hats 토론** | PRD에 대한 6모자 관점 토론 자동 수행 | P2 |
| 4 | **진행률 대시보드** | Discovery 9개 기준 달성 현황을 시각적으로 표시 | P2 |

### 4.3 제외 범위 (Out of Scope)
- **Foundry-X 코드 구현**: 본 PRD는 프로세스/가이드 정의만. F-item 실제 개발은 별도 Sprint
- **GTM/마케팅 (5~6단계)**: Discovery~형상화(1~3단계)까지만 범위
- **사내망 연동**: 사외망 환경만 지원. 사내 시스템 직접 연동은 제외
- **맞춤형 AI 모델 학습**: pm-skills 기본 프롬프트 사용. 커스텀 파인튜닝 제외

### 4.4 외부 연동

| 시스템 | 연동 방식 | 필수 여부 |

## 5. 성공 기준

### 5.1 정량 지표 (KPI)

| 지표 | 현재값 | 목표값 | 측정 방법 |
|------|--------|--------|-----------|
| PRD 산출 건수 | 비정기 | 2주에 1개 | PRD 파일 생성 이력 |
| Discovery 9개 커버리지 | 측정 불가 (기준 없음) | 9/9 (100%) | PRD 내 9개 항목 완료 여부 체크 |
| 분석 소요 시간 | 담당자별 상이 | [미확인] | 시작~PRD 생성까지 경과 시간 |

### 5.2 MVP 최소 기준
- [ ] 팀원 1명이 실제 사업 아이템으로 전체 파이프라인(시작점 선택 → 9기준 충족 → PRD 생성)을 완주
- [ ] 생성된 PRD가 팀장 검토를 통과 (Go 또는 Conditional)
- [ ] "쓸만하다" — 사용자 주관 평가 긍정

### 5.3 실패/중단 조건
- 팀원 3명이 각각 시도했으나 전원 "쓸 수 없다" 평가
- Claude Desktop 구독 비용 대비 효과가 수동 대비 개선 없음
- 보안/컴플라이언스 이슈로 사용 자체가 불가능해진 경우

## 6. 제약 조건

### 6.1 일정
- 목표 완료일: 명시적 데드라인 없음 (점진적 개선)
- 마일스톤: MVP (팀원 1명 실사용 검증) 우선 달성

### 6.2 기술 스택
- **플랫폼**: Claude Desktop (Anthropic)
- **플러그인**: pm-skills (phuryn/pm-skills, GitHub Marketplace)
- **보조 도구**: ax-req-interview 검토 스크립트 (Node.js)
- **문서 포맷**: Markdown (PRD, 분석 결과)

### 6.3 인력/예산
- 투입 가능 인원: **1명** (Sinclair, 파트타임 — 다른 업무 병행)
- 예산: Claude Desktop Pro/Max 구독 비용 (팀원 전원 확산 시 N명분)

### 6.4 컴플라이언스
- **KT DS 보안 정책**: 내부 데이터(고객명, 매출 수치, 내부 전략)를 외부 AI에 입력 시 보안 정책 준수 필요
  - 민감 정보 마스킹 또는 추상화 후 입력 원칙
  - 사외망 PC에서만 사용 (사내망 격리)
- **데이터 보존**: Claude Desktop 대화 이력의 데이터 보존/삭제 정책 확인 필요
