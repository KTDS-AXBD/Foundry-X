# 인터뷰 로그 — 사업개발 Harness Engineering

**인터뷰 일시**: 2026-03-24
**인터뷰이**: Sinclair Seo (AX BD팀)
**방식**: ax-req-interview 스킬 (AskUserQuestion 기반)

---

## Part 1: 왜 (목적/문제)

**핵심 문제**: 담당자별 분석 방법론 부재
- 시간 낭비: 담당자마다 분석에 소요되는 시간이 다르고 비효율적
- 품질 편차: 산출물 품질이 담당자별로 들쭉날쭉 → 의사결정 신뢰도 문제
- 아이템 누락: Discovery 완료 기준 없이 분석하다 보니 놓치는 아이템 발생

**시급성**:
- 팀 신뢰도: "뭐하는 팀?" 인식이 개선되지 않으면 존속 위험
- 수동 유지의 한계: 현재 방식으로는 팀 성장에 한계

---

## Part 2: 누구를 위해 (사용자/이해관계자)

**주 사용자**: AX BD팀 담당자
- 기술 수준: 비개발자~중급 혼합 (팀원별 수준 상이)
- 사용 환경: 사외망 PC + Claude Desktop + 모바일 포함

**의사결정 구조**: 다단계 승인
- 팀장 → 본부장 → 임원 순차 승인 (BDP-001 4단계 프로세스)

---

## Part 3: 무엇을 (범위/기능)

**핵심 기능**: 5시작점 → 9완료기준 → PRD 자동 생성까지 전체 파이프라인
- 5가지 시작점별 분석 경로 안내
- Discovery 9개 완료기준 체크리스트
- pm-skills 18개 스킬 매핑 및 실행 가이드
- 완료 시 PRD 자동 생성

**Out-of-scope**:
- 실제 코드 구현 (Foundry-X F-item 개발)
- GTM/마케팅 (5~6단계)

## Part 4: 성공 기준

**KPI**:
- PRD 산출 건수 (목표: 2주에 1개 아이템)
- Discovery 9개 커버리지 (산출된 PRD에서 9개 기준 충족률)

**MVP 기준**: 팀원 1명이 실제 사업 아이템으로 전체 파이프라인을 돌려보고 쓸만하다고 판단
