### 단계별 상세

#### ① KT 연계성 — 100점
- 온누리상품권 SI 프로젝트가 KT DS 수행 → 원천 코드/지식 보유
- 기존 SI 고객사 네트워크 활용 가능 (지자체, 공공기관)
- KT 통신 인프라 연계 (SMS 알림, 본인 인증)

#### ② 사업성 — 85점
- 전자상품권 시장 규모: 2025년 기준 약 8조원 (연 15% 성장)
- 경쟁사 대비 차별점: Spec 기반 빠른 커스터마이징 (발행자별 맞춤)
- 수익 모델: 플랫폼 수수료 0.5~1.5% + 발행 수수료

#### ③ 리스크 수준 — 72점 (주의)
- **규제 리스크**: 전자금융업 등록 필요, 자본금 요건
- **기술 리스크**: 결제 PG사 연동 복잡도
- **시장 리스크**: 카카오페이/네이버페이 등 빅테크 경쟁

#### ④ AI-Ready Data — 90점
- 비즈니스 로직 47개 BL 중 83% 직접 재활용
- 데이터 모델 14개 테이블 중 12개 재활용 (2개 신규)
- API 엔드포인트 8개 중 6개 재활용 (2개 수정)
- 테스트 케이스 24개 중 20개 재활용

#### ⑤ 구체화 수준 — 87점
- 6종 Spec 문서 완비 (BL, 데이터 모델, 기능, 아키텍처, API, 화면)
- Working-version 실동작 코드 존재 (domain 738줄, tests 518줄)
- DB migration 자동 생성 가능
