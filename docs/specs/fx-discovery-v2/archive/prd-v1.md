## 1. 요약 (Executive Summary)

**한 줄 정의:**
AX BD팀 7명을 위해, 발굴(2-0~2-10)→형상화(Phase A~F) 전체 파이프라인을 자동화하여 수동 개입을 최소화하고 E2E 데모가 가능한 상태로 만든다.

**배경:**
F270 1차 정비(Sprint 101)에서 사이드바 투명도, 대시보드 통합, 다음 단계 네비게이션, 404 에러, Agent 스킬 실행 UI 등 6가지 기초 이슈를 해결했다. 그러나 핵심 문제인 **파이프라인 단절**(단계 간 수동 전환, 발굴→형상화 미연결, E2E 테스트 부재)은 여전히 남아 있다.

**목표:**
- 발굴 시작(2-0)부터 형상화 완료(Phase F)까지 자동 파이프라인 동작
- HITL 체크포인트에서만 사용자 개입, 나머지는 자동
- Discovery 위저드 전체 흐름 E2E 테스트로 회귀 방지

## 2. 문제 정의

### 2.1 현재 상태 (As-Is)
- **수동 스킬 실행**: 각 단계(2-0~2-10)의 스킬을 사용자가 하나씩 클릭해서 실행해야 함. 11단계 × 복수 스킬 = 수십 회 수동 클릭
- **발굴→형상화 단절**: 2-10 최종보고서 완료 후 형상화(Stage 3)로의 자동 전환 없음. 사용자가 별도로 형상화 프로세스를 수동 시작해야 함
- **E2E 테스트 부재**: Discovery 위저드, 아이템 상세 페이지, 스킬 실행 흐름에 대한 E2E가 없어 UI 변경 시 회귀 감지 불가
- **현재 E2E 커버리지**: Discovery 관련 테스트 = 기본 네비게이션 수준만 존재

### 2.2 목표 상태 (To-Be)
- 2-0 아이디어 분류 완료 → 2-1~2-10 자동 순차 실행 (HITL 체크포인트에서만 정지)
- 2-10 최종보고서 완료 → 형상화 Phase A~F 자동 트리거
- Discovery 위저드 E2E 테스트 커버리지 확보 (위저드 흐름 + 상세 페이지 + 스킬 실행)

### 2.3 시급성
- 발굴만 완료되고 형상화로 이어지지 않으면 BD 프로세스 전체가 미완성
- 팀원들이 수동 반복 작업에 시간을 소비하면 도구 채택률 저하
- F270 1차 정비 직후가 후속 개선의 최적 타이밍 (코드 컨텍스트가 신선)

## 4. 기능 범위

### 4.1 핵심 기능 (Must Have)

| # | 기능 | 설명 | 우선순위 |
|---|------|------|----------|
| 1 | **형상화 자동 전환** | 2-10 최종보고서 완료 시 형상화 Phase A~F 자동 파이프라인 트리거. 발굴 산출물을 형상화 입력으로 전달 | P0 |
| 2 | **형상화 Phase A~F 자동 실행** | Phase A(입력점검) → B(req-interview) → C(O-G-D 형상화 루프) → D(교차검토+Six Hats) → E(전문가 5종 리뷰) → F(HITL 최종게이트) 순차 자동 실행 | P0 |
| 3 | **발굴 연속 스킬 파이프라인** | 2-0 완료 → 2-1 → ... → 2-10 자동 순차 실행. 각 단계 스킬 완료 시 다음 단계 자동 추천 및 실행 | P1 |
| 4 | **HITL 체크포인트** | 사업성 체크포인트(2-1~2-7 질문, 2-5 Commit Gate) 및 형상화 Phase 전환 시 사용자 승인 UI | P1 |
| 5 | **Discovery E2E 테스트** | 위저드 전체 흐름 + 아이템 상세 에러 처리 + 스킬 실행 + HITL 리뷰 E2E | P2 |

### 4.2 부가 기능 (Should Have)

| # | 기능 | 설명 | 우선순위 |
|---|------|------|----------|
| 1 | 파이프라인 진행 모니터링 | 현재 어떤 단계가 실행 중인지, 어디서 정지했는지 대시보드에 시각화 | P2 |
| 2 | 실패 복구 UI | 스킬 실행 실패 시 재시도/건너뛰기 옵션 + 에러 상세 | P2 |
| 3 | 파이프라인 히스토리 | 완료된 파이프라인 실행 이력 (소요 시간, 통과/실패 단계) | P3 |

### 4.3 제외 범위 (Out of Scope)
- **새로운 스킬 개발**: 기존 68개 스킬을 활용하며, 새 스킬 추가는 이번 범위 아님
- **외부 AI 모델 연동 변경**: 현재 Anthropic/OpenRouter 구성 그대로 유지
- **모바일 최적화**: PC 우선, 모바일은 기존 수준 유지
- **실시간 협업**: 여러 사용자가 동시에 같은 파이프라인을 조작하는 기능

### 4.4 외부 연동

| 시스템 | 연동 방식 | 필수 여부 |
|--------|-----------|-----------|
| ax-bd-shaping 스킬 | CLI subprocess + API | 필수 |
| BdSkillExecutor 서비스 | REST API (`POST /ax-bd/skills/:skillId/execute`) | 필수 |
| ax-bd-discovery 스킬 | CLI subprocess | 필수 |
| D1 Database | SQL (파이프라인 상태 저장) | 필수 |
| O-G-D Agent Loop | API + CLI 에이전트 | 필수 (형상화 Phase C) |

### 5.2 MVP 최소 기준
- [ ] 2-10 완료 시 형상화 Phase A 자동 트리거 동작

- [ ] HITL 체크포인트에서 사용자 승인/거부 UI 동작
- [ ] Discovery 위저드 E2E 테스트 5건 이상 통과

### 5.3 실패/중단 조건

- 목표 완료: **5 Sprint 이내** (Sprint 132~136 예상)

- 에이전트: `.claude/agents/` (ogd-*, shaping-*, six-hats-*, expert-*)
