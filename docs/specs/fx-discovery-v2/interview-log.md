## Part 1: 왜 (목적/문제)

**Q: 현재 발굴 프로세스에서 가장 크게 느끼는 문제는?**

A: 3가지 문제가 모두 심각함:
1. 수동 스킬 실행이 너무 번거로움 (단계별 사용자 직접 클릭)
2. 발굴→형상화 전환이 단절됨 (2단계 완료 후 3단계로 자동 전환 없음)
3. 테스트 부재로 회귀 리스크 (Discovery 위저드 E2E 없음)

**Q: 가장 시급한 것?**

A: **형상화 연동이 가장 시급**. 발굴만 되고 형상화가 연결 안 되면 프로세스 전체가 미완성.

**우선순위:** P0 형상화 연동 > P1 Agent 파이프라인 > P2 E2E 테스트

## Part 3: 무엇을 (범위/기능)

**Q: 형상화 연동의 자동 전환 레벨?**

A: **자동 파이프라인 (Phase A~F)**. 2-10 완료 → Phase A 입력점검 → ... → Phase F까지 자동 실행.

**Q: Agent 오케스트레이션의 연속 스킬 실행 범위?**

A: **HITL 체크포인트 포함 파이프라인**. 2-0~2-10 자동 실행하되 사업성 체크포인트(2-5 Commit Gate 등)에서 사용자 확인 필수.

## Part 4: 성공 기준

**Q: 성공했다는 걸 어떻게 알 수 있나?**

A: 2가지 기준:
1. **E2E 데모 동작 여부** — 발굴 시작부터 형상화 완료까지 자동 파이프라인 동작
2. **E2E 테스트 통과율** — Discovery 관련 E2E 테스트 커버리지 확보
