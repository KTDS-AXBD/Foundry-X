## 프로젝트 이름/코드네임

**Skill Evolution** — Foundry-X Phase 10의 OpenSpace Self-Evolving 개념 내재화 프로젝트.
단, O-G-D Agent Loop(F270~F273)은 별도 PRD로 이미 등록됨. 이번 PRD는 나머지 Track(A/C/D/E)을 다룸.

---

## Part 1: 왜 (목적/문제)

**Q: 지금 어떤 문제가 있어서, 또는 어떤 기회를 잡으려고 이걸 만들려고 하시나요?**

참조 문서에서 파악된 핵심 문제:
- BD 스킬이 정적 파일로 수동 관리됨
- 실행 결과에서 자동 진화하지 않음
- 스킬 성공/실패 메트릭이 수집되지 않음
- 에이전트 간 스킬 공유·검색 메커니즘 부재
- 토큰 효율성을 측정하는 경제적 벤치마크 없음

**Q: 이 문제가 지금 어느 정도로 심각한가요?**

**높음 — 반복 작업 많음.** 같은 스킬 수정을 여러 번 반복, 성공/실패 패턴이 유실되고 있음. 토큰 비용도 최적화 여지 없이 소모 중.

## Part 2: 누구를 위해 (사용자/이해관계자)

**우선 1인 사용 후 확대.** Sinclair가 먼저 단독 사용하며 검증, 이후 AX BD팀 7명 전체로 확대.

---

## Part 3: 무엇을 (범위/기능)

**Q: 핵심 기능을 딱 한 문장으로?**

BD 스킬 실행 메트릭을 자동 수집하고, 성공 패턴을 자동 추출하며, 팀 내 스킬을 공유·검색하고, 경제적 ROI를 벤치마크하는 시스템.

**의사결정 사항:**

| 항목 | 결정 |
|------|------|
| 메트릭 저장소 (Track A) | D1 테이블 + 기존 F143/F158~F161 인프라 활용 |
| DERIVED 범위 (Track C) | BD 7단계 전체 (발굴→검증→제안→수주→수행→완료→회고) |
| 스킬 레지스트리 (Track D) | ax-marketplace 플러그인 기반 확장 (F152 대신) |
| O-G-D 관계 | 별도 PRD(F270~F273). Track C의 DERIVED는 O-G-D 루프와 연동 |

---

## Part 4: 어떻게 판단할 것인가 (성공 기준)

**Q: 이게 성공했다는 걸 어떻게 알 수 있을까요?**

**복합 지표:**
- 토큰 절감률 (Cold Start vs Warm Run 비교)
- 스킬 재사용률 (자동 생성 스킬의 실제 재사용 횟수)
- 품질 점수 (사업성 신호등 연계)
- BD_ROI = (스킬로 생성한 산출물 가치) / (소비 토큰 비용)
