## 3. 온보딩 시나리오 5종

### S1: 신규 사용자 가입 + 로그인

| 항목 | 내용 |
|------|------|
| **사전 조건** | fx.minu.best 접근 가능 |
| **절차** | 1. 랜딩 페이지 접근 → 2. "시작하기" 클릭 → 3. 이메일/비밀번호 입력 → 4. 계정 생성 → 5. 대시보드 진입 |
| **기대 결과** | 대시보드가 정상 렌더링, 사이드바 네비게이션 동작 |
| **KPI 이벤트** | `user_signup`, `page_view(dashboard)` |
| **예상 소요** | 2분 |

### S2: 첫 프로젝트 생성 + 에이전트 실행

| 항목 | 내용 |
|------|------|
| **사전 조건** | S1 완료 (로그인 상태) |
| **절차** | 1. 프로젝트 목록 → 2. "새 프로젝트" → 3. Git 리포 URL 입력 → 4. 프로젝트 생성 → 5. 에이전트 태스크 생성 → 6. 실행 결과 확인 |
| **기대 결과** | 프로젝트 카드에 건강도 표시, 에이전트 태스크 목록에 결과 |
| **KPI 이벤트** | `project_created`, `agent_task_created`, `agent_task_completed` |
| **예상 소요** | 5분 |

### S3: 온보딩 가이드 완주

| 항목 | 내용 |
|------|------|
| **사전 조건** | S1 완료 |
| **절차** | 1. /getting-started 접근 → 2. 기능카드 5개 탐색 → 3. FAQ 확인 → 4. 체크리스트 항목 완료 |
| **기대 결과** | 체크리스트 진행률 100%, KPI 이벤트 기록 |
| **KPI 이벤트** | `onboarding_step_completed` × N, `onboarding_completed` |
| **예상 소요** | 5분 |

### S4: KPI 대시보드 확인

| 항목 | 내용 |
|------|------|
| **사전 조건** | S2 완료 + 데이터 축적 (1일+) |
| **절차** | 1. /dashboard 접근 → 2. KPI 위젯 확인 (K7 WAU, K8 에이전트 완료율, K9 서비스 전환율) → 3. 수치 의미 이해 |
| **기대 결과** | KPI 위젯에 실데이터 표시 (0이 아닌 값) |
| **KPI 이벤트** | `page_view(dashboard)` |
| **예상 소요** | 2분 |

### S5: 피드백 제출

| 항목 | 내용 |
|------|------|
| **사전 조건** | S3 완료 |
| **절차** | 1. /getting-started NPS 폼 → 2. 점수 선택 (0~10) → 3. 코멘트 입력 → 4. 제출 |
| **기대 결과** | "제출 완료" 확인, 관리자가 GET /api/onboarding/feedback으로 조회 가능 |
| **KPI 이벤트** | `nps_submitted` |
| **예상 소요** | 2분 |

## 5. 판정 기준

### Phase 4 Final Go 조건 (PRD v5 기준)

| 조건 | 목표 | 측정 방법 |
|------|------|-----------|
| NPS 6+ (5명 대상) | 평균 6점 이상 | GET /api/onboarding/feedback 집계 |
| WAU 5명+ (K7) | 주간 활성 5명 | KPI 대시보드 |
| 에이전트 완료율 70%+ (K8) | 자동 완료 70% | agent_tasks 통계 |
| "기존 방식 복귀" 0건 (K5) | 0건 | 주간 인터뷰 |

### Kill 신호

- 전환 후 2주 내 이전 서비스 사용률 > 30%
- 지원 채널 문의 주 50건+
- NPS 3점 미만
