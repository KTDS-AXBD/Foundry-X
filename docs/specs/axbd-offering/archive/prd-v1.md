## 1. 요약 (Executive Summary)

**한 줄 정의:**
AX BD팀 전원이 발굴 산출물(2-0~2-8)을 기반으로 KT 연계 사업기획서를 표준 품질로 자동 생성할 수 있는 Claude Code 스킬 v2.

**배경:**
현재 offering-html 스킬(v1)은 담당자별 품질 차이가 크고, KT 연계 원칙 누락·교차검증 형식화 등의 문제로 임원 보고 때마다 목차/포맷/톤 수정이 반복됨. 표준 가이드(`AX BD팀 사업기획서 생성 스킬.md`)가 정립되었으나, 스킬에 아직 반영되지 않은 상태.

**목표:**
가이드의 표준 목차(18섹션) + 디자인 시스템 + 작성 원칙을 스킬에 내장하여, 누가 실행해도 동일 품질의 사업기획서가 생성되도록 함.

---

## 2. 문제 정의

### 2.1 현재 상태 (As-Is)
- offering-html v1은 표준 가이드 반영 전 버전 → 목차/톤/디자인이 가이드와 불일치
- 담당자별 사업기획서 품질 편차가 큼 (경영 언어 원칙, KT 연계 원칙 등 누락 빈번)
- 임원 보고 시 목차/포맷/톤 지적 → 수정 사이클 반복 (1건당 2~3회 수정)
- 교차검증(05-4) 섹션이 추진론 편향으로 작성되는 경향

### 2.2 목표 상태 (To-Be)
- 가이드의 필수 섹션 20개 + 디자인 시스템(컬러/타이포/레이아웃/컴포넌트) 100% 준수
- 경영 언어 원칙, KT 연계 원칙, 고객 유형별 톤 조정이 스킬 로직에 내장
- GAN 기반 교차검증이 냉철하게 자동 생성 (No-Go 판정도 가능한 수준)
- 임원 보고 시 포맷/톤 수정 지적 0건 목표

### 2.3 시급성
- 다음 보고/공유회 임박 — 즉시 필요
- 팀 전원이 동일 품질로 기획서를 작성해야 하는 상황

## 6. 제약 조건

### 6.1 일정
- 목표 완료일: 즉시 (보고/공유회 임박)
- 마일스톤:
  - M1: 표준 목차 + 디자인 시스템 적용 (MVP)
  - M2: GAN 교차검증 + 경영 언어 원칙
  - M3: PPTX + 버전 이력 + 피드백 자동화

### 6.2 기술 스택
- 구현 방식: Claude Code 스킬 (`.claude/skills/ax-bd/shape/offering-html/SKILL.md`)
- 기존 스킬 v1 개선 방식
- HTML 단일 파일 출력 (외부 의존 없음)
- 에이전트: ax-bd-offering-agent 활용

### 6.3 인력/예산
- AI 에이전트 기반 개발 (별도 인력 투입 없음)
- API 비용: OpenRouter 경유 교차검증 시 라운드당 약 $0.01~0.05

### 6.4 컴플라이언스
- KT DS 내부 보안 정책 준수 (시크릿 하드코딩 금지)
- 사업기획서 내 고객 정보는 파일 기반 로컬 관리 (외부 전송 없음)
