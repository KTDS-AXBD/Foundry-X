## 1. Overview

### 1.1 Purpose

Foundry-X 프로젝트의 첫 번째 ADR로, PRD v4가 이전 모든 기획 문서를 대체한다는 공식 결정을 기록해요.

ADR-000은 프로젝트의 "기원 선언" 역할이에요:
- Discovery-X → AI Foundry → Foundry-X 네이밍 변천 공식화
- 멀티리포 → 모노리포 전환 결정 기록
- PRD v1~v3을 archived 상태로 확정

### 1.2 Background

- PRD가 4차례 진화 (v1: 인터뷰 초안 → v2: 구체화 → v3: AI 검토 반영 → v4: synthnoosh harness bootstrap 반영)
- v1~v3은 `docs/archive/`에 이미 이동되어 있으나, 공식 선언 문서는 부재
- CLAUDE.md Sprint 1 Checklist에 "ADR-000: v3가 기존 멀티리포/AI Foundry 문서를 대체한다는 선언"으로 명시

### 1.3 Related Documents

- 현행 PRD: `docs/specs/prd-v4.md`
- 아카이브: `docs/archive/prd-v1.md`, `prd-v2.md`, `prd-v3.md`
- PRD v4 §3: Discovery-X → AI Foundry → Foundry-X 변천 기록
- 인터뷰 로그: `docs/specs/interview-log.md`
- AI 검토: `docs/review/round-1/`, `docs/review/round-2/`

## 6. Architecture Considerations

### 6.1 ADR 저장 위치

| Option | 장점 | 단점 | 선택 |
|--------|------|------|:----:|
| `docs/adr/` | ADR 전용 디렉토리, 명확한 분리 | 디렉토리 추가 | ✅ |
| `docs/specs/` | 기존 스펙 문서와 함께 | ADR과 스펙 혼재 | ☐ |

### 6.2 ADR 형식

Michael Nygard의 경량 ADR 형식 채택:
- **Title**: 넘버링 + 제목
- **Status**: Accepted / Deprecated / Superseded
- **Context**: 결정 배경
- **Decision**: 결정 내용
- **Consequences**: 결정 결과 (긍정/부정)
