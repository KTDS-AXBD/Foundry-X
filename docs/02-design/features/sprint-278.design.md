# Sprint 278 Design — F524 E2E 자동 추출 + F525 Gap-E2E 통합 점수

## 1. 목표

Design 문서 §4+§5에서 Playwright E2E 시나리오를 자동 추출하고,
Gap Analysis 보고서에 E2E 결과를 통합하여 종합 품질 점수(Composite Score)를 산출한다.

## 2. 아키텍처 결정

### E2E 자동 추출 파이프라인

```
Design 문서 (sprint-N.design.md)
    ↓ parseDesignDocument()
ScenarioList[]          ← §5 파일 매핑 (라우트/컴포넌트 추출)
    ↓                   ← §4 기능 명세 (시나리오명 추출)
generateE2ESpec()
    ↓
packages/web/e2e/generated/sprint-{N}.spec.ts
```

### Composite Score 계산

```
Composite = Gap×0.6 + E2E×0.4

- Gap: Design↔Code 파일 일치율 (기존 Gap Analysis)
- E2E: PASS/(PASS+FAIL) × 100 (SKIP 제외)
- E2E 데이터 없으면: Composite = Gap (하위 호환)
```

### 위치 결정: `packages/cli/src/services/`

**근거**: 기존 `health-score.ts`, `harness-freshness.ts` 패턴과 동일한 위치.
CLI 커맨드에서 직접 호출하거나 향후 autopilot에서 import 가능.
별도 패키지 분리는 F526(Sprint 279) 이후 고려.

```typescript
export interface Scenario {
  name: string;          // 시나리오명 (§4 행 제목 또는 §5 파일명에서)
  route: string;         // 대상 URL 경로 (예: /dashboard, /discovery)
  selector?: string;     // 주요 확인 selector (예: [data-testid="..."])
  actions?: string[];    // 추가 액션 설명 (텍스트 only)
  source: 'section4' | 'section5';  // 추출 출처
}

export interface ParseResult {
  scenarios: Scenario[];
  parsedSectionCount: number;   // 파싱 성공한 섹션 수
  totalSectionCount: number;    // 전체 §4+§5 섹션 수
  confidenceRate: number;       // parsedSectionCount / totalSectionCount
}

export interface GenerateResult {
  filePath: string;     // 생성된 spec 파일 경로
  scenarioCount: number;
  content: string;      // spec 파일 내용
}
```

## 4. F525: Gap-E2E 통합 점수 설계

### GapScorer 타입 정의

```typescript
// packages/cli/src/services/gap-scorer.ts

export interface E2EResult {
  specFile: string;     // e2e spec 파일 경로
  total: number;
  pass: number;
  fail: number;
  skip: number;
}

export interface CompositeInput {
  gapRate: number;       // 0~100 (기존 Gap Analysis Match Rate)
  e2eResults?: E2EResult[];  // undefined = E2E 없음 (하위 호환)
}

export interface CompositeScore {
  gapRate: number;
  e2ePassRate: number | null;   // null = E2E 미실행
  compositeRate: number;        // 최종 종합 점수
  formula: string;              // 계산식 표시용
  status: 'PASS' | 'FAIL';      // ≥90% = PASS
}
```

### 계산 로직

```typescript
function computeCompositeScore(input: CompositeInput): CompositeScore {
  if (!input.e2eResults || input.e2eResults.length === 0) {
    // 하위 호환: E2E 없으면 Gap = Composite
    return {
      gapRate: input.gapRate,
      e2ePassRate: null,
      compositeRate: input.gapRate,
      formula: `Gap ${input.gapRate}% (E2E 미실행)`,
      status: input.gapRate >= 90 ? 'PASS' : 'FAIL',
    };
  }
  
  const totalTests = input.e2eResults.reduce((s, r) => s + r.pass + r.fail, 0);
  const totalPass = input.e2eResults.reduce((s, r) => s + r.pass, 0);
  const e2ePassRate = totalTests === 0 ? 100 : (totalPass / totalTests) * 100;
  
  const compositeRate = input.gapRate * 0.6 + e2ePassRate * 0.4;
  
  return {
    gapRate: input.gapRate,
    e2ePassRate,
    compositeRate,
    formula: `Gap ${input.gapRate}%×0.6 + E2E ${e2ePassRate.toFixed(1)}%×0.4`,
    status: compositeRate >= 90 ? 'PASS' : 'FAIL',
  };
}
```

### Gap Analysis 보고서 확장 포맷

```markdown

| spec | total | pass | fail | skip |
|------|-------|------|------|------|
| generated/sprint-278.spec.ts | 5 | 5 | 0 | 0 |

## Composite Score
Gap Score: 94% × 0.6 = 56.4

**Composite: 96.4% — PASS**
```
