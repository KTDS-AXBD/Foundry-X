## 1. Overview

### 1.1 Design Goals

- ADR-000을 1페이지 이내 간결하게 유지
- 상세 히스토리는 기존 문서 링크로 대체
- 향후 ADR 작성 규칙도 함께 수립

### 1.2 Design Principles

- **간결성**: 결정 사항만 명시, 배경 상세는 링크로 대체
- **자기 완결성**: ADR만 읽어도 핵심 결정 3가지 파악 가능
- **확장성**: docs/adr/ 디렉토리 + INDEX.md로 향후 ADR 추가 용이

## 3. ADR-000 내용 설계

### 3.1 형식

Michael Nygard 경량 ADR 형식 (Title → Status → Context → Decision → Consequences).

### 3.2 내용 아웃라인

```markdown
# ADR-000: PRD v4를 유일한 현행 명세로 확정

**Status**: Accepted
**Date**: 2026-03-16
**Author**: AX BD팀

## Context

- Foundry-X는 Discovery-X → AI Foundry → Foundry-X로 진화
- PRD가 v1(인터뷰 초안) → v2(구체화) → v3(AI 검토) → v4(harness bootstrap) 4회 개정
- 다중 AI 검토 2라운드 통과 후 착수 판정 완료
- 현행 권위 문서에 대한 공식 선언 부재

## Decision

### D1. PRD v4 = 유일한 현행 명세
- `docs/specs/prd-v4.md`가 Foundry-X의 유일한 PRD
- 이전 버전(v1~v3)은 `docs/archive/`에 보존하되 참조 전용

### D2. 모노리포 구조 확정
- TypeScript(CLI) + Python(Plumb) 공존 모노리포
- pnpm workspace + Turborepo
- 근거: PRD v3 AI 검토 — Gemini, ChatGPT 권고

### D3. 프로젝트 네이밍 공식화
- Discovery-X(탐색) + AI Foundry(구축) = Foundry-X(통합 협업)
- "Foundry-X"가 유일한 공식 프로젝트명

## Consequences

### 긍정
- 신규 팀원/에이전트가 읽어야 할 문서가 명확 (prd-v4.md 하나)
- SSOT 원칙 강제: "Git이 진실" 철학의 문서 레벨 적용
- archive에서 진화 과정 추적 가능 (투명성 유지)

### 부정
- v1~v3의 유용한 논의가 archive 접근 없이는 보이지 않음
  → Mitigation: ADR Context에 핵심 참조 링크 포함
- ADR 형식이 팀 내 아직 익숙하지 않음
  → Mitigation: INDEX.md에 작성 가이드 포함

## References
- PRD v4: docs/specs/prd-v4.md
- 인터뷰 로그: docs/specs/interview-log.md
- AI 검토 Round 1: docs/review/round-1/
- AI 검토 Round 2: docs/review/round-2/
- Archive: docs/archive/prd-v1.md, prd-v2.md, prd-v3.md
```
