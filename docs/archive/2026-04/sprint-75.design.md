# Sprint 75 Design — F220 Brownfield-first Init + F222 Changes Directory

## 1. 개요

### F220: Brownfield-first Init 강화
`foundry-x init` Brownfield 모드를 확장하여 기존 코드베이스의 문서·구조·스펙을 깊이 스캔하고, `project-context.md`를 자동 생성한다.

### F222: Structured Changes Directory
OpenSpec `changes/` 패턴을 도입하여 변경 단위별 proposal/design/tasks/spec-delta를 묶어 관리하고, `foundry-x sync` 시 Δspec을 자동 감지한다.

## 2. 타입 설계

### 2.2 Changes Directory 타입 (packages/shared/src/types.ts)

```typescript
/** 변경 상태 */
export type ChangeStatus = 'proposed' | 'approved' | 'implemented' | 'rejected';

/** 단일 변경 항목 */
export interface ChangeEntry {
  id: string;                  // 디렉토리명 (e.g., '001-add-auth')
  status: ChangeStatus;
  hasProposal: boolean;
  hasDesign: boolean;
  hasTasks: boolean;
  hasSpecDelta: boolean;
  specDelta?: SpecDelta;
}

/** 스펙 변경분 */
export interface SpecDelta {
  added: string[];             // 추가된 스펙 항목
  modified: string[];          // 변경된 스펙 항목
  removed: string[];           // 제거된 스펙 항목
}

/** changes/ 인덱스 */
export interface ChangesIndex {
  version: string;
  changes: ChangeEntry[];
}

/** Sync 결과에 changes 정보 추가 */
export interface SyncRunResult {
  triangle: SyncResult['triangle'];
  decisions: SyncResult['decisions'];
  healthScore: HealthScore;
  syncResult: SyncResult;
  /** F222: 변경 디렉토리 스캔 결과 */
  changes?: ChangeEntry[];
}
```

## 3. F220 상세 설계

### 3.1 discover.ts 확장

기존 `discoverStack()` 은 유지하고, 2개 함수를 추가한다:

```typescript
// packages/cli/src/harness/discover.ts

/** 기존 문서 파일 탐지 */
export async function discoverDocs(cwd: string): Promise<DocFile[]>

/** 주요 디렉토리 구조 분석 (깊이 2) */
export async function discoverDirectoryStructure(cwd: string): Promise<DirNode[]>

/** 프로젝트 컨텍스트 생성 */

  cwd: string,
  docs: DocFile[],
  dirs: DirNode[],
  profile: RepoProfile,

```

**discoverDocs 탐지 대상:**

| 패턴 | 타입 |
|------|------|
| `README.md`, `README` | readme |
| `CHANGELOG.md`, `CHANGES.md` | changelog |
| `docs/adr/`, `adr/`, `ADR-*.md` | adr |
| `SPEC.md`, `*.spec.md` | spec |
| `docs/`, `documentation/` | docs |
| `CONTRIBUTING.md` | contributing |

**discoverDirectoryStructure 역할 추론:**

| 디렉토리명 패턴 | 역할 |
|------------------|------|
| `src/routes/`, `routes/`, `api/` | routes |
| `src/services/`, `services/`, `lib/` | services |
| `src/components/`, `components/` | components |
| `test/`, `tests/`, `__tests__/`, `*.test.*` | tests |

## 4. F222 상세 설계

### 4.1 changes-parser.ts (신규)

```typescript
// packages/cli/src/harness/changes-parser.ts

import type { ChangeEntry, SpecDelta } from '@foundry-x/shared';

/** .foundry-x/changes/ 디렉토리를 파싱하여 변경 목록 반환 */
export async function parseChanges(cwd: string): Promise<ChangeEntry[]>

/** 개별 변경의 spec-delta.md를 파싱하여 SpecDelta 추출 */

```

**changes/ 디렉토리 규약:**
```
.foundry-x/changes/
  ├── {change-id}/
  │   ├── proposal.md      # 변경 제안 (what & why)
  │   ├── design.md         # 설계 (how)
  │   ├── tasks.md          # 작업 항목
  │   └── spec-delta.md     # Δspec — Added/Modified/Removed 섹션
  └── index.json            # 선택적 인덱스
```

**spec-delta.md 형식:**
```markdown
# Spec Delta

## Added
- New user authentication endpoint
- Session management service

## Modified
- Updated RepoProfile type with new fields

## Removed
- Deprecated legacy sync method
```

### 4.2 changes-scanner.ts (신규)

```typescript
// packages/cli/src/harness/changes-scanner.ts

import type { ChangeEntry } from '@foundry-x/shared';

/** changes/ 디렉토리 전체 스캔 + 상태 판별 */
export async function scanChanges(cwd: string): Promise<ChangeEntry[]>
```

**상태 판별 로직:**
- `proposal.md`만 있으면 → `proposed`
- `design.md`도 있으면 → `approved`
- `tasks.md`에 모든 항목 체크되면 → `implemented`
- `proposal.md`에 "REJECTED" 마커 → `rejected`
