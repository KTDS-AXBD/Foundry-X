## 1. 개요

Prototype Builder의 3가지 핵심 한계를 병렬로 해결한다:
- **트랙 A** (디자인 품질): impeccable 디자인 스킬 시스템 통합
- **트랙 B** (판별 고도화): LLM 기반 5차원 판별 + Vision API
- **트랙 C** (비용 최적화): max-cli 본격 통합

성공 기준: 생성된 프로토타입을 추가 수정 없이 고객 데모 가능.

## 2. 선행 조건 (M0 PoC — ✅ 2026-04-07 완료)

> 상세: `docs/specs/fx-builder-v2/poc-results.md`

| # | 검증 항목 | 결과 | 통과 기준 | 판정 |
|---|----------|------|----------|:----:|
| 1 | Vision API 비용 | **$0.0096/회** (72KB 스크린샷) | < $0.50 | **PASS** |
| 2 | API rate limit | **10회 연속 성공**, 평균 2.0초 | 일 10회+ | **PASS** |
| 3 | impeccable 토큰 | **~8K 토큰** (7도메인 전체, 30K의 27%) | 시스템 프롬프트 내 수용 | **PASS** |
| 4 | LLM 판별 재현성 | **표준편차 0.0** (5회 반복, temp=0) | 표준편차 < 10점 | **PASS** |

**M0 판정: GO** — 4건 모두 통과. 당초 우려했던 리스크 대부분 해소.

**PoC 반영 설계 변경:**
- impeccable 4도메인 축소 계획 → **7도메인 전체 적용으로 확대** (토큰 여유 충분)
- Vision API 선택적 기능 계획 → **ui 차원 주평가로 적극 활용** ($0.01/회로 경제적)
- LLM 판별 재현성 보강 계획 → **temperature=0 + structured JSON으로 충분** (추가 조치 불필요)
- max-cli CLI 인증 → 별도 세션에서 `claude auth login` 수행 필요 (API fallback 동작 확인 완료)
