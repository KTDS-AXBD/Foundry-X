## specs/ 디렉토리 구조 가이드

specs/ 하위 18개 디렉토리는 **PRD 프로젝트별 독립 폴더** 구조예요.
각 PRD 프로젝트는 `/ax:req-interview` 스킬이 자동 생성한 하위 구조를 포함해요.

```
specs/{project}/
├── prd-final.md          # 확정 PRD (frontmatter 있음)
├── interview-log.md      # 인터뷰 로그
├── review-history.md     # 리뷰 이력 요약
├── review/round-N/       # AI 다중 검토 (scorecard, feedback, apply-diff)
└── archive/              # 이전 PRD 버전 + 리뷰
```

| 폴더 | PRD 프로젝트 | 상태 |
|------|-------------|:----:|
| [agent-evolution/](specs/agent-evolution/) | Agent Evolution (Track A/B) | ✅ 완료 |
| [ax-bd-atoz/](specs/ax-bd-atoz/) | AX BD A-to-Z 7단계 | ✅ 확정 |
| [fx-bd-v1/](specs/fx-bd-v1/) | BD Pipeline E2E 통합 | ✅ 착수 |
| [fx-discovery-ux/](specs/fx-discovery-ux/) | 발굴 UX 개선 (F263~F266) | ✅ 착수 |
| [openclaw/](specs/openclaw/) | OpenClaw MVP | Draft |
| [bizdevprocess-2/](specs/bizdevprocess-2/) | 사업개발 Harness Engineering | ✅ Go |
| [FX-PLAN-012/](specs/FX-PLAN-012/) | Phase 6 Ecosystem Integration | ✅ 완료 |

> **참고 — GOV-001 frontmatter 비대상 명시** (S302+ 2026-04-19, C82 관련):
> - `review/round-N/` 하위의 `scorecard.json`, `actionable-items.json`, `sixhats-data.json` — req-interview 중간 산출물
> - `specs/{project}/review-history.md` — 인터뷰 라운드 요약, 자동 생성 산출물 (frontmatter 비대상으로 정식 명시. 이전엔 암묵적 비대상으로 처리됨)
> - `specs/{project}/interview-log.md` — 인터뷰 원본 로그, GOV 문서 아님
>
> 위 3종은 `/ax:gov-doc check`에서 NOFM으로 카운트되더라도 정상이에요. GOV 표준 적용 대상은 `prd-final.md`와 PDCA 디렉토리(01~04) + specs/ 직속 SPEC 코드 부여 문서들이에요.
