# ADR-000: PRD v4를 유일한 현행 명세로 확정

**Status**: Accepted
**Date**: 2026-03-16
**Author**: AX BD팀

## Context

- Foundry-X는 Discovery-X(탐색) → AI Foundry(구축) → Foundry-X(통합 협업)로 진화한 프로젝트
- PRD가 v1(인터뷰 초안) → v2(구체화) → v3(AI 검토 반영) → v4(synthnoosh harness bootstrap 반영)까지 4회 개정
- 다중 AI 검토 2라운드(ChatGPT, Gemini, Claude, Grok) 통과 후 착수 판정 완료
- Sprint 1~2 구현 완료, CLI MVP npm publish(foundry-x@0.1.1) 완료
- 현행 권위 문서에 대한 공식 선언이 부재하여, 어떤 PRD를 읽어야 하는지 불명확

## Decision

### D1. PRD v4 = 유일한 현행 명세

`docs/specs/prd-v4.md`가 Foundry-X의 유일한 PRD이다. 이전 버전(v1~v3)은 `docs/archive/`에 보존하되 참조 전용으로만 사용한다.

### D2. 모노리포 구조 확정

TypeScript(CLI) + Python(Plumb) 공존 모노리포 구조를 확정한다. pnpm workspace + Turborepo 기반. 근거: PRD v3 다중 AI 검토에서 Gemini, ChatGPT가 모노리포 시작을 권고.

### D3. 프로젝트 네이밍 공식화

Discovery-X(탐색의 정신) + AI Foundry(구축의 정신) = **Foundry-X**(통합 협업 플랫폼). "Foundry-X"가 유일한 공식 프로젝트명이다.

## Consequences

### 긍정

- 신규 팀원/에이전트가 읽어야 할 문서가 명확해짐 (prd-v4.md 하나)
- SSOT 원칙 강제 — "Git이 진실, Foundry-X는 렌즈" 철학의 문서 레벨 적용
- archive에서 PRD 진화 과정을 추적 가능 (의사결정 투명성 유지)

### 부정

- v1~v3의 유용한 논의가 archive 접근 없이는 보이지 않음 → Context의 참조 링크로 보완
- ADR 형식이 팀 내 아직 익숙하지 않음 → INDEX.md에 작성 가이드 포함
