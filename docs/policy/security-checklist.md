# KT DS 보안 체크리스트 (AI 서비스 운영)

## 1. 목적

Foundry-X를 KT DS 내부 및 고객사 환경에서 운영할 때 준수해야 할 보안 항목을 체크리스트로 정리해요.

## 2. 데이터 보안

- [ ] **SC-01**: 외부 AI API 호출 시 HTTPS(TLS 1.2+) 사용
- [ ] **SC-02**: API 키를 코드/DB에 저장하지 않고 Secrets Manager/환경변수에만 보관
- [ ] **SC-03**: PII(개인식별정보) 마스킹 미들웨어 활성화 확인
- [ ] **SC-04**: L4(Restricted) 등급 데이터가 외부 API로 전송되지 않는지 확인
- [ ] **SC-05**: 프롬프트 원본을 DB에 저장하지 않음 (SHA-256 해시만 보존)
- [ ] **SC-06**: AI 응답 캐시에 L3~L4 데이터가 저장되지 않는지 확인
- [ ] **SC-07**: 마스킹 전 원본 데이터가 로그에 기록되지 않는지 확인

## 3. 인증/인가

- [ ] **SC-08**: JWT 토큰 만료 시간 설정 (기본 1시간)
- [ ] **SC-09**: Refresh Token 만료 시간 설정 (기본 7일)
- [ ] **SC-10**: RBAC 역할 기반 접근 제어 활성화
- [ ] **SC-11**: Guest 역할의 AI API 호출 차단 확인
- [ ] **SC-12**: SSO/OAuth IdP 연동 시 PKCE 사용
- [ ] **SC-13**: API 키 로테이션 주기 설정 (90일 권장)

## 4. 감사/추적

- [ ] **SC-14**: 모든 AI 생성 활동에 대한 감사 로그 기록 확인
- [ ] **SC-15**: 감사 로그 보존 기간 설정 (최소 1년)
- [ ] **SC-16**: 감사 로그 삭제 불가 (soft delete만 허용) 확인
- [ ] **SC-17**: 에이전트 실행 이력이 추적 가능한지 확인
- [ ] **SC-18**: 코드 리뷰 승인 이력이 기록되는지 확인

## 5. 네트워크/인프라

- [ ] **SC-19**: CORS 허용 오리진이 최소 범위인지 확인
- [ ] **SC-20**: Rate Limiting 설정 (테넌트별 100 req/min 기본)
- [ ] **SC-21**: 외부 API 프록시 설정 확인 (사내망 환경)
- [ ] **SC-22**: Workers/Functions 환경 변수에 민감 정보가 노출되지 않는지 확인
- [ ] **SC-23**: D1/SQL 데이터베이스 접근이 서비스 레이어를 경유하는지 확인

## 6. 코드 품질/라이선스

- [ ] **SC-24**: AI 생성 코드의 사람 검토 완료 확인
- [ ] **SC-25**: GPL/AGPL 라이선스 오염 스캔 (자동화 전: 수동 확인)
- [ ] **SC-26**: 자동 커밋 비활성화 확인 (--no-verify 우회 < 20%)
- [ ] **SC-27**: 테스트 커버리지 기준 충족 확인

## 7. 운영/장애 대응

- [ ] **SC-28**: 장애 알림 채널 설정 (Slack/이메일)
- [ ] **SC-29**: 롤백 절차 문서화 및 테스트 완료
- [ ] **SC-30**: 비상 시 외부 AI API 차단 스위치 확인
- [ ] **SC-31**: 데이터 퍼지(삭제) API 작동 확인 (고객 요청 대응)
- [ ] **SC-32**: 정기 보안 점검 일정 수립 (분기 1회 권장)
