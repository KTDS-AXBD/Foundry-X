# Marker.io 피드백 가이드

Foundry-X 웹(fx.minu.best)에는 **Marker.io 위젯**이 내장되어 있어요.
화면에서 버그나 개선사항을 발견하면, 스크린샷과 함께 피드백을 바로 등록할 수 있어요.
등록된 피드백은 자동으로 GitHub Issue가 되고, AI Agent가 수정 PR을 생성해요.

---

## 1. 피드백 등록하기

### 1-1. 위젯 열기

fx.minu.best 어떤 페이지에서든 **화면 오른쪽 하단**에 Marker.io 위젯 버튼이 보여요.
버튼을 클릭하면 피드백 캡처 모드로 진입해요.

### 1-2. 스크린샷 캡처

1. 위젯 버튼 클릭 → 화면이 캡처 모드로 전환
2. **드래그**로 문제 영역을 선택하거나, **전체 화면** 캡처 선택
3. 선택 영역에 **화살표**, **박스**, **텍스트** 등 주석 도구로 표시 가능

### 1-3. 피드백 작성

캡처가 끝나면 피드백 작성 폼이 나타나요:

| 필드 | 설명 | 필수 |
|------|------|------|
| **제목** | 간단한 한 줄 요약 | O |
| **설명** | 상세 내용 (재현 경로, 기대 동작, 실제 동작 등) | - |
| **스크린샷** | 자동 첨부 (캡처한 이미지) | - |

> **팁**: 제목에 어떤 페이지/기능에서 발생한 문제인지 명시하면 처리가 빨라요.
> 예: "발굴 상세 페이지에서 탭 전환 시 빈 화면 표시됨"

### 1-4. 전송

**Send feedback** 버튼을 누르면 피드백이 전송돼요.
Marker.io가 자동으로 다음 정보를 함께 수집해요:

- 브라우저 정보 (Chrome/Safari/Edge 등)
- 화면 해상도
- 현재 URL
- Console 에러 로그 (있으면)

---

## 2. 피드백이 처리되는 과정

피드백이 전송되면 아래 파이프라인이 자동으로 동작해요:

```
Marker.io 위젯
  ↓ (피드백 전송)
GitHub Issue 자동 생성
  ↓ (Webhook → Foundry-X API)
  ↓ [Marker.io] 제목 패턴 또는 visual-feedback 라벨 감지
feedback_queue 테이블에 등록 (status: pending)
  ↓ (feedback-consumer.sh 폴링)
Claude Code Agent가 코드 분석 + 수정
  ↓
GitHub PR 자동 생성
  ↓
feedback_queue status → done (PR URL 기록)
```

### 상태 흐름

| 상태 | 의미 |
|------|------|
| **pending** | 대기 중 — Agent가 아직 처리하지 않음 |
| **processing** | Agent가 현재 처리 중 |
| **done** | 완료 — PR이 생성됨 |
| **failed** | 실패 — Agent가 PR을 생성하지 못함 (자동 재시도 최대 3회) |
| **skipped** | 수동으로 건너뜀 처리 |

---

## 3. 피드백 관리 대시보드 (Admin)

관리자(admin)는 **fx.minu.best/feedback-dashboard**에서 전체 피드백 현황을 확인할 수 있어요.

### 접근 방법

1. Foundry-X에 admin 계정으로 로그인
2. 좌측 사이드바 → **Admin** 그룹 → **피드백** 메뉴 클릭

### 대시보드 기능

| 기능 | 설명 |
|------|------|
| **상태 필터** | 전체 / 대기 / 처리중 / 완료 / 실패 탭으로 필터링 |
| **카드 목록** | Issue 번호, 제목, 상태 뱃지, 등록 시간 표시 |
| **카드 상세** | 카드 클릭 시 본문, 라벨, 에러 메시지, 재시도 횟수 확인 |
| **PR 링크** | 완료(done) 항목은 "PR 보기" 링크로 수정 내용 확인 |
| **재처리** | 실패(failed) 항목을 "재처리" 버튼으로 다시 대기(pending)로 전환 |
| **건너뛰기** | 불필요한 항목을 "건너뛰기" 버튼으로 skipped 처리 |

### GitHub Issue에서 직접 확인

Marker.io 피드백은 GitHub Issue로도 확인할 수 있어요:
- 리포: [KTDS-AXBD/Foundry-X](https://github.com/KTDS-AXBD/Foundry-X/issues)
- 라벨: `visual-feedback`
- 제목 패턴: `[Marker.io] ...`

Agent가 PR을 생성하면 해당 Issue에 자동 코멘트가 달려요.

## 4. 좋은 피드백 작성 팁

1. **한 건에 하나의 이슈만** — 여러 문제를 한 피드백에 넣지 말아요
2. **재현 경로를 명시** — "OO 페이지 → XX 버튼 클릭 → 에러 발생"
3. **스크린샷에 주석** — 화살표나 박스로 문제 위치를 표시해요
4. **기대 동작 vs 실제 동작** — "A가 되어야 하는데 B로 표시됨"
5. **브라우저 특이사항** — 특정 브라우저에서만 발생하면 기재해요

## 5. FAQ

### Q: 위젯이 보이지 않아요
Marker.io 위젯은 로그인 후 앱 레이아웃(사이드바가 있는 페이지)에서만 표시돼요.
랜딩 페이지나 로그인 페이지에는 위젯이 없어요.

### Q: 피드백을 등록했는데 Issue가 안 만들어져요
Marker.io → GitHub 연동은 Marker.io 프로젝트 설정에서 관리돼요.
프로젝트 ID: `69d07c3d8dffcc4911465244`. 문제가 지속되면 관리자에게 문의해요.

### Q: Agent가 PR을 못 만들면 어떻게 되나요?
- 자동으로 최대 **3회 재시도**돼요 (retry_count < 3인 failed 항목)
- 3회 실패 후에도 안 되면 관리자가 대시보드에서 수동 "재처리"하거나, 직접 코드를 수정해요

### Q: Agent가 수정할 수 있는 범위는?
- `packages/web/src/` 디렉토리만 수정 가능해요
- API, CLI, 설정 파일 등은 수정하지 않아요
- 복잡한 수정이 필요하면 개발자가 직접 처리해요

### Q: 피드백을 취소하고 싶어요
GitHub Issue를 직접 닫거나, 관리자가 대시보드에서 "건너뛰기" 처리하면 돼요.
