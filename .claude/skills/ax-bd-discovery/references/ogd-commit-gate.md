# 2-5 Commit Gate O-G-D 실행 가이드

> Commit Gate 진입 시, 2-1~2-4 산출물을 O-G-D 적대적 루프로 종합 검증하는 가이드

## 실행 조건

O-G-D Commit Gate 검증을 실행하려면 다음 조건을 충족해야 한다:

1. **최소 4단계 산출물 존재**: 2-0(분류) + 2-1~2-4 중 최소 3개 단계의 산출물이 완료
2. **사업성 체크포인트 이력**: 2-1~2-4 각 단계에서 Go/Pivot/Drop 판단이 기록됨
3. **Commit Gate 4질문 시작 전**: O-G-D를 먼저 실행 → 결과를 Commit Gate 논의에 활용

## 중간 발굴 보고서 자동 생성

O-G-D 실행 전, AI가 2-0~2-4 산출물을 종합하여 **중간 발굴 보고서**를 자동 생성한다.
이 보고서가 Generator의 초기 산출물(Round 0) 역할을 한다.

### 보고서 구조 (Rubric R1~R7 매핑)

| Rubric 항목 | 원본 단계 | 보고서 섹션 |
|-------------|-----------|------------|
| R1 시장 기회 | 2-2 수요 시장 검증 | §1. 시장 규모 및 타이밍 |
| R2 기술 실현성 | 2-1 레퍼런스 분석 + 2-4 아이템 도출 | §2. 기술 스택 및 PoC 가능성 |
| R3 경쟁 차별성 | 2-3 경쟁·자사 분석 | §3. 경쟁 환경 및 차별화 |
| R4 수익 모델 | 2-4 아이템 도출 (초기 BM 가설) | §4. 수익 모델 초안 |
| R5 규제/법률 리스크 | 2-1~2-4 전반 | §5. 규제 환경 및 리스크 |
| R6 실행 계획 | 2-4 아이템 도출 | §6. 실행 계획 초안 |
| R7 파트너십 시너지 | 2-0 AX BD 전략 적합성 + 2-3 | §7. 파트너십 및 시너지 |

### 누락 항목 처리

2-1~2-4에서 다루지 않은 Rubric 항목은 "미분석" 표시하고, O-G-D가 해당 항목을 보완한다.
예: 2-7 BM 정의 전이므로 R4 수익 모델은 초안 수준이 정상이다.

## Rubric R1~R7 ↔ Commit Gate 4질문 매핑

O-G-D 품질 보고서의 항목별 점수를 Commit Gate 4질문과 연결하여 논의 깊이를 높인다.

### Q1: "이 아이템에 앞으로 4주를 투자한다면, 그 시간이 아깝지 않을까요?"

**관련 Rubric**: R1(시장 기회) + R2(기술 실현성) + R6(실행 계획)

| 점수 구간 | 해석 |
|-----------|------|
| 3항목 평균 ≥ 0.8 | 시장·기술·실행 모두 충분 — 투자 가치 높음 |
| 3항목 평균 0.6~0.8 | 일부 보완 필요하나 추진 가능 |
| 3항목 평균 < 0.6 | 근거 부족 — 보완 후 재검토 권장 |

### Q2: "우리 조직이 이걸 해야 하는 이유가 명확한가요?"

**관련 Rubric**: R3(경쟁 차별성) + R7(파트너십 시너지)

| 점수 구간 | 해석 |
|-----------|------|
| 2항목 평균 ≥ 0.8 | 차별화 + 조직 적합성 충분 |
| 2항목 평균 0.6~0.8 | 차별점 있으나 조직 적합성 논의 필요 |
| 2항목 평균 < 0.6 | "왜 우리가?" 질문에 답이 약함 |

### Q3: "지금까지 Pivot한 부분이 있었다면, 그 방향 전환에 확신이 있나요?"

**관련 Rubric**: 전체 점수 변화 추이 (O-G-D 라운드별)

| 패턴 | 해석 |
|------|------|
| Round 0 → Round N 점수 상승 | 피드백 반영이 잘 됨 — Pivot 방향 적절 |
| 점수 정체 또는 역전 | 구조적 문제 가능성 — Pivot 방향 재검토 |

누적 사업성 신호등의 Pivot 이력과 교차 확인한다.

### Q4: "이 아이템이 안 되면, 우리가 잃는 것과 얻는 것은 뭔가요?"

**관련 Rubric**: R5(규제/법률 리스크) + R4(수익 모델)

| 점수 구간 | 해석 |
|-----------|------|
| R5 ≥ 0.7 + R4 ≥ 0.7 | 리스크 관리 가능 + 수익 근거 있음 — 실패 비용 제한적 |
| R5 < 0.5 or R4 < 0.5 | 리스크 미파악 또는 수익 근거 부족 — 실패 시 매몰 비용 우려 |

## O-G-D 결과 → Commit Gate 연결

### CONVERGED (Quality Score ≥ 0.85)

```
✅ O-G-D 검증 완료 — Quality Score: {score}

품질 충분. 아래 점수를 참고하여 Commit Gate 4질문을 논의합니다.
잔여 이슈: {minor_count}건 (참고 수준)

[Rubric 항목별 점수 표]
[Commit Gate 4질문별 연관 분석]
```

→ Commit Gate 4질문으로 자연스럽게 전환

### FORCED_STOP (Quality Score < 0.85)

```
⚠️ O-G-D 검증 — Quality Score: {score} (목표 0.85 미달)

보완 필요 영역:
- {critical/major findings 목록}

이 상태에서 Commit Gate를 진행하되, 미달 영역을 인지한 위에서 판단합니다.
대안: 해당 단계를 보완 후 O-G-D 재실행
```

→ Commit Gate 4질문은 진행하되, 미달 영역을 명시적으로 논의

### 핵심 원칙

**O-G-D는 판단을 돕지만, 대체하지 않는다.**

- O-G-D 점수가 높아도 담당자가 Drop할 수 있다 (조직 내부 사정, 정치적 요인)
- O-G-D 점수가 낮아도 담당자가 Commit할 수 있다 (전략적 판단, 선점 효과)
- O-G-D가 놓치는 것: 암묵지, 조직 역학, 타이밍의 직관, 인맥/관계 자산

## 산출물 보관

O-G-D 실행 결과는 `_workspace/` 디렉토리에 자동 생성된다:

```
_workspace/
├── rubric.md                     — 사용된 Rubric
├── ogd-state.yaml                — 루프 상태
├── ogd-report.md                 — 최종 품질 보고서
├── round-0/
│   ├── generator-artifact.md     — 중간 발굴 보고서 (초기)
│   └── discriminator-feedback.md — 평가 피드백
└── round-1/
    ├── generator-artifact.md     — 개선된 보고서
    └── discriminator-feedback.md — 재평가 피드백
```

아이템별 발굴이 완료되면, `_workspace/ogd-report.md`를 해당 아이템의 기록에 첨부한다.
