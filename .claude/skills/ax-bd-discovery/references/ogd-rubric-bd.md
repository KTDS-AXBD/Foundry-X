# O-G-D BD 발굴 Rubric 템플릿 (v1.1)

> Discriminator가 Generator 산출물을 평가할 때 사용하는 품질 기준

## 기본 Rubric (7항목)

| ID | 기준 | 가중치 | 임계값 | 평가 관점 |
|----|------|--------|--------|----------|
| R1 | 시장 기회 | 0.15 | ≥ 0.7 | TAM/SAM/SOM 수치 + 출처 신뢰성 + 성장성 + 시기적절성 |
| R2 | 기술 실현성 | 0.15 | ≥ 0.6 | 기술 스택 구체성 + PoC 가능 여부 + 핵심 기술 리스크 식별 |
| R3 | 경쟁 차별성 + 협상 레버리지 | 0.15 | ≥ 0.6 | 경쟁사 분석 완전성 + 차별화 포인트 + 우리의 협상 카드 |
| R4 | 수익 모델 | 0.15 | ≥ 0.6 | 과금 구조 + 단가 + 예상 매출 근거 + 투자 대비 수익 |
| R5 | 규제/법률 리스크 | 0.15 | ≥ 0.5 | 관련 법규 식별 + 대응 방안 + 운영 리스크 |
| R6 | 실행 계획 + 관계자 매핑 | 0.15 | ≥ 0.6 | 일정 + 자원 + 마일스톤 + 의사결정자/영향자 식별 |
| R7 | 파트너십 시너지 | 0.10 | ≥ 0.5 | 상호 보완성 + 문화 적합성 + 전략 일치도 |

## 종합 품질 점수

```
Quality Score = Σ (weight_i × criterion_score_i)
             = 0.15×R1 + 0.15×R2 + 0.15×R3 + 0.15×R4 + 0.15×R5 + 0.15×R6 + 0.10×R7

수렴 조건:
  Quality Score ≥ 0.85
  AND Critical 결함 = 0
  AND Major 결함 ≤ 1
  AND verdict = "PASS"
```

## 항목별 채점 가이드

### R1 시장 기회 (0.15)

| 점수 | 기준 |
|------|------|
| 0.9~1.0 | TAM/SAM/SOM 수치 + 공인 출처(Gartner/IDC/Statista) + CAGR + 시기적절성 근거 |
| 0.7~0.8 | TAM/SAM 수치 + 출처 있으나 일부 불명확 + 성장 추세 언급 |
| 0.5~0.6 | 시장 규모 언급은 있으나 출처 미비 + 정량적 근거 약함 |
| 0.3~0.4 | 시장 규모 추정만 있고 근거 없음 |
| 0.0~0.2 | 시장 분석 섹션 없음 또는 피상적 |

### R2 기술 실현성 (0.15)

| 점수 | 기준 |
|------|------|
| 0.9~1.0 | 기술 스택 명시 + 유사 PoC 사례 + 데이터셋 확보 방안 + 리스크 3가지 대응 |
| 0.7~0.8 | 기술 스택 명시 + PoC 가능성 설명 + 리스크 1~2개 |
| 0.5~0.6 | 기술 언급은 있으나 구체성 부족 |
| 0.3~0.4 | "AI로 해결" 수준의 추상적 기술 언급 |
| 0.0~0.2 | 기술 분석 없음 |

### R3 경쟁 차별성 + 협상 레버리지 (0.15)

| 점수 | 기준 |
|------|------|
| 0.9~1.0 | 주요 경쟁사 3사 이상 분석 + SWOT/기능 비교 + 명확한 차별화 + 협상 카드 식별 |
| 0.7~0.8 | 경쟁사 2~3사 분석 + 차별화 포인트 있으나 협상 레버리지 미약 |
| 0.5~0.6 | 경쟁사 1~2사 언급 + 차별화 모호 |
| 0.3~0.4 | "경쟁사 대비 우위" 수준의 추상적 주장 |
| 0.0~0.2 | 경쟁 분석 없음 |

### R4 수익 모델 (0.15)

| 점수 | 기준 |
|------|------|
| 0.9~1.0 | 과금 구조 + 단가 근거 + Bottom-up 매출 추정(Y1~Y3) + BEP 분석 |
| 0.7~0.8 | 과금 구조 + 매출 추정 있으나 가정 명시 부족 |
| 0.5~0.6 | 수익 모델 언급 있으나 구체적 수치 미비 |
| 0.3~0.4 | "구독 모델" 수준의 추상적 언급 |
| 0.0~0.2 | 수익 모델 없음 |

### R5 규제/법률 리스크 (0.15)

| 점수 | 기준 |
|------|------|
| 0.9~1.0 | 관련 법규 구체 명시 + 규제 동향 + 대응 방안 + 리스크 매트릭스 |
| 0.7~0.8 | 주요 법규 식별 + 대응 방안 개요 |
| 0.5~0.6 | 규제 환경 언급은 있으나 구체적 법규 미명시 |
| 0.3~0.4 | "규제 리스크 있음" 수준 |
| 0.0~0.2 | 규제 분석 없음 |

### R6 실행 계획 + 관계자 매핑 (0.15)

| 점수 | 기준 |
|------|------|
| 0.9~1.0 | 구체적 일정 + 자원 배분 + 마일스톤 + 의사결정자/영향자/반대자 식별 + 접근 경로 |
| 0.7~0.8 | 일정 + 마일스톤 + 핵심 관계자 1~2명 식별 |
| 0.5~0.6 | 대략적 일정은 있으나 관계자 매핑 없음 |
| 0.3~0.4 | "3개월 내 PoC" 수준의 추상적 계획 |
| 0.0~0.2 | 실행 계획 없음 |

### R7 파트너십 시너지 (0.10)

| 점수 | 기준 |
|------|------|
| 0.9~1.0 | 상호 보완성 분석 + 문화 적합성 + 전략 일치도 + 과거 협업 이력 |
| 0.7~0.8 | 시너지 근거 있으나 일부 항목 누락 |
| 0.5~0.6 | 파트너십 가치 언급은 있으나 근거 약함 |
| 0.3~0.4 | "파트너와 협력" 수준 |
| 0.0~0.2 | 파트너십 분석 없음 |

## 산업 템플릿 오버라이드

기본 가중치는 범용이며, 특정 산업/도메인에서는 아래처럼 조정한다:

### 규제 산업 (헬스케어, 핀테크)
```
R5 규제/법률 리스크: 0.15 → 0.20
R7 파트너십 시너지: 0.10 → 0.05
```

### 기술 파트너십
```
R2 기술 실현성: 0.15 → 0.20
R4 수익 모델: 0.15 → 0.10
```

### 유통 파트너십
```
R1 시장 기회: 0.15 → 0.20
R6 실행 계획: 0.15 → 0.20
R2 기술 실현성: 0.15 → 0.05
```

> 오버라이드 적용 시 Orchestrator가 `_workspace/rubric.md`에 조정된 가중치를 기록한다.
