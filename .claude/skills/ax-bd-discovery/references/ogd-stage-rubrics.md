## 기본 Rubric (참조)

기본 가중치는 `references/ogd-rubric-bd.md` 참조. 7항목 각 0.15 + R7만 0.10 (총합 1.00).

```
기본: R1(0.15) R2(0.15) R3(0.15) R4(0.15) R5(0.15) R6(0.15) R7(0.10) = 1.00
```

## 2-3 경쟁·자사 분석 오버라이드

경쟁 분석 보고서 검증에 최적화된 가중치. R3(경쟁 차별성)과 R1(시장 기회)을 강화한다.

| 항목 | 기본 | 2-3 오버라이드 | 변경 |
|------|------|---------------|------|
| R1 시장 기회 | 0.15 | **0.20** | +0.05 (경쟁 맥락에서 시장 이해 중요) |
| R2 기술 실현성 | 0.15 | **0.10** | -0.05 (이 단계에서 기술 깊이 낮음) |
| R3 경쟁 차별성 | 0.15 | **0.25** | +0.10 (핵심 초점) |
| R4 수익 모델 | 0.15 | **0.10** | -0.05 (2-7에서 상세화) |
| R5 규제/법률 | 0.15 | 0.15 | 유지 |
| R6 실행 계획 | 0.15 | 0.15 | 유지 |
| R7 파트너십 | 0.10 | **0.05** | -0.05 |
| **합계** | 1.00 | **1.00** | |

### 2-3 특화 평가 관점

R3 평가 시 다음을 추가로 확인:
- **경쟁사 3사 이상 분석 여부** (SWOT 또는 기능 비교표)
- **Disruption Risk 식별** (AI 네이티브 스타트업/빅테크 진입 가능성)
- **Imitation Difficulty Score** (모방 난이도 3단계 평가)
- **Blue Ocean ERRC 매트릭스** (제거/축소/증가/창조)

BMC/Unit Economics 검증에 최적화된 가중치. R4(수익 모델)와 R7(파트너십)을 강화한다.

### 2-7 특화 평가 관점

R4 평가 시 다음을 추가로 확인:

## 산업 템플릿과의 중첩

`ogd-rubric-bd.md`에 정의된 산업 템플릿(규제 산업, 기술 파트너십, 유통 파트너십)과 단계 오버라이드가 동시에 적용될 수 있다.

### 중첩 규칙: 곱연산 후 정규화

```
최종 가중치 = 기본 × 산업 계수 × 단계 계수
→ 정규화 (총합 = 1.00)

예: 2-3 경쟁 분석 + 규제 산업(헬스케어)
  R3 = 0.15 × (0.25/0.15) × 1.0 = 0.25
  R5 = 0.15 × (0.20/0.15) × 1.0 = 0.20
  → 나머지 정규화하여 총합 1.00
```

### 중첩 시 주의

- 2개 이상의 오버라이드가 같은 항목을 강화하면 과도한 편중 가능
- 정규화 후 **어떤 항목도 0.30을 초과하지 않도록** 상한선 적용
- 상한선 초과분은 가장 낮은 항목에 재분배

## 커스텀 오버라이드

담당자가 특정 항목을 더 강조하고 싶으면 Orchestrator 호출 시 context에 가중치를 직접 지정할 수 있다:

```
/ogd-orchestrator
task: "경쟁 분석 보고서 검증"
rubric_template: "bd-discovery"
context: "R3 가중치 0.30으로 강화, R2 0.05로 최소화"
max_rounds: 2
```

Orchestrator가 context를 파싱하여 `_workspace/rubric.md`에 조정된 가중치를 기록한다.
