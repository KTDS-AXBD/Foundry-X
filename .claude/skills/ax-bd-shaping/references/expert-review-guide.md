# Expert Review Guide — BD 형상화 Phase E

5종 전문가 AI 페르소나가 PRD를 리뷰하는 프로세스 가이드.

## 1. 전문가 리뷰 프로세스 개요

Phase D(교차 검토 + Six Hats) 통과 후, 5종 전문가가 **병렬로** PRD를 리뷰.
각 전문가는 독립적으로 평가하며, 교차 영향 분석으로 전문가 간 시너지/충돌을 식별.

```
Phase D 통과 PRD
    |
    v
[병렬] TA / AA / CA / DA / QA 리뷰 (Agent 5개 동시)
    |
    v
교차 영향 분석
    |
    v
통합 리뷰 보고서
    |
    v
수렴 판정 (Major 0 + Score >= 0.85)
```

## 2. 5종 Rubric 상세

## 3. Severity 분류 기준

| Severity | 정의 | Phase C 회귀 | 예시 |
|----------|------|--------------|------|
| Critical | 아키텍처/보안 결함, 즉시 해결 필수 | 즉시 | 인증 없는 API, SQL injection 가능성 |
| Major | 주요 품질 이슈 | 회귀 트리거 | 확장성 미고려, 데이터 모델 결함 |
| Minor | 개선 권고, 진행에 영향 없음 | 아님 | 네이밍 불일치, 문서 보완 필요 |
| Info | 참고 사항 | 아님 | 대안 기술 제안, 트렌드 정보 |

## 4. 교차 영향 분석 가이드

5종 전문가의 findings를 교차 분석하여 시너지와 충돌을 식별:

### 교차 영향 매트릭스

| From \ To | TA | AA | CA | DA | QA |
|-----------|-----|-----|-----|-----|-----|
| TA | — | API 계약 영향 | 인프라 요구사항 | 데이터 접근 패턴 | NFR 테스트 범위 |
| AA | 모듈 구조 제약 | — | 배포 단위 영향 | 데이터 흐름 설계 | API 테스트 범위 |
| CA | 확장 제약 조건 | API 게이트웨이 | — | 저장소 비용 영향 | 환경 분리 전략 |
| DA | 스키마 변경 영향 | 데이터 API 설계 | 스토리지 선택 | — | 데이터 검증 범위 |
| QA | 테스트 가능성 | API 테스트 | 인프라 테스트 | 데이터 검증 | — |

### 교차 분석 절차

1. 각 전문가의 Critical/Major findings 수집
2. 매트릭스 셀별로 영향 관계 판정:
   - **Synergy**: 한 전문가의 권고가 다른 전문가의 우려를 해소
   - **Conflict**: 두 전문가의 권고가 상충 (예: 보안 강화 vs 성능 최적화)
   - **Dependency**: 한 전문가의 권고 실행이 다른 영역에 선행 조건
3. Conflict 항목은 통합 리뷰에서 우선순위 결정

## 6. 수렴 기준

```
PASS 조건 (모두 충족):
  1. Critical findings = 0건
  2. 평균 Quality Score >= 0.85
  3. 개별 전문가 Score >= 0.70 (모두)

FAIL (하나라도 해당):
  - Critical >= 1건 → 즉시 Phase C 회귀
  - 평균 Score < 0.85 → Phase C 회귀 (Major findings 피드백)
  - 개별 Score < 0.70 → 해당 영역 집중 보강 후 재리뷰
```

## 7. Gotchas

- 5종 에이전트는 **haiku 모델**로 실행 — 비용 효율성 (병렬 5개 동시)
- 각 전문가는 PRD 전체가 아닌 **자기 영역 중심**으로 리뷰 — 범위 벗어난 findings는 Info로 분류
- 교차 영향 분석은 SKILL.md Step 5c에서 수행 (에이전트가 아닌 오케스트레이터 레벨)
- Score는 Rubric 4항목의 **단순 평균** (가중치 없음, 향후 도메인별 가중치 추가 가능)
