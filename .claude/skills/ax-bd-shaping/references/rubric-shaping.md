# BD 형상화 Rubric 템플릿 (v1.0)

> Discriminator가 Generator의 3단계 PRD를 평가할 때 사용하는 품질 기준

## 형상화 Rubric (5차원)

| ID | 차원 | 가중치 | 임계값 | 평가 관점 |
|----|------|--------|--------|-----------|
| S1 | 사업 타당성 | 0.25 | >= 0.70 | 시장 규모 근거 + 경쟁 우위 명확성 + 수익 모델 구체성 |
| S2 | 기술 실현성 | 0.25 | >= 0.70 | 아키텍처 완성도 + 기술 스택 적합성 + PoC 가능성 |
| S3 | 요구사항 추적성 | 0.20 | >= 0.65 | BR->Feature 매핑 + TC->아키텍처 반영 + SC->KPI 연결 |
| S4 | 리스크 커버리지 | 0.15 | >= 0.60 | 기술/시장/조직 리스크 식별 + 완화 전략 구체성 |
| S5 | 문서 완성도 | 0.15 | >= 0.65 | 섹션 완전성 + 용어 일관성 + 참조 무결성 |

## 종합 품질 점수

```
Quality Score = 0.25*S1 + 0.25*S2 + 0.20*S3 + 0.15*S4 + 0.15*S5

수렴 조건:
  Quality Score >= 0.85
  AND Critical 결함 = 0
  AND Major 결함 <= 1
  AND verdict = "PASS"

강제 종료:
  round >= max_rounds (기본 3) -> best artifact 채택 + residual findings 첨부
```

## 항목별 채점 가이드

### S1 사업 타당성 (0.25)

| 점수 | 기준 |
|------|------|
| 0.9~1.0 | TAM/SAM/SOM 수치 + 공인 출처 + CAGR + 경쟁사 3사+ SWOT + 과금 구조 + Y1~Y3 매출 추정 + BEP |
| 0.7~0.8 | 시장 규모 + 출처 있으나 일부 불완전 + 경쟁사 2사+ + 수익 모델 개요 |
| 0.5~0.6 | 시장 언급 + 경쟁사 1~2사 + 수익 모델 추상적 |
| 0.3~0.4 | 시장/경쟁/수익 중 2개 이상 근거 없음 |
| 0.0~0.2 | 사업 타당성 분석 거의 없음 |

### S2 기술 실현성 (0.25)

| 점수 | 기준 |
|------|------|
| 0.9~1.0 | 시스템 구조도 + 기술 스택 + 선정 근거 + 데이터 모델 + PoC 사례 + 기술 리스크 3가지 대응 |
| 0.7~0.8 | 기술 스택 명시 + 아키텍처 개요 + PoC 가능성 설명 + 리스크 1~2개 |
| 0.5~0.6 | 기술 언급 있으나 아키텍처 구체성 부족 |
| 0.3~0.4 | "AI/클라우드 활용" 수준의 추상적 기술 언급 |
| 0.0~0.2 | 기술 분석 없음 |

### S3 요구사항 추적성 (0.20)

| 점수 | 기준 |
|------|------|
| 0.9~1.0 | BR->Feature 100% 매핑 + TC->NFR/아키텍처 반영 + SC->KPI 연결 + 마일스톤 일정 구체적 |
| 0.7~0.8 | BR 80%+ 매핑 + TC 대부분 반영 + SC->KPI 연결 일부 누락 |
| 0.5~0.6 | BR 50%+ 매핑 + TC 일부 반영 + 마일스톤 대략적 |
| 0.3~0.4 | 요구사항 언급은 있으나 매핑 테이블 없음 |
| 0.0~0.2 | 요구사항 추적 없음 |

### S4 리스크 커버리지 (0.15)

| 점수 | 기준 |
|------|------|
| 0.9~1.0 | 기술/시장/조직/규제 리스크 식별 + 영향도x확률 매트릭스 + 구체적 완화 전략 |
| 0.7~0.8 | 주요 리스크 3개+ 식별 + 완화 전략 개요 |
| 0.5~0.6 | 리스크 언급은 있으나 완화 전략 모호 |
| 0.3~0.4 | "리스크 있음" 수준 |
| 0.0~0.2 | 리스크 분석 없음 |

### S5 문서 완성도 (0.15)

| 점수 | 기준 |
|------|------|
| 0.9~1.0 | 9개 섹션 완전 + 용어 일관 + 교차 참조 무결 + 출처 완비 + 가정 명시 |
| 0.7~0.8 | 8개+ 섹션 + 용어 대체로 일관 + 출처 대부분 |

| 0.3~0.4 | 절반 미만 섹션 + 구조 불완전 |
| 0.0~0.2 | 문서 체계 없음 |

## 발굴 Rubric(R1~R7)과의 관계

| 발굴 (R) | 형상화 (S) | 관계 |
|-----------|-----------|------|
| R1 시장 기회 | S1 사업 타당성 | R1이 S1의 입력 (시장 데이터 -> 사업 모델로 확장) |
| R2 기술 실현성 | S2 기술 실현성 | R2 기초 판단 -> S2 상세 아키텍처로 심화 |
| R3 경쟁 차별성 | S1 사업 타당성 | R3이 S1에 흡수 (경쟁 우위 = 사업 타당성의 일부) |
| R4 수익 모델 | S1 사업 타당성 | R4가 S1에 흡수 |
| R5 규제/법률 | S4 리스크 커버리지 | R5가 S4에 흡수 |
| R6 실행 계획 | S3 요구사항 추적성 | R6 -> S3 (실행 계획 = 요구사항 실현 경로) |
| R7 파트너십 | S4 리스크 커버리지 | R7이 S4에 흡수 (파트너 리스크) |

> 발굴 R1~R7은 7개 독립 항목이었지만, 형상화 S1~S5는 관련 항목을 통합하여 5차원으로 압축. 이는 형상화 단계에서 사업 타당성(S1)과 요구사항 추적성(S3)이 더 중요하기 때문.
