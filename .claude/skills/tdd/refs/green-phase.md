# GREEN Phase — 테스트를 통과하는 최소 구현

> "동작하는 코드를 가장 빠르게 만들어라. 아름다움은 나중에."

## 원칙

1. **최소 구현**: 테스트가 요구하는 것만 구현한다
2. **YAGNI**: 향후 필요할 수 있는 기능을 미리 추가하지 않는다
3. **기존 보존**: 기존 테스트를 깨뜨리지 않는다

## 구현 규칙

### 1. 최소의 정의

- 테스트가 통과하는 가장 단순한 코드
- 하드코딩도 허용 (Refactor에서 개선)
- 불필요한 추상화 금지
- 에러 처리는 테스트가 요구하는 범위만

### 2. 구현 순서

```
1. 테스트 파일에서 import하는 함수 확인
2. 함수 시그니처 작성 (타입 포함)
3. happy path 테스트 통과하는 구현
4. edge case 테스트 통과하는 분기 추가
5. error case 테스트 통과하는 에러 처리 추가
```

### 3. 검증

```bash
# 새 테스트만 실행
pnpm test -- --grep "{테스트파일명}"

# 전체 테스트 통과 확인
pnpm test

# 타입 체크
pnpm typecheck
```

### 4. 기존 테스트 깨짐 대응

기존 테스트가 깨지면:

1. 새 구현이 기존 동작을 변경했는지 확인
2. 기존 동작이 보존되도록 구현 수정
3. 기존 테스트의 기대값이 잘못된 경우가 아닌지 확인
4. 절대로 기존 테스트를 수정하여 통과시키지 않는다
