# 스케일업 플레이북 (Scale Playbook)

사용자가 제시한 AI 사업 아이템의 **파일럿 → 상용화 전환 전략**을 수립하세요.

## 전환 프레임워크

### Phase 1: 파일럿 졸업 판정
- **파일럿 결과 리뷰**: 성공 기준 달성 여부
- **고객 피드백 종합**: NPS, 재계약 의향, 개선 요청
- **기술 안정성**: 장애율, 응답 시간, 정확도 추이
- **Go/No-Go 판정**: 경영진 보고 및 승인

### Phase 2: 상용화 준비 (0~3개월)
- **제품 보강**:
  - 파일럿 피드백 기반 기능 개선
  - 엔터프라이즈 요구사항 (SSO, 감사 로그, SLA)
  - 멀티테넌시, 확장성, 보안 강화
- **운영 체계**:
  - CS/기술지원 프로세스 수립
  - 모니터링/알림 체계 구축
  - 장애 대응 매뉴얼
- **영업 준비**:
  - 가격표 확정
  - 영업 자료 (데모, 제안서, ROI 계산기)
  - 파일럿 성공 사례 정리

### Phase 3: 초기 고객 확보 (3~9개월)
- **Beachhead 전략**: 1차 타겟 세그먼트 집중 공략
- **레퍼런스 확보**: 파일럿 고객 → 정식 계약 전환
- **파이프라인 구축**: 영업 파이프라인 10배 룰
- **성공 지표**: 월간 신규 고객 수, ARR, 파이프라인 규모

### Phase 4: 성장 가속 (9~18개월)
- **세그먼트 확장**: 2차, 3차 타겟 세그먼트로 확장
- **제품 확장**: 추가 기능/모듈, 연동 확대
- **채널 확장**: 직접 영업 + 파트너 채널
- **조직 확장**: 전담 팀 구성, 채용 계획

### Phase 5: 성숙/최적화 (18개월+)
- **운영 효율화**: 자동화, 셀프서비스 확대
- **수익 최적화**: 업셀/크로스셀, 가격 인상
- **플랫폼화**: 에코시스템 구축, API/마켓플레이스
- **해자 강화**: 데이터/네트워크 효과 극대화

## 핵심 체크리스트

### 상용화 전 필수 항목
- [ ] 가격 정책 확정
- [ ] SLA 정의 및 보증 체계
- [ ] 법무 검토 완료 (이용약관, 개인정보처리방침)
- [ ] 보안 점검 완료 (취약점 진단, 침투 테스트)
- [ ] CS/기술지원 체계 가동
- [ ] 모니터링/장애 대응 체계 가동
- [ ] 영업 자료 및 데모 환경 준비
- [ ] 파일럿 고객 전환 계약 준비

## 출력 형식
1. **전환 판정 리포트** (파일럿 결과 요약)
2. **상용화 로드맵** (Phase별 타임라인)
3. **필수 체크리스트** (항목별 상태)
4. **리소스 계획** (인력/예산/인프라)
5. **KPI 대시보드** (Phase별 핵심 지표)

$ARGUMENTS

## Gotchas

- **파일럿→상용화 전환율은 산업 평균 20~30% — 전환 실패 원인 Top 3: 조직 저항, 데이터 품질, 비용 초과. Phase Gate를 명확히 설정하고 각 Gate에서 Kill 옵션 유지**
- **KT DS 맥락 유지**: 모든 분석은 KT DS의 SI/SM 역량, KT 그룹 고객 기반을 고려해야 함. 일반적인 스타트업 관점과 다름
- **HITL 필수**: AI 분석 초안 제공 후 반드시 담당자 검증 요청. 결과를 그대로 수용하지 않음
- **산출물 언어**: 한국어 작성, 프레임워크 용어는 원어 병기
