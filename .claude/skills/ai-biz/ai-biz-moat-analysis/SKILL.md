# AI 경쟁 해자 분석 (Moat Analysis)

사용자가 제시한 AI 사업 아이템의 **지속 가능한 경쟁 우위(Moat)를 분석**하세요.

## 해자 유형별 분석

### 1. 데이터 해자 (Data Moat)
- **독점 데이터**: 경쟁사가 쉽게 확보하기 어려운 고유 데이터가 있는가?
- **데이터 플라이휠**: 사용자가 늘수록 데이터가 풍부해지고, 데이터가 풍부해질수록 서비스가 좋아지는 선순환이 가능한가?
- **데이터 네트워크 효과**: 다수 사용자의 데이터가 결합되어 개별 사용자에게 더 큰 가치를 제공하는가?
- **전환 비용**: 고객이 축적한 데이터가 이탈을 어렵게 만드는가?
- **해자 강도 평가**: 약함 / 보통 / 강함 / 매우 강함

### 2. 기술 해자 (Technology Moat)
- **알고리즘 우위**: 독자 알고리즘/모델이 경쟁사 대비 성능 우위가 있는가?
- **특허/IP**: 핵심 기술의 지식재산권 보호가 가능한가?
- **엔지니어링 복잡도**: 쉽게 복제하기 어려운 기술적 복잡성이 있는가?
- **인재 해자**: 핵심 기술 인력 확보 및 유지 경쟁력
- **해자 강도 평가**: 약함 / 보통 / 강함 / 매우 강함

### 3. 네트워크 해자 (Network Moat)
- **직접 네트워크 효과**: 같은 측 사용자가 늘수록 가치 증가 (예: 커뮤니티)
- **간접 네트워크 효과**: 양면 시장에서 한쪽이 늘면 다른 쪽 가치 증가 (예: 마켓플레이스)
- **플랫폼 효과**: 서드파티 개발자/파트너 생태계 구축 가능성
- **해자 강도 평가**: 약함 / 보통 / 강함 / 매우 강함

### 4. 브랜드/신뢰 해자 (Brand Moat)
- **KT 그룹 브랜드**: 대기업 신뢰도, 공공/엔터프라이즈 영업력
- **도메인 전문성**: 특정 산업에서의 전문성과 레퍼런스
- **규제 대응**: 컴플라이언스/인증이 진입 장벽이 되는가?
- **해자 강도 평가**: 약함 / 보통 / 강함 / 매우 강함

### 5. 규모 해자 (Scale Moat)
- **비용 우위**: 규모가 커질수록 단위 비용이 낮아지는 구조인가?
- **학습 곡선**: 운영 경험이 쌓일수록 효율이 높아지는가?
- **전환 비용**: 고객이 한번 도입하면 바꾸기 어려운 구조인가?
- **해자 강도 평가**: 약함 / 보통 / 강함 / 매우 강함

## 종합 분석

### 해자 포트폴리오
- 각 해자 유형의 현재 강도와 구축 가능성을 5점 척도로 평가
- 해자 간 시너지 분석 (데이터 해자 + 네트워크 해자 등)
- 시간에 따른 해자 강화/약화 전망

### 해자 구축 전략
- 단기 (0~6개월): 즉시 구축 가능한 해자
- 중기 (6~18개월): 점진적 강화 해자
- 장기 (18개월+): 시간이 지날수록 강해지는 해자

## 출력 형식
1. **해자 유형별 평가표** (5개 유형 × 강도/근거/전략)
2. **해자 레이더 차트** (텍스트 기반)
3. **경쟁사 대비 해자 비교**
4. **해자 구축 로드맵**
5. **리스크**: 해자가 무너질 수 있는 시나리오

$ARGUMENTS

## Gotchas

- **해자는 시간이 지나야 형성됨 — 초기 스타트업은 해자가 없는 것이 정상. '해자가 없다'가 곧 'Drop'은 아님, 해자 구축 계획이 있는지가 핵심**
- **KT DS 맥락 유지**: 모든 분석은 KT DS의 SI/SM 역량, KT 그룹 고객 기반을 고려해야 함. 일반적인 스타트업 관점과 다름
- **HITL 필수**: AI 분석 초안 제공 후 반드시 담당자 검증 요청. 결과를 그대로 수용하지 않음
- **산출물 언어**: 한국어 작성, 프레임워크 용어는 원어 병기
