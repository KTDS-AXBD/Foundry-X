# AI 규제 / 컴플라이언스 체크 (Regulation Check)

사용자가 제시한 AI 사업 아이템의 **규제 리스크와 컴플라이언스 요구사항**을 점검하세요.

## 체크리스트 프레임워크

### 1. AI 기본법 (한국)
- [ ] AI 시스템 위험 등급 분류 (고위험/일반)
- [ ] 고위험 AI 해당 시 영향평가 의무 확인
- [ ] AI 사용 사실 고지 의무 (투명성)
- [ ] 알고리즘 설명가능성 요구 수준
- [ ] AI 윤리 가이드라인 준수 여부

### 2. 개인정보보호법 / GDPR
- [ ] 개인정보 수집·이용 동의 필요 범위
- [ ] 가명처리·익명처리 필요 데이터 식별
- [ ] 개인정보 국외 이전 해당 여부
- [ ] 자동화된 의사결정에 대한 거부권 보장
- [ ] 개인정보 영향평가 필요 여부
- [ ] DPO(개인정보보호책임자) 지정 필요성

### 3. 산업별 규제
- **금융**: 금융위 AI 가이드라인, 신용정보법
- **의료**: 의료기기법, SaMD 인허가
- **공공**: 디지털 플랫폼 정부법, 공공데이터법
- **교육**: 교육기본법, 학습데이터 보호
- **제조**: 산업안전보건법, 제조물책임법

### 4. 지식재산권
- [ ] AI 생성물 저작권 이슈
- [ ] 학습 데이터 저작권 리스크
- [ ] 특허 침해 가능성
- [ ] 영업비밀 보호 방안

### 5. 계약/책임
- [ ] AI 오류 시 책임 소재 (개발사 vs 운영사 vs 사용자)
- [ ] SLA 및 성능 보증 범위
- [ ] 면책 조항 설계
- [ ] 보험 가입 필요성

## 출력 형식
1. **규제 리스크 매트릭스** (규제별 해당 여부 / 리스크 수준 / 대응 방안)
2. **필수 컴플라이언스 체크리스트** (항목별 상태: 충족/미충족/해당없음)
3. **리스크 등급** (High/Medium/Low)
4. **규제 대응 로드맵** (사전 준비 → 출시 전 → 운영 중)
5. **법무 검토 필요 사항** (외부 자문 권고 영역)

$ARGUMENTS

## Gotchas

- **AI 규제는 급변 중 — 2026년 기준 EU AI Act 시행, 한국 AI기본법 제정 진행 중. 분석 시점의 규제를 명시하고 분기별 재검토 필요. 이 스킬은 법률 자문을 대체하지 않음**
- **KT DS 맥락 유지**: 모든 분석은 KT DS의 SI/SM 역량, KT 그룹 고객 기반을 고려해야 함. 일반적인 스타트업 관점과 다름
- **HITL 필수**: AI 분석 초안 제공 후 반드시 담당자 검증 요청. 결과를 그대로 수용하지 않음
- **산출물 언어**: 한국어 작성, 프레임워크 용어는 원어 병기
