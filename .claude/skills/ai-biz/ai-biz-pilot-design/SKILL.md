# PoC / 파일럿 설계 (Pilot Design)

사용자가 제시한 AI 사업 아이템의 **PoC 또는 파일럿 프로젝트를 설계**하세요.

## 설계 프레임워크

### 1. 파일럿 목적 정의
- **검증 가설**: 이 파일럿으로 무엇을 증명하려는가?
- **핵심 질문**: 답을 얻어야 하는 3가지 질문
- **범위**: PoC(기술 검증) vs Pilot(사업 검증) vs MVP(시장 검증) 중 어디에 해당하는지

### 2. 성공 기준 (Success Criteria)
- **정량 지표**: 정확도, 처리 시간, 비용 절감율, 사용자 만족도 등
- **정성 지표**: 사용자 피드백 품질, 내부 이해관계자 수용도
- **Kill Criteria**: 이 수치 이하면 중단하는 기준
- **Go 기준**: 이 수치 이상이면 본 사업 전환

### 3. 실행 계획
- **기간**: 전체 일정 (통상 4~12주)
- **단계**: Phase 1(환경 구축) → Phase 2(개발/적용) → Phase 3(평가)
- **참여자**: 내부 팀 + 파일럿 고객 + 외부 파트너
- **리소스**: 인력, 인프라, 예산, 데이터

### 4. 파일럿 고객 선정
- **선정 기준**: Pain Point 심각도, 협조 의지, 데이터 접근성
- **고객 수**: 1~3개사 권장
- **계약 형태**: 무상 PoC / 할인 파일럿 / 공동 개발

### 5. 리스크 관리
- **기술 리스크**: 모델 성능 미달, 데이터 품질 이슈
- **운영 리스크**: 일정 지연, 고객 이탈
- **사업 리스크**: 경쟁사 선점, 내부 우선순위 변경
- **완화 방안**: 각 리스크별 Plan B

### 6. 평가 및 전환
- **평가 회의**: 파일럿 종료 후 Go/No-Go 판정 프로세스
- **전환 계획**: 파일럿 → 정식 서비스 전환 시 필요 사항
- **스케일업 요건**: 인프라, 인력, 프로세스 확장 계획

## 출력 형식
1. **파일럿 개요** (1페이지 요약)
2. **성공 기준표** (지표/목표치/측정방법)
3. **실행 타임라인** (주차별 마일스톤)
4. **리스크 매트릭스** (확률×영향도)
5. **Go/No-Go 판정 체크리스트**
